# Practice Ladder Pilot Implementation Plan

> **For Codex:** REQUIRED SUB-SKILL: Use `superpowers:executing-plans` to implement this plan task-by-task.

**Goal:** Xây nền móng đáng tin cậy và hai Practice Ladder hoàn chỉnh (`python-fibonacci`, `numpy-reshape-matrix`) để giúp người mới chuyển từ hiểu bài sang tự giải challenge và đo được mức độ độc lập.

**Architecture:** Giữ `/practice/[id]` và challenge system hiện tại, thêm ladder content versioned tùy chọn. Progress checkbox chuyển từ positive-only set union sang per-item LWW state có outbox/reset epoch; ladder progress dùng event bất biến, idempotent. Python và JavaScript chạy trong Worker có timeout/cancel; React sở hữu code state và đồng bộ CodeMirror bằng guarded transaction.

**Tech Stack:** Next.js 16, React 19, TypeScript strict, Supabase/PostgreSQL/RLS/RPC, Vitest, CodeMirror 6, Pyodide 0.26.4, Web Workers, pnpm.

**Approved spec:** `docs/superpowers/specs/2026-07-10-practice-ladder-design.md`

**Plan artifact:** Khi bắt đầu implementation, lưu kế hoạch tại `docs/plans/2026-07-11-practice-ladder-pilot.md` trước khi sửa production code.

---

## Locked decisions

- Pilot report chỉ tính event phát sinh khi user đã đăng nhập. Anonymous vẫn học/resume local; event imported sau login giữ `origin: "anonymous"` và bị loại khỏi cohort mặc định.
- Hai vertical slice đầu tiên: `python-fibonacci`, `numpy-reshape-matrix`.
- Không thêm AI tutor, gamification, secure grading, CMS hoặc analytics SaaS.
- Walkthrough unlock khi solved hoặc đủ 3 failed Submit; endpoint chỉ là UX gate.
- Không commit/push nếu người dùng chưa yêu cầu riêng.

## Core contracts

```ts
type ProgressItemState = {
  scope: "lesson" | "project_feature";
  itemKey: string;
  completed: boolean;
  clientUpdatedAt: string;
  mutationId: string;
};

type ProgressSyncStatus = "local-only" | "syncing" | "synced" | "failed";

type PracticeEvent = {
  eventId: string;
  challengeId: string;
  contentVersion: string | null;
  origin: "anonymous" | "authenticated";
  eventType: "challenge_started" | "step_viewed" | "step_completed" |
    "run" | "submit" | "hint_opened" | "walkthrough_opened" |
    "challenge_passed" | "transfer_passed";
  step: PracticeLadderStep | null;
  hintLevel: 0 | 1 | 2 | 3 | null;
  passed: boolean | null;
  occurredAt: string;
};
```

LWW order là `(clientUpdatedAt, mutationId)`. Public `completed` và `projectFeatures` vẫn được expose để tránh sửa hàng loạt component, nhưng luôn derive từ item states.

### Task 3: Supabase authoritative state, events, reset epoch

**Files:** create `supabase/migrations/202607110001_progress_lww_practice_events.sql`, `lib/progress-remote.test.ts`; modify `lib/progress-remote.ts`.

4. Enable RLS. Mutations go through authenticated RPCs:
   - `apply_progress_item_mutations(expected_epoch, mutations)` with conditional LWW upsert;
   - `append_practice_events(expected_epoch, events)` with `ON CONFLICT DO NOTHING`;
   - `reset_learning_progress(expected_epoch)` that locks, increments epoch and deletes progress transactionally.
5. Backfill positive legacy snapshot/normalized rows; keep legacy schema for rollback window.
6. Remote adapter stops browser read→union→snapshot for lesson/project and stops browser mirroring positive-only normalized rows.
7. Mock-Supabase tests cover payload mapping, ack, duplicate event and epoch mismatch.

**Run:** `pnpm exec vitest run lib/progress-remote.test.ts`

**Gate:** apply/test migration on staging with two accounts before production; do not auto-apply remote migrations.
