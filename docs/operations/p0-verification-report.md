# P0 Final Verification Report

> **Template status:** This file is **not evidence of a completed release** and makes no P0-ready claim until a named candidate SHA, approved environment, command outputs, reviewer-accessible evidence links, operator, and ISO date populate **every required row**. Do not replace missing evidence with `PASS`. **Default: P0 Not Ready — evidence pending.**

## Result vocabulary

- **PASS** — The documented procedure completed successfully and the linked sanitized evidence is available to release reviewers.
- **BLOCKED** — A required environment, credential, service, command, assertion, or result is unavailable or failed. This blocks P0 readiness.
- **N/A** — Permitted only for an objectively inapplicable criterion with explicit justification and reviewer approval recorded in the row.

`N/A` cannot bypass the local database test, staging two-user Supabase/RLS/RPC proof, or authenticated end-to-end proof. Those required rows must be `PASS`; they must never be skipped or marked `N/A`.

## Evidence and artifact-link policy

Each evidence link must be reviewer-accessible for the retention period, either as a protected CI/staging artifact URL or a repository-relative sanitized output path. Link to the precise command output, Playwright report, migration/change record, redacted diagnostics review, or signed manual-check record—not merely a dashboard home page.

Before linking an artifact, sanitize it. Artifacts and report cells must not contain credentials, service-role values, tokens, passwords, session cookies, raw learner code, private notes, quiz answers, or personally identifying test data. Use stable error codes, redacted metadata, and test-user aliases instead. A link that reviewers cannot access, is not specific enough to verify the row, or exposes prohibited material makes that row `BLOCKED`.
