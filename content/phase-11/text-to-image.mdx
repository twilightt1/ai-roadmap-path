Bạn đã hiểu Diffusion Models ở mức cơ chế: noise, denoising, scheduler, CFG, latent space và ControlNet. Nhưng trong sản phẩm thật, rất ít team tự train hoặc tự viết sampling loop từ đầu. Phần lớn AI Engineers tích hợp text-to-image qua API hoặc inference service như DALL-E, Stable Diffusion, Flux, Replicate, Hugging Face endpoints hoặc backend GPU nội bộ.

Text-to-Image API không phải chỉ là `POST /generate`. Đây là một product pipeline bất đồng bộ: prompt moderation, style controls, queueing, provider calls, asset storage, output safety, cost budget, retry/re-roll policy và user experience khi generation mất 10-60 giây.

## 1. The Incident (Tình huống sự cố)

<Callout type="danger" title="Báo cáo sự cố: Gọi image API đồng bộ làm backend timeout và bill tăng vọt">
**Bối cảnh:** Một startup launch tính năng tạo ảnh marketing bằng API sinh ảnh. Backend nhận prompt từ frontend, gọi provider trực tiếp trong request HTTP, chờ ảnh xong rồi trả URL về client.

**Sự cố:** Khi traffic tăng, nhiều request giữ kết nối 30-45 giây rồi timeout. User bấm generate lại nhiều lần, tạo duplicate jobs. Một số prompt unsafe bị gửi lên provider trước khi moderation, vẫn bị tính phí dù output bị chặn.

1. Image generation chạy đồng bộ trên request thread.
2. Không có job id, queue, polling/webhook hoặc idempotency key.
3. Không có pre-generation moderation và user quota.
4. Lưu URL tạm của provider nên ảnh biến mất sau vài giờ.

**Hậu quả:** Backend quá tải, provider cost vượt ngân sách tuần trong một ngày, và team phải rebuild pipeline theo mô hình async job.
</Callout>

Bài học rút ra: sinh ảnh là workload chậm, đắt và rủi ro. Production cần async architecture, moderation hai chiều, asset lifecycle và cost guardrails.

## 2. Architecture Blueprint (Async Image Generation Pipeline)

Một hệ thống text-to-image production nên tách request khỏi generation job:

```text
[ Frontend request ]
  - prompt
  - style preset
  - optional image/mask
        │
        ▼
[ API Gateway ]
  - auth
  - quota/rate limit
  - idempotency key
        │
        ▼
[ Pre-generation safety ]
  - prompt moderation
  - brand/copyright rules
  - policy decision
        │
        ▼
[ Job queue ]
  - status: queued/running/completed/failed
  - retry policy
  - timeout
        │
        ▼
[ Worker ]
  - prompt rewrite
  - provider routing
  - API/self-host call
        │
        ▼
[ Output processing ]
  - download provider asset
  - upload to S3/R2/CDN
  - output moderation
  - watermark/provenance metadata
        │
        ▼
[ Notify user ]
  - polling
  - webhook
  - WebSocket/SSE
```

Mục tiêu: mỗi image request có lifecycle rõ, có thể retry an toàn, audit được và không làm nghẽn request path chính.

## 3. Provider Map: DALL-E, Stable Diffusion, Flux, Ideogram, Midjourney-style UX

Không có provider nào tốt nhất cho mọi sản phẩm.

| Provider / model family | Điểm mạnh | Giới hạn | Khi chọn |
|---|---|---|---|
| DALL-E-style APIs | Prompt adherence tốt, managed safety | Ít control sâu, cost cao hơn | App cần launch nhanh và UX ổn định |

Chọn provider theo constraint:

1. Latency SLA.
2. Cost per successful image.
3. Control: seed, negative prompt, ControlNet, mask, image-to-image.
4. Safety/compliance requirements.
5. Data retention/privacy.
6. Asset ownership và licensing.

## 4. Prompt Engineering cho Image Generation

Một prompt ảnh tốt thường cụ thể hơn prompt text.

```text
Subject: premium ceramic coffee mug on wooden table
Composition: centered product shot, shallow depth of field
Lighting: soft natural window light, warm highlights
Style: minimalist Scandinavian commercial photography
Camera: 85mm lens, f/2.8, high detail
Constraints: no text, no logo, no watermark
```

### Style controls

Thay vì để user viết prompt tự do hoàn toàn, product nên cung cấp controls:

| Control | Ví dụ | Lợi ích |
|---|---|---|
| Style preset | photorealistic, watercolor, 3D render | Giảm prompt ambiguity |
| Aspect ratio | 1:1, 16:9, 9:16 | Phù hợp social channels |
| Quality preset | fast, balanced, high | Kiểm soát cost/latency |
| Brand palette | colors, fonts, mood | Consistency |
| Negative prompt | watermark, blurry, extra fingers | Giảm lỗi phổ biến |

### Negative prompts

Negative prompt giúp mô hình tránh chi tiết không mong muốn:

```text
blurry, low quality, distorted hands, extra fingers, watermark, text artifacts, logo
```

Nhưng negative prompt quá dài có thể làm model mất focus hoặc tạo output cứng nhắc.

## 5. Image-to-Image, Inpainting và Control Inputs

Text-to-image chỉ là một mode. Product creative thường cần chỉnh sửa ảnh có sẵn.

| Mode | Input | Use case | Control chính |
|---|---|---|---|
| Image-to-image | image + prompt | biến đổi style, tạo variation | `strength` |
| Inpainting | image + mask + prompt | sửa vùng nhỏ | mask quality |
| Outpainting | image + canvas extension | mở rộng khung hình | boundary consistency |
| ControlNet pose/depth/edge | control image + prompt | giữ layout/pose | control weight |

### Strength trong image-to-image

```text
strength thấp -> giữ ảnh gốc nhiều, thay đổi nhẹ
strength cao  -> sáng tạo nhiều, dễ mất identity/layout
```

Production cần lưu input image, mask, strength, seed và model version vì output phụ thuộc mạnh vào các tham số này.

<Callout type="warning" title="Image-to-image có rủi ro deepfake và brand misuse">
Khi user upload ảnh người thật, logo hoặc sản phẩm có bản quyền, hệ thống cần policy rõ: consent, face manipulation, trademark usage, retention và audit log.
</Callout>

## 6. Safety, Moderation và Asset Lifecycle

Image generation cần kiểm duyệt cả trước và sau generation.

### Pre-generation controls

- Prompt moderation.
- Blocklist cho prohibited content.
- Policy classifier cho face, minors, violence, sexual content, political persuasion.
- Quota/rate limit theo user/tenant.
- Brand/copyright rules.

### Post-generation controls

- NSFW/harmful image classifier.
- Face/deepfake checks nếu có người thật.
- Watermark/provenance metadata.
- Human review cho high-risk outputs.
- Store moderation scores và model metadata.

### Asset lifecycle

Không lưu URL tạm của provider làm source of truth.

```text
provider_url -> download -> virus/content scan -> upload to owned storage -> CDN URL -> database record
```

Database record nên có:

```json
{
  "job_id": "img_123",
  "user_id": "user_456",
  "status": "completed",
  "prompt_hash": "sha256:...",
  "provider": "replicate",
  "model": "sdxl",
  "seed": 42,
  "image_url": "https://cdn.example.com/img_123.png",
  "moderation_status": "passed",
  "cost_usd": 0.032,
  "created_at": "2026-07-01T00:00:00Z"
}
```

## 7. Implementation Playbook: FastAPI Async Job Skeleton

Skeleton dưới đây minh họa API tạo job, polling status và worker giả lập.

```python
from enum import Enum
from uuid import uuid4
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel, Field

app = FastAPI()


class JobStatus(str, Enum):
    queued = "queued"
    running = "running"
    completed = "completed"
    failed = "failed"
    rejected = "rejected"


class GenerateRequest(BaseModel):
    prompt: str = Field(min_length=3, max_length=2000)
    style: str = "photorealistic"
    aspect_ratio: str = "1:1"
    idempotency_key: str | None = None


jobs = {}


def moderate_prompt(prompt: str):
    blocked_terms = ["explicit_unsafe_term", "private_person_deepfake"]
    if any(term in prompt.lower() for term in blocked_terms):
        return False, "prompt_policy_violation"
    return True, None


@app.post("/images/generate", status_code=202)
def create_generation_job(req: GenerateRequest):
    allowed, reason = moderate_prompt(req.prompt)
    if not allowed:
        return {"status": JobStatus.rejected, "reason": reason}

    job_id = f"img_{uuid4().hex}"
    jobs[job_id] = {
        "status": JobStatus.queued,
        "prompt": req.prompt,
        "style": req.style,
        "aspect_ratio": req.aspect_ratio,
        "image_url": None,
        "error": None,
    }

    # In production: enqueue job_id to Redis/Celery/RQ/SQS.
    return {"job_id": job_id, "status": JobStatus.queued}


@app.get("/images/jobs/{job_id}")
def get_job(job_id: str):
    if job_id not in jobs:
        raise HTTPException(status_code=404, detail="job_not_found")
    return jobs[job_id]


def worker_process_job(job_id: str):
    job = jobs[job_id]
    job["status"] = JobStatus.running
    try:
        # 1. Rewrite prompt with style preset.
        # 2. Call provider/self-hosted model.
        # 3. Download output asset.
        # 4. Run output moderation.
        # 5. Upload to owned storage.
        job["image_url"] = "https://cdn.example.com/generated.png"
        job["status"] = JobStatus.completed
    except Exception as exc:
        job["status"] = JobStatus.failed
        job["error"] = str(exc)
```

Production checklist:

- Idempotency key để tránh duplicate jobs khi client retry.
- Queue riêng cho fast/high-quality jobs.
- Provider timeout và retry policy.
- Webhook signature verification nếu dùng provider callbacks.
- Download provider asset về storage của bạn.
- Log cost và moderation status cho từng job.

## 9. Post-Mortem & Troubleshooting (SRE/Gỡ lỗi)

<Callout type="warning" title="Pitfall 1: Backend timeout">
**Triệu chứng:** Requests fail after 30-60 seconds, users retry repeatedly.

**Nguyên nhân:** Generation chạy đồng bộ trong HTTP request path.

**Cách khắc phục:** Dùng async jobs, queue, 202 Accepted, polling/webhook và idempotency key.
</Callout>

<Callout type="warning" title="Pitfall 2: Provider rate limit">
**Triệu chứng:** Jobs fail hàng loạt khi traffic tăng hoặc provider trả lỗi quota.

**Nguyên nhân:** Không có concurrency limit, backoff hoặc provider routing.

**Cách khắc phục:** Thêm queue concurrency, exponential backoff, circuit breaker và fallback provider nếu phù hợp.
</Callout>

<Callout type="warning" title="Pitfall 3: Lưu URL tạm của provider">
**Triệu chứng:** Ảnh hiển thị tốt hôm nay nhưng 404 ngày mai.

**Nguyên nhân:** Provider URL hết hạn, không upload về storage của bạn.

**Cách khắc phục:** Download asset sau khi job completed, upload S3/R2/CDN và lưu URL owned storage.
</Callout>

<Callout type="warning" title="Pitfall 4: Cost spike vì re-roll">
**Triệu chứng:** User bấm generate nhiều lần, cost tăng nhanh nhưng conversion không tăng.

**Nguyên nhân:** Prompt UX kém, không có quota, không có preview/preset, không đo successful image rate.

**Cách khắc phục:** Style presets, prompt suggestions, quota, cost budget, log re-rolls per accepted image.
</Callout>

## 10. Cost & Product Unit Economics

Image generation cost nên tính theo successful output, không chỉ per API call.

| Cost driver | Ví dụ | Control |
|---|---|---|
| Provider call | per image / per GPU second | presets, routing, caching |
| Re-rolls | user tạo lại nhiều lần | quota, prompt UX, variations |
| Storage/CDN | lưu ảnh lâu dài | retention policy, compression |
| Moderation | input/output classifiers | cascade cheap-first |
| Human review | high-risk content | risk tiering |
| Failed jobs | timeout, NSFW, rate limit | pre-checks, retry policy |

Formula:

```text
cost_per_accepted_image = raw_generation_cost * average_generations_per_accepted_image
                         + moderation_cost
                         + storage_cost
                         + support/incident_cost
```

Production takeaway: tối ưu prompt UX và giảm re-roll rate có thể tiết kiệm nhiều hơn việc đổi provider rẻ hơn 10%.

## Answer key

1. Vì generation chậm và dễ timeout; async jobs giúp không giữ request thread, hỗ trợ queue, retry, polling/webhook và scale tốt hơn.
2. Nó mô tả các chi tiết model nên tránh như blur, watermark, extra fingers hoặc artifacts.
3. Vì URL đó có thể hết hạn hoặc bị provider xóa; cần download và lưu vào storage của hệ thống.
4. Nó điều khiển mức độ output giữ ảnh gốc hay biến đổi mạnh theo prompt.
5. Pre-generation moderation chặn prompt/input trước khi tốn tiền sinh ảnh; post-generation moderation kiểm tra output thực tế trước khi hiển thị/lưu hành.

## Điểm cần nhớ

- Text-to-image production là async workflow, không phải một API call đồng bộ.
- Provider selection phụ thuộc latency, cost, control, safety và licensing.
- Style controls và presets giúp giảm prompt ambiguity và re-roll cost.
- Image-to-image/inpainting tăng giá trị sản phẩm nhưng cũng tăng rủi ro consent, deepfake và brand misuse.
- Asset lifecycle phải download về storage của bạn và lưu metadata đầy đủ.
- Cost nên đo theo accepted image, bao gồm re-rolls, moderation, storage và failed jobs.
