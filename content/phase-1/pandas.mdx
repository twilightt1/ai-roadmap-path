Pandas là công cụ **xử lý dữ liệu dạng bảng** — được dùng để khám phá, làm sạch, và biến đổi dataset trước khi đưa vào model. Nếu NumPy là nền tảng tính toán, Pandas là lớp thao tác dữ liệu "con người" (đọc CSV, lọc, group, pivot). Bài này covers DataFrame cơ bản, indexing, xử lý missing, và group/aggregate.

## Series và DataFrame

```python
import pandas as pd

# Series — 1D (như cột)
s = pd.Series([1, 2, 3, 4], name="score")

# DataFrame — 2D (như bảng)
df = pd.DataFrame({
    "model": ["gpt-4", "claude", "gemini", "llama"],
    "score": [0.92, 0.88, 0.95, 0.80],
    "cost":  [0.03, 0.015, 0.01, 0.001],
})
```

## Khám phá dữ liệu

```python
df.head()          # 5 dòng đầu
df.head(2)         # 2 dòng đầu
df.info()          # kiểu mỗi cột + memory
df.describe()      # thống kê (count, mean, std, min, max)
df.shape           # (4, 3)
df.columns         # Index(['model','score','cost'])
df.dtypes          # kiểu mỗi cột
```

## loc vs iloc — 2 cách index

```python
# Đặt index để demo
df_idx = df.set_index("model")

# loc — label-based (dùng giá trị index/tên cột)
df_idx.loc["claude"]              # hàng có index="claude"
df_idx.loc[["gpt-4", "gemini"]]   # nhiều hàng
df_idx.loc["claude", "score"]     # 1 ô

# iloc — positional (dùng số nguyên)
df.iloc[0]          # hàng đầu (bất kể index)
df.iloc[1:3]        # hàng 1,2
df.iloc[:, 1]       # tất cả hàng, cột thứ 1
```

<Callout type="warning" title="loc vs iloc dễ nhầm">
`loc` dùng **label** (bao gồm cả slice **bằng tên**, inclusive); `iloc` dùng **số nguyên** (slice Python thường, exclusive cuối). `df.loc[0:2]` lấy 3 hàng nếu label 0,1,2; `df.iloc[0:2]` lấy 2 hàng đầu. Nhầm rất phổ biến.
</Callout>

## Lọc (boolean filtering)

```python
# Lọc theo điều kiện
good = df[df["score"] > 0.85]
cheap_good = df[(df["score"] > 0.85) & (df["cost"] < 0.02)]

# isin — lọc theo danh sách
top = df[df["model"].isin(["gpt-4", "claude"])]

# query — cú pháp gần SQL
df.query("score > 0.85 and cost < 0.02")
```

## Missing values

```python
df = pd.DataFrame({
    "a": [1, 2, None, 4],
    "b": [10, None, 30, 40],
})

df.isna()              # mask True tại NaN
df.isna().sum()        # đếm missing mỗi cột
df.dropna()            # bỏ hàng có bất kỳ NaN
df.dropna(subset=["a"])  # chỉ bỏ nếu 'a' NaN
df.fillna(0)           # thay NaN bằng 0
df["a"].fillna(df["a"].mean())   # thay bằng mean cột
```

<Callout type="tip" title="Chiến lược missing data">
Không phải lúc nào cũng drop. Với số liệu liên tục, điền `mean`/`median` hoặc dùng interpolation. Với phân loại, điền `"unknown"`. Trong AI, missing data ảnh hưởng chất lượng model — cần xử lý có chủ đích.
</Callout>

## groupby và aggregate

```python
logs = pd.DataFrame({
    "user": ["a","a","b","b","b","c"],
    "tokens": [100, 200, 50, 150, 300, 80],
    "model": ["gpt-4","claude","gpt-4","gpt-4","claude","gpt-4"],
})

# Tổng tokens mỗi user
logs.groupby("user")["tokens"].sum()
# a: 300, b: 500, c: 80

# Nhiều aggregate
logs.groupby("user")["tokens"].agg(["count", "sum", "mean"])

# group nhiều cột
logs.groupby(["user", "model"])["tokens"].sum()
```

## pivot_table và value_counts

```python
# pivot_table — bảng chéo
logs.pivot_table(values="tokens", index="user", columns="model",
                 aggfunc="sum", fill_value=0)

# value_counts — tần suất
logs["model"].value_counts()
# gpt-4: 4, claude: 2
```

## Merge, concat, melt, pivot

```python
users = pd.DataFrame({"user": ["a","b","c"], "plan": ["pro","free","pro"]})
usage = pd.DataFrame({"user": ["a","b","d"], "tokens": [300,500,90]})

# merge — join SQL
pd.merge(users, usage, on="user", how="inner")   # chỉ user có ở cả 2
pd.merge(users, usage, on="user", how="left")    # giữ tất cả user bên trái
pd.merge(users, usage, on="user", how="outer")   # giữ tất cả

# concat — nối dòng
pd.concat([df1, df2], ignore_index=True)

# melt — wide -> long (tốt cho vẽ biểu đồ)
df_wide.melt(id_vars=["model"], var_name="metric", value_name="val")

# pivot — long -> wide
df_long.pivot(index="model", columns="metric", values="val")
```

<Callout type="info" title="how= trong merge = JOIN SQL">
`how="inner"` = INNER JOIN, `left` = LEFT JOIN, `outer` = FULL OUTER JOIN. Chọn `how` tùy: khi gắn metadata user vào log, thường `left` để không mất log của user chưa có profile.
</Callout>

## Parse dates

```python
logs = pd.DataFrame({
    "ts": ["2026-01-01", "2026-01-02", "2026-01-03"],
    "tokens": [100, 200, 150],
})
logs["ts"] = pd.to_datetime(logs["ts"])
logs = logs.set_index("ts")

# Giờ có thể resample theo thời gian
logs.resample("D").sum()         # tổng theo ngày
logs.resample("W").mean()        # trung bình theo tuần
```

## Điểm cần nhớ

- `df.head/info/describe/shape` khám phá nhanh; `dtypes` kiểm tra kiểu.
- `loc` = label (slice inclusive), `iloc` = số nguyên (slice exclusive) — **đừng nhầm**.
- Lọc: `df[df["col"] > x]`, kết hợp `&`/`|` (mỗi điều kiện trong ngoặc), hoặc `df.query(...)`.
- Missing: `isna().sum()` đếm, `dropna`/`fillna` xử lý — chọn chiến lược phù hợp (không phải lúc nào cũng drop).
- `groupby + agg` là bread-and-butter cho phân tích theo nhóm.
- `merge(how=...)` tương đương JOIN SQL; `melt`/`pivot` đổi dạng wide/long.
- `pd.to_datetime` + `set_index` + `resample` cho phân tích chuỗi thời gian.
