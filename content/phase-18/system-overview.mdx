Capstone là nơi toàn bộ roadmap hội tụ. Bạn không còn xây một chatbot demo gọi thẳng OpenAI API, mà xây một **nền tảng AI production đa tenant** có thể phục vụ nhiều sản phẩm: PrinceSinghAI, PrinceSinghDev, AskAI, RoadmapAI, CodeLLM và các workflow AI tương lai.

Mục tiêu của Capstone là chứng minh bạn có thể đi từ kỹ năng rời rạc đến một hệ thống hoàn chỉnh: gateway, routing, fallback, RAG, agents, MCP tools, observability, cost tracking, safety gateway, privacy controls, red-team regression, audit logs và deployment pipeline.

## 1. The Incident (Tình huống sự cố)

<Callout type="danger" title="Báo cáo sự cố: Chatbot demo viral và biến thành platform outage">
**Bối cảnh:** Một developer xây chatbot nội bộ rất nhanh: React frontend gọi FastAPI, FastAPI gọi OpenAI, conversation lưu trong memory tạm. Demo thành công, team quyết định dùng luôn cho nhiều sản phẩm: support bot, code assistant, RAG docs, agent research.

**Sự cố:** Khi traffic tăng, hệ thống bắt đầu vỡ theo nhiều hướng cùng lúc:

1. Không có tenant isolation, user A vô tình thấy document của tenant B trong RAG.
2. Không có router/fallback, khi OpenAI timeout toàn bộ sản phẩm ngừng hoạt động.
3. Không có cost tracking, một tenant free-tier đốt 2,000 USD trong một ngày.
4. Không có prompt/version registry, team không biết release nào làm hallucination tăng.
5. Không có safety gateway, fallback model bypass output moderation.
6. Không có trace/audit, incident không thể debug từ request đến model, prompt, chunks và tool calls.

**Hậu quả:** Một chatbot demo trở thành liability production. Team phải dừng rollout và thiết kế lại từ đầu như một AI platform thật sự.
</Callout>

Bài học rút ra: Production AI platform không phải “chat endpoint + model API”. Nó là control plane quản lý identity, tenants, models, prompts, tools, data, cost, safety và observability.

## 2. Architecture Blueprint (Capstone Reference Architecture)

Capstone platform có thể hình dung như sau:

```text
[ Product Surfaces ]
  - PrinceSinghAI
  - PrinceSinghDev
  - AskAI
  - RoadmapAI
  - CodeLLM
          │
          ▼
[ API Gateway ]
  - auth / tenant context
  - rate limits / quotas
  - request validation
  - streaming proxy
  - idempotency keys
          │
          ▼
[ Safety & Privacy Gateway ]
  - PII masking
  - prompt injection checks
  - high-risk domain routing
  - input moderation
          │
          ▼
[ AI Control Plane ]
  ├── Prompt Registry
  ├── Model Registry
  ├── Multi-LLM Router
  ├── Fallback Manager
  ├── Cost Tracker
  ├── Cache Manager
  ├── Experiment Manager
  └── Tenant Policy Engine
          │
          ├─────────────┬──────────────┬──────────────┐
          ▼             ▼              ▼              ▼
[ LLM Providers ]   [ RAG Engine ]  [ Agent Graph ] [ MCP Tools ]
 OpenAI/Claude       pgvector/      LangGraph        DB/API/Web/File
 Gemini/Mistral      Qdrant/Chroma  state machine    tool servers
          │             │              │              │
          └─────────────┴──────────────┴──────────────┘
                          ▼
[ Output Safety / Validation / Audit ]
  - output moderation
  - schema validation
  - citation/evidence checks
  - tool action review
  - audit events
                          │
                          ▼
[ Observability & Admin ]
  - Langfuse traces
  - Prometheus metrics
  - Grafana dashboards
  - cost analytics
  - safety incidents
  - admin controls
```

Kiến trúc này tách rõ **data plane** (request/response/model/tool execution) và **control plane** (policies, routing, cost, safety, observability). Đây là khác biệt giữa demo và platform.

## 5. System Scope

### In scope cho Capstone

- Multi-tenant user management.
- API Gateway cho chat, RAG, agents và admin.
- Streaming responses qua SSE.
- Multi-LLM provider abstraction.
- Intelligent model routing và fallback.
- RAG ingestion/retrieval đa document types.
- Conversation memory bằng Redis.
- Prompt template/version registry.
- Agent orchestration bằng LangGraph-style state machine.
- MCP tool connectivity.
- Cost tracking per tenant/user/feature/model.
- Usage limits và budget controls.
- Safety gateway: PII, prompt injection, moderation, output validation.
- Red-team regression suite.
- Audit logging và incident response artifacts.
- Admin dashboard với observability.
- Docker Compose local và Kubernetes deploy path.

### Out of scope cho MVP

- Training foundation model from scratch.
- Full payment processor integration.
- Complex marketplace/plugin ecosystem.
- Large-scale GPU self-hosting nếu API providers đủ cho MVP.
- Fully automated high-stakes decisions không có human review.

<Callout type="tip" title="Capstone MVP nên nhỏ nhưng đúng boundary">
Đừng cố build mọi tính năng hoàn hảo. Hãy build một vertical slice production-grade: tenant → chat/RAG → routing → safety → cost → trace → admin dashboard → deploy.
</Callout>

## 6. Non-Functional Requirements

Capstone được đánh giá mạnh ở non-functional requirements, vì đó là điểm demo thường thiếu.

| Category | Requirement | Example acceptance criteria |
|---|---|---|
| Reliability | fallback, retries, circuit breakers | Provider outage không làm toàn bộ app sập |
| Latency | streaming, async, cache | TTFT p95 dưới target cho chat thường |
| Cost | routing, budgets, usage tracking | Mỗi request có cost estimate và actual usage log |
| Security | auth, tenant isolation, key protection | User không thể đọc document tenant khác |
| Privacy | PII masking, retention, erasure path | PII không vào provider/log nếu policy cấm |
| Governance | audit logs, policy versions, incident workflow | Bad response truy vết được model/prompt/chunks/tool calls |
| Observability | traces, metrics, dashboards | Có Langfuse trace và Grafana metrics cho request |
| Maintainability | modular services, ADRs, typed contracts | Thêm provider mới không sửa toàn bộ app |
| Safety | red-team regression, moderation | Critical jailbreak tests fail build nếu regression |

## 10. Post-Mortem & Troubleshooting (SRE/Gỡ lỗi)

<Callout type="danger" title="Pitfall 1: Xây chatbot rồi gọi là platform">
**Triệu chứng:** Có chat UI đẹp nhưng không có tenants, router, RAG, cost tracking, audit logs hoặc safety controls.

**Nguyên nhân:** Team tối ưu demo flow thay vì platform capabilities.

**Cách khắc phục:** Build vertical slice platform: auth → gateway → routing → safety → trace → admin visibility trước khi mở rộng UI.
</Callout>

<Callout type="warning" title="Pitfall 2: Không có tenant context xuyên suốt">
**Triệu chứng:** Một số service quên filter `tenant_id`, dẫn đến data leak hoặc cost attribution sai.

**Nguyên nhân:** Tenant context không phải dependency bắt buộc ở service layer.

**Cách khắc phục:** Tenant context phải được resolve ở gateway và truyền typed qua mọi service. DB queries, cache keys, vector filters và audit logs đều phải include tenant.
</Callout>

<Callout type="warning" title="Pitfall 3: Observability thêm sau cùng">
**Triệu chứng:** Khi có bad response, không biết model nào, prompt nào, chunks nào hoặc safety scores nào liên quan.

**Nguyên nhân:** Tracing/logging không được thiết kế từ đầu.

**Cách khắc phục:** Mọi request cần trace ID từ ngày đầu. Log model call, prompt version, cost, latency, retrieved sources, tool calls và safety decisions.
</Callout>

<Callout type="danger" title="Pitfall 4: Safety là middleware cuối cùng">
**Triệu chứng:** Primary path có guardrails, nhưng agent tools hoặc fallback provider bypass safety.

**Nguyên nhân:** Safety không nằm ở gateway/control plane chung.

**Cách khắc phục:** Safety gateway độc lập trước/sau mọi provider, RAG, agent và fallback. Tool authorization nằm server-side.
</Callout>

## 11. Platform Unit Economics

| Đòn bẩy | Tác động | Metric cần theo dõi |
|---|---|---|
| Model routing | Giảm cost, tăng reliability | cost/request, fallback rate, quality score |
| Streaming | Tăng UX perceived speed | TTFT, stream interruption rate |
| Cache | Giảm repeated calls | cache hit rate, stale response rate |
| RAG citations | Tăng trust, giảm hallucination | faithfulness, citation accuracy |
| Safety gateway | Giảm incident risk | block rate, false refusal rate, critical failures |
| Admin dashboard | Giảm debug time | MTTR, trace completeness |
| CI red-team tests | Giảm regression | safety pass rate per release |

Capstone thành công nếu bạn chứng minh được: platform không chỉ trả lời được, mà còn **đo được, kiểm soát được và vận hành được**.
