Agentic AI Production không phải tối đa hóa autonomy. Nó là **bounded autonomy**: cho AI đủ khả năng tiến triển task, nhưng giới hạn actions, cost, permissions, time, data access và failure modes. Đây là chương capstone của Phase 9 — nơi tất cả agent concepts gặp production constraints.

---

## 1. The Incident (Tình huống sự cố)

<Callout type="danger" title="Báo cáo sự cố: Agent research tốn 2,800 USD trong 4 giờ, không có kill switch">
**Bối cảnh:** Một team build "research agent" production. Agent có planner, search tool, RAG, source extractor, synthesizer. Mục tiêu: tạo báo cáo thị trường. Architecture ban đầu rất đơn giản: agent chạy đến khi "xong", không có budget, không có kill switch.

**Sự cố:** Một user thử câu hỏi mơ hồ: "Tìm hiểu mọi thứ về thị trường X". Agent chạy, không có done condition rõ. Nó tìm source, đọc, tìm tiếp, đọc tiếp, đổi query, tìm lại. 4 giờ sau, agent vẫn chạy. 2.8 triệu token, 8,000 lần search tool, 2,800 USD.

1. Không có max steps, max tokens, max cost, max wall-clock.
2. Không có kill switch — không có cách dừng ngay ongoing run.
3. Không có budget per tenant/agent.
4. Không có alert on abnormal spend.
5. Agent trông competent ngay cả khi off-track.

**Hậu quả:** Hóa đơn API tăng 2,800 USD trong 4 giờ, agent phải tắt khẩn cấp, và team phải thiết kế lại toàn bộ budget, kill switch, và audit trail.
</Callout>

Bài học rút ra: production question không phải "agent có làm được không?" — câu hỏi đúng là "Agent có làm task an toàn, observable, lặp lại được, trong budget, và recover khi sai không?" Agent production fail hard hơn chatbot vì chúng thực hiện hành động lặp lại, dùng tools có side effects, persist state, truy cập sensitive data, chạy lâu, và tích lũy cost.

Production agent là hệ thống có guardrails, không phải hệ thống tự do. Runtime quyết định: max steps, max cost, max time, kill switch, audit trail, approval gates, và termination.

## 3. Bounded Autonomy

```text
Give agent enough ability to make progress.
Constrain actions, cost, permissions, time, data, and failure modes.
```

Bounded autonomy dimensions:

| Dimension | Control |
|---|---|
| Time | max runtime |
| Steps | max actions |
| Cost | token/tool budget |
| Tools | allowlist |
| Data | ACL filters |
| Side effects | approval gates |
| Scope | task boundaries |
| Confidence | escalation threshold |
| Memory | write policy |
| Output | validation |

Mỗi dimension là một control. Production agent không phải là agent có thể làm mọi thứ — đó là agent có thể làm một số thứ, trong một số giới hạn, với một số guardrails.

## 4. Task Decomposition và Progress Tracking

Production agent chuyển goal thành subtasks có structure:

```json
{
  "goal": "Prepare a competitor research brief",
  "subtasks": [
    {
      "id": "s1",
      "title": "Identify competitors",
      "status": "pending",
      "acceptance_criteria": "At least 5 relevant competitors with sources",
      "depends_on": []
    },
    {
      "id": "s2",
      "title": "Collect pricing information",
      "status": "pending",
      "depends_on": ["s1"]
    }
  ],
  "done_condition": "Report includes citations, confidence scores, and limitations"
}
```

Bad:

```text
Keep working until done.
```

Good:

```text
Stop after producing a report with exactly 5 cited sources, confidence scores, and known limitations.
```

Progress trace cho bạn biết agent đang ở đâu, còn budget bao nhiêu, có stuck không, và khi nào nên escalate.

```json
{
  "run_id": "agent_run_123",
  "step": 7,
  "subtask": "collect pricing information",
  "action": "web_search",
  "observation_summary": "Found pricing page for Vendor A",
  "confidence": 0.78,
  "budget_remaining_usd": 1.42,
  "next_action": "extract pricing table"
}
```

## 5. Error Recovery Policies

| Error | Recovery |
|---|---|
| Tool timeout | retry with backoff if idempotent |
| Invalid tool args | ask model repair once |
| Unauthorized action | block and escalate |
| Low confidence | gather more evidence |
| Conflicting sources | report conflict or escalate |
| Budget near limit | summarize partial result |
| Repeated failure | stop and ask human |
| Sandbox failure | return diagnostic |
| Prompt injection detected | ignore untrusted instruction |
| Missing data | ask user or abstain |

<Callout type="warning" title="Recovery phải là policy">
Đừng để model tự invent unlimited recovery behavior. Mỗi recovery path nên được code rõ với budget riêng. Nếu không, model có thể invent recovery path tốn tiền, không an toàn, hoặc loop.
</Callout>

## 6. Human Escalation

Escalate khi:

- high-risk action,
- unclear user intent,
- policy conflict,
- low confidence,
- conflicting evidence,
- repeated tool failure,
- budget exceeded,
- legal/medical/financial risk,
- sensitive data involved,
- user explicitly requests approval.

Escalation payload:

```json
{
  "reason": "Low confidence and conflicting sources",
  "summary": "The agent found two contradictory policy documents.",
  "evidence": ["doc_a#v3", "doc_b#v1"],
  "recommended_action": "Ask policy owner to confirm current version",
  "options": ["continue", "stop", "ask_user", "escalate_to_owner"]
}
```

Human escalation không phải là "model hỏi user". Nó là runtime gate: agent đề xuất, human approve, runtime execute.

## 8. Safety Controls

### Action confirmation

Require approval cho:

- sending email,
- posting messages,
- deleting files,
- updating records,
- making purchases,
- issuing refunds,
- running code with network,
- changing permissions,
- deploying software.

### Scope limitation

```text
Allowed: summarize invoices for tenant A.
Not allowed: access invoices for tenant B.

Allowed: draft refund email.
Not allowed: issue refund without approval.

Allowed: run tests in sandbox.
Not allowed: access host filesystem.
```

### Cost controls

- max tokens per run,
- max model calls,
- max tool calls,
- max wall-clock time,
- max parallel branches,
- per-tenant budgets,
- per-agent budgets,
- alert on abnormal spend,
- kill switch.

<Callout type="danger" title="Kill switch là feature production">
Agent cần có kill switch dừng ngay ongoing run khi phát hiện anomaly, cost spike hoặc security concern. Kill switch không optional — nó là feature production.
</Callout>

## Implementation Playbook: Production Research Agent Skeleton

```python
from dataclasses import dataclass, field


@dataclass
class ResearchState:
    question: str
    subtasks: list[dict] = field(default_factory=list)
    evidence: list[dict] = field(default_factory=list)
    draft: str | None = None
    confidence: float = 0.0
    budget_remaining_usd: float = 2.0
    max_steps: int = 12
    step: int = 0
    requires_human: bool = False
    status: str = "running"


def should_escalate(state: ResearchState) -> bool:
    if state.confidence < 0.6 and state.step >= state.max_steps - 2:
        return True
    if state.budget_remaining_usd < 0.2:
        return True
    return False
```

Production checklist:

1. Có max steps/time/cost không?
2. Có kill switch không?
3. Có audit trail đầy đủ không?
4. Có human escalation policy không?
5. Có budget per tenant/agent không?
6. Có eval regression suite không?
7. Có red-team tests không?
8. Có trace đầy đủ cho mỗi run không?
9. Agent/prompt/model có version không?
10. Có monitor cost/escalation/failure không?

## Post-Mortem & Troubleshooting (SRE/Gỡ lỗi)

<Callout type="warning" title="Pitfall 1: Không có kill switch">
**Triệu chứng:** Agent chạy dài không có cách dừng ngay, cost spike, không có cách dừng.

**Fix:** Agent cần kill switch dừng ngay ongoing run khi phát hiện anomaly, cost spike hoặc security concern. Kill switch là feature production.
</Callout>

<Callout type="warning" title="Pitfall 2: Không version agent/prompt">
**Triệu chứng:** Agent behavior regression, không biết agent version, prompt version, model nào tạo ra.

**Fix:** Mỗi run có agent_version, prompt_version, model. Khi agent fail, bạn có thể reproduce failure, so sánh versions, và biết regression.
</Callout>

<Callout type="danger" title="Pitfall 3: Agent trông competent khi off-track">
**Triệu chứng:** Agent output tự tin nhưng sai, không có verifier độc lập, không ai phát hiện.

**Fix:** Agent có thể output tự tin nhưng sai. Cần verifier độc lập, không chỉ tin final answer. Đừng để model tự chấm điểm mình.
</Callout>

## Cost / Latency / Reliability Unit Economics

| Quyết định | Lợi ích | Chi phí/rủi ro |
|---|---|---|
| Bounded autonomy | an toàn, observable | hạn chế adaptive |
| Kill switch | dừng ngay anomaly | implementation overhead |
| Human escalation | safety | latency, SLA |
| Audit trail | reproduce, audit | storage, log cost |
| Budget per role | cost control | over-budget stops |
| Red-team tests | find weaknesses | cost, time |
| Verifier độc lập | catch off-track | extra LLM calls |

Production agent = bounded autonomy, không maximum autonomy. Mỗi control thêm cost, nhưng cũng thêm safety. Đừng cắt control để giảm cost — bạn sẽ mất safety.

## Answer key

1. Cho agent đủ khả năng tiến triển task nhưng giới hạn actions/cost/permissions/time/data/failure.
2. Subtasks, dependencies, acceptance criteria, risk, cost estimate, done condition.
3. Policy-driven, code rõ, có budget riêng; không để model invent unlimited recovery.
4. High-risk action, unclear intent, policy conflict, low confidence, conflicting evidence, repeated failure, budget exceeded, sensitive data.
5. User/tenant, agent/prompt/model version, tool calls, args, results, approvals, denials, errors, cost, latency, output, evidence, memory writes.
6. Max tokens/calls/time/parallel, per-tenant/agent budgets, alerts, kill switch.
7. Unit, scenario, regression, human, red-team.

## Điểm cần nhớ

- Production agent = bounded autonomy, không maximum autonomy.
- Task decomposition cần acceptance criteria và done condition.
- Error recovery là policy, không để model tự invent.
- Audit trails và traces là bắt buộc cho agent production.
- Cost/step/time controls và kill switch là feature, không optional.
- Agent evaluation cần unit/scenario/regression/human/red-team levels.
