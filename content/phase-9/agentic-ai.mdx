Agentic AI Production không phải tối đa hóa autonomy. Nó là **bounded autonomy**: cho AI đủ khả năng tiến triển task, nhưng giới hạn actions, cost, permissions, time, data access và failure modes. Đây là chương capstone của Phase 9 — nơi tất cả agent concepts gặp production constraints.

## Why this matters

Agent production fail hard hơn chatbot vì chúng:

- thực hiện hành động lặp lại,
- dùng tools có side effects,
- persist state,
- truy cập sensitive data,
- chạy lâu,
- phối hợp nhiều agents,
- tích lũy cost,
- trông competent ngay cả khi off-track.

<Callout type="danger" title="Production question không phải 'agent có làm được không?'">
Câu hỏi đúng là: 'Agent có làm task an toàn, observable, lặp lại được, trong budget, và recover khi sai không?'
</Callout>

## Core intuition: bounded autonomy

```text
Give agent enough ability to make progress.
Constrain actions, cost, permissions, time, data, and failure modes.
```

Production agent là hệ thống có guardrails, không phải hệ thống tự do.

## Bounded autonomy dimensions

| Dimension | Control |
|---|---|
| Time | max runtime |
| Steps | max actions |
| Cost | token/tool budget |
| Tools | allowlist |
| Data | ACL filters |
| Side effects | approval gates |
| Scope | task boundaries |
| Confidence | escalation threshold |
| Memory | write policy |
| Output | validation |

## Task decomposition

Production agent chuyển goal thành subtasks có structure:

```json
{
  "goal": "Prepare a competitor research brief",
  "subtasks": [
    {
      "id": "s1",
      "title": "Identify competitors",
      "status": "pending",
      "acceptance_criteria": "At least 5 relevant competitors with sources",
      "depends_on": []
    },
    {
      "id": "s2",
      "title": "Collect pricing information",
      "status": "pending",
      "depends_on": ["s1"]
    }
  ],
  "done_condition": "Report includes citations, confidence scores, and limitations"
}
```

Bad:

```text
Keep working until done.
```

Good:

```text
Stop after producing a report with exactly 5 cited sources, confidence scores, and known limitations.
```

## Progress tracking

Track mỗi step:

```json
{
  "run_id": "agent_run_123",
  "step": 7,
  "subtask": "collect pricing information",
  "action": "web_search",
  "observation_summary": "Found pricing page for Vendor A",
  "confidence": 0.78,
  "budget_remaining_usd": 1.42,
  "next_action": "extract pricing table"
}
```

Progress trace cho bạn biết:

- agent đang ở đâu,
- còn budget bao nhiêu,
- có stuck không,
- khi nào nên escalate.

## Error recovery policies

| Error | Recovery |
|---|---|
| Tool timeout | retry with backoff if idempotent |
| Invalid tool args | ask model repair once |
| Unauthorized action | block and escalate |
| Low confidence | gather more evidence |
| Conflicting sources | report conflict or escalate |
| Budget near limit | summarize partial result |
| Repeated failure | stop and ask human |
| Sandbox failure | return diagnostic |
| Prompt injection detected | ignore untrusted instruction |
| Missing data | ask user or abstain |

<Callout type="warning" title="Recovery phải là policy">
Đừng để model tự invent unlimited recovery behavior. Mỗi recovery path nên được code явно với budget riêng.
</Callout>

## Human escalation

Escalate khi:

- high-risk action,
- unclear user intent,
- policy conflict,
- low confidence,
- conflicting evidence,
- repeated tool failure,
- budget exceeded,
- legal/medical/financial risk,
- sensitive data involved,
- user explicitly requests approval.

Escalation payload:

```json
{
  "reason": "Low confidence and conflicting sources",
  "summary": "The agent found two contradictory policy documents.",
  "evidence": ["doc_a#v3", "doc_b#v1"],
  "recommended_action": "Ask policy owner to confirm current version",
  "options": ["continue", "stop", "ask_user", "escalate_to_owner"]
}
```

## Safety controls

### Action confirmation

Require approval cho:

- sending email,
- posting messages,
- deleting files,
- updating records,
- making purchases,
- issuing refunds,
- running code with network,
- changing permissions,
- deploying software.

### Scope limitation

```text
Allowed: summarize invoices for tenant A.
Not allowed: access invoices for tenant B.

Allowed: draft refund email.
Not allowed: issue refund without approval.

Allowed: run tests in sandbox.
Not allowed: access host filesystem.
```

### Cost controls

- max tokens per run,
- max model calls,
- max tool calls,
- max wall-clock time,
- max parallel branches,
- per-tenant budgets,
- per-agent budgets,
- alert on abnormal spend,
- kill switch.

<Callout type="danger" title="Kill switch là feature production">
Agent cần có kill switch dừng ngay ongoing run khi phát hiện anomaly, cost spike hoặc security concern.
</Callout>

## Implementation lab: production research agent skeleton

Build agent với:

- planner,
- search tool,
- RAG/KB tool,
- source extractor,
- synthesizer,
- verifier,
- human escalation mock,
- budget controls,
- audit trace.

Required features:

- task decomposition,
- parallel research branches,
- citation tracking,
- confidence scoring,
- no-answer behavior,
- max cost/step controls,
- source quality scoring,
- final report with limitations.

```python
from dataclasses import dataclass, field


@dataclass
class ResearchState:
    question: str
    subtasks: list[dict] = field(default_factory=list)
    evidence: list[dict] = field(default_factory=list)
    draft: str | None = None
    confidence: float = 0.0
    budget_remaining_usd: float = 2.0
    max_steps: int = 12
    step: int = 0
    requires_human: bool = False
    status: str = "running"


def should_escalate(state: ResearchState) -> bool:
    if state.confidence < 0.6 and state.step >= state.max_steps - 2:
        return True
    if state.budget_remaining_usd < 0.2:
        return True
    return False
```

Deliverables:

## Common mistakes & debugging checklist

<Callout type="warning" title="Mistake 1: không có kill switch">
Agent chạy dài không có cách dừng ngay sẽ tạo cost spike và nguy cơ security.
</Callout>

<Callout type="warning" title="Mistake 2: không version agent/prompt">
Khi agent behavior regression, bạn cần biết agent version, prompt version và model nào tạo ra.
</Callout>

<Callout type="danger" title="Mistake 3: agent trông competent khi off-track">
Agent có thể output tự tin nhưng sai. Cần verifier独立, không chỉ tin final answer.
</Callout>

Checklist:

1. Có max steps/time/cost không?
2. Có kill switch không?
3. Có audit trail đầy đủ không?
4. Có human escalation policy không?
5. Có budget per tenant/agent không?
6. Có eval regression suite không?
7. Có red-team tests không?
8. Có trace đầy đủ cho mỗi run không?
9. Agent/prompt/model có version không?
10. Có monitor cost/escalation/failure không?

## Answer key

1. Cho agent đủ khả năng tiến triển task nhưng giới hạn actions/cost/permissions/time/data/failure.
2. Subtasks, dependencies, acceptance criteria, risk, cost estimate, done condition.
3. Policy-driven, code rõ, có budget riêng; không để model invent unlimited recovery.
4. High-risk action, unclear intent, policy conflict, low confidence, conflicting evidence, repeated failure, budget exceeded, sensitive data.
5. User/tenant, agent/prompt/model version, tool calls, args, results, approvals, denials, errors, cost, latency, output, evidence, memory writes.
6. Max tokens/calls/time/parallel, per-tenant/agent budgets, alerts, kill switch.
7. Unit, scenario, regression, human, red-team.

## Điểm cần nhớ

- Production agent = bounded autonomy, không maximum autonomy.
- Task decomposition cần acceptance criteria và done condition.
- Error recovery là policy, không để model tự Invent.
- Audit trails và traces là bắt buộc cho agent production.
- Cost/step/time controls và kill switch là feature, không optional.
- Agent evaluation cần unit/scenario/regression/human/red-team levels.
