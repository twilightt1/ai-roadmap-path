AI Agent không đơn giản là “LLM + tools”. Một agent production là **stateful control loop**: nó nhận goal, quan sát state, chọn action, gọi tools, nhận observation, cập nhật state/memory, rồi quyết định tiếp tục, trả lời, hỏi user, hoặc escalate. Điểm nguy hiểm của agent không phải vì nó “suy nghĩ”; nguy hiểm là nó có thể **hành động lặp lại** dưới uncertainty.

## Why this matters

Agents hấp dẫn vì chúng có thể:

- tự chia task nhiều bước,
- dùng tools ngoài LLM,
- recover từ lỗi nhỏ,
- search web/KB/database,
- viết code, chạy test, sửa lỗi,
- phối hợp nhiều bước mà user không cần chỉ từng bước.

Nhưng agents nguy hiểm vì:

- tool calls tốn tiền,
- loop vô hạn,
- hallucinated plan,
- gọi sai tool,
- argument sai gây side effects,
- prompt injection qua tool result,
- memory pollution,
- khó reproduce behavior.

<Callout type="danger" title="Autonomy without budgets is an outage waiting to happen">
Agent loop phải có giới hạn step, time, token, tool và cost. Nếu stop condition duy nhất là “model tự quyết nó đã xong”, hệ thống chưa production-safe.
</Callout>

## Core intuition: agent as a control loop

```text
Goal / User Task
   ↓
State
   ↓
LLM/controller decides next action
   ↓
Validate + authorize action
   ↓
Tool call / response / ask user / final answer
   ↓
Observation
   ↓
Update state + memory
   ↓
Continue or terminate
```

LLM không phải toàn bộ agent. Runtime quyết định:

- tools nào available,
- state nào visible,
- action nào được phép,
- max steps/cost/time,
- khi nào cần human approval,
- log những gì,
- điều kiện success là gì.

## Formal model: state, action, policy, observation

Một cách nhìn có tính hệ thống:

```text
s_t = current state at step t
a_t = action chosen at step t
o_t = observation after action
π(a_t | s_t) = policy choosing action given state
```

Trong agent LLM:

- `π` thường là LLM + prompt + tool schema + runtime policy.
- `a_t` có thể là `call_tool`, `ask_user`, `write_memory`, `final_answer`, `escalate`.
- `o_t` là tool result, user response, error, observation.
- `s_t` gồm goal, plan, history, memory, budget, errors.

Professor-level point:

```text
Agent failure = bad state, bad action space, bad policy, bad observation, or bad termination.
```

## Agent vs prompt vs chain vs workflow

| System | Control flow | Decision maker | Best for | Risk |
|---|---|---|---|---|
| Prompt | single call | developer | simple generation | no multi-step control |
| Chain | fixed sequence | developer | predictable tasks | brittle to ambiguity |
| Workflow | conditional deterministic | code/rules | business automation | rule complexity |
| Tool-using chatbot | model calls tools in one turn | model within narrow scope | assisted tasks | tool misuse |
| Agent | dynamic multi-step loop | model + runtime policy | uncertain multi-step tasks | cost/safety/loop risk |

Key idea:

```text
Chains encode the plan in code.
Agents infer part of the plan at runtime.
```

## Least autonomy principle

Dùng hệ thống ít autonomous nhất mà giải quyết được bài toán.

```text
prompt < chain < workflow < agent < multi-agent autonomous system
```

Nếu task có steps cố định, dùng workflow. Nếu task mơ hồ, cần tool selection động, có partial failures và cần planning, agent mới đáng cân nhắc.

<Callout type="warning" title="Đừng dùng agent khi workflow đủ">
Nếu bạn biết chính xác 5 bước cần chạy, viết workflow deterministic sẽ dễ test, rẻ, nhanh và an toàn hơn agent tự quyết.
</Callout>

## Types of agents

### ReAct agent

ReAct dùng vòng lặp:

```text
Reason → Act → Observe → Reason → Act → Observe → Final
```

Strength:

- đơn giản,
- dynamic tool use,
- dễ học.

Weakness:

- myopic,
- dễ loop,
- planning dài hạn kém,
- phụ thuộc tool descriptions.

### Plan-and-Execute

Planner tạo plan trước, executor chạy từng bước.

```text
Planner → steps → Executor → observations → maybe Replanner
```

Tốt cho task dài nhưng plan có thể stale khi observations thay đổi.

### MRKL / modular routing agent

Agent route subproblems tới modules chuyên biệt:

- calculator,
- search,
- database,
- code interpreter,
- retriever.

Tốt vì tools deterministic hơn model cho nhiều subtask.

### Reflection / critique agent

Agent tự kiểm tra output hoặc dùng critic model.

Tốt cho drafts/code/reports, nhưng self-critique có thể chỉ là “performative reasoning” nếu không có external evidence/tests.

### Multi-agent systems

Nhiều agents có roles khác nhau. Có thể hữu ích nhưng tăng coordination cost. Chương 9.4 sẽ đi sâu.

## Levels of autonomy

| Level | Description | Example | Required control |
|---|---|---|---|
| L0 | No tool use | chat answer | prompt validation |
| L1 | Suggested actions only | draft email | user approval |
| L2 | Read-only tools | search docs/web | access control |
| L3 | Low-risk write tools | create ticket | audit log |
| L4 | High-impact actions with approval | refund/email customer | human checkpoint |
| L5 | Autonomous high-impact actions | trading/security changes | usually unacceptable |

Production design nên assign autonomy per action, không per agent.

## Agent failure taxonomy

| Failure | Cause | Symptom | Mitigation |
|---|---|---|---|
| Goal ambiguity | vague user task | wrong objective | clarify before acting |
| Bad decomposition | poor plan | irrelevant steps | planner eval |
| Tool hallucination | model invents tool | invalid action | schema-constrained tools |
| Argument hallucination | wrong args | failed/unsafe call | validation |
| Observation misread | model ignores result | repeats errors | structured observations |
| Looping | no termination | cost spike | max steps + loop detector |
| Over-planning | too many steps | latency/cost explosion | budget-aware planner |
| Memory pollution | stores wrong facts | persistent future errors | memory validation |
| Tool side effect | bad external action | data changed incorrectly | approval/sandbox |
| Prompt injection | untrusted content controls agent | unsafe action | instruction isolation |
| Premature termination | stops too early | incomplete output | verifier/done criteria |

## Minimal framework-free agent loop

```python
from dataclasses import dataclass, field


@dataclass
class AgentState:
    goal: str
    observations: list[dict] = field(default_factory=list)
    steps: list[dict] = field(default_factory=list)
    max_steps: int = 8
    cost_usd: float = 0.0


def run_agent(goal, policy, tools, guard, terminator):
    state = AgentState(goal=goal)

    for step in range(state.max_steps):
        action = policy.decide(state=state, tools=tools)
        state.steps.append({"step": step, "action": action})

        if action["type"] == "final":
            return {"status": "done", "answer": action["answer"], "trace": state.steps}

        if not guard.allowed(action, state):
            return {"status": "blocked", "reason": "action_not_allowed", "trace": state.steps}

        tool = tools[action["tool_name"]]
        observation = tool.execute(action["arguments"])
        state.observations.append(observation)

        if terminator.should_stop(state):
            return {"status": "stopped", "reason": "terminator", "trace": state.steps}

    return {"status": "escalate", "reason": "max_steps_exceeded", "trace": state.steps}
```

Điểm quan trọng:

- model chỉ đề xuất action,
- guard kiểm soát action,
- tool execute có contract,
- state lưu trace,
- terminator có quyền dừng.

Agent trace tối thiểu:

```json
  "steps": [

      "step": 1,

      "arguments": {"query": "refund policy current"},
      "observation_summary": "Found policy v7",
      "latency_ms": 420,
      "cost_usd": 0.001
    }
  ],
  "status": "completed",
  "final_answer": "..."
}
```

Without trace, agent debugging is storytelling.

## When not to use an agent

Không dùng agent nếu:

- task deterministic,
- steps cố định,
- side effects high-risk,
- latency/cost rất tight,
- eval không rõ,
- tool permissions chưa có,
- bạn không thể log/audit trajectory,
- failure không chấp nhận được.

## Common mistakes & debugging checklist

<Callout type="warning" title="Mistake 1: định nghĩa agent quá rộng">
Không phải mọi LLM app có tool là agent. Agent cần dynamic multi-step control flow, state, action selection và termination.
</Callout>

<Callout type="warning" title="Mistake 2: không có termination conditions">
Agent không có max steps/time/cost rất dễ loop hoặc tiêu tiền không kiểm soát.
</Callout>

<Callout type="danger" title="Mistake 3: để model tự quyết side effects">
Model đề xuất action không đồng nghĩa được phép execute. Runtime phải validate, authorize và audit.
</Callout>

Checklist:

1. Agent có state schema không?
2. Có max steps/time/cost không?
3. Actions có allowed set không?
4. Tools có validation không?
5. Có trace từng step không?
6. Có human escalation không?
7. Có success/done criteria không?
8. Có eval tasks không?
9. Có prompt injection tests không?
