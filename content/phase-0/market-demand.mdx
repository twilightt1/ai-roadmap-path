## 1. Multi-LLM orchestration (Core)

Không model nào tốt cho mọi task. Routing, fallback, cost, caching — **đây là cốt lõi** của AI Engineer.

```text
User query → classify → route
  ├── simple → GPT-4o-mini (rẻ, nhanh)
  ├── reasoning → Claude Opus
  ├── code → GPT-4o
  ├── multimodal → Gemini
  └── fallback → GPT-4o → Claude 3.5 → GPT-3.5 → cache
```

Công ty tuyển **gấp** vì mỗi model có điểm mạnh, cost, rate limit khác — và **bạn** phải biết khi nào dùng cái nào.

**Phase 6, 7** — đây là domain cốt lõi.

## 2. RAG & vector DB (Core)

LLM có **knowledge cutoff** — không biết dữ liệu private. RAG = Embedding search + LLM. Đây là **cách phổ biến nhất** để LLM truy cập kiến thức riêng.

```text
Document → Chunk → Embed → Store (vector DB)
Query → Embed → Retrieve top-k → LLM → Answer
```

**Phase 8** — RAG, vector DB, chunking, hybrid search, reranking.

## 3. AI Agents & agentic systems (Hot)

Agent = LLM + Tools + Memory + Planning. LLM **tự quyết định** gọi tool nào, bao nhiêu bước, và khi nào dừng. Đây là **lĩnh vực lên nhanh nhất** 2026.

```text
User: "Tìm tin tức AI 2026 và viết tóm tắt"
Agent → tool: web_search("AI 2026")
Agent → tool: read_top_3()
Agent → tool: write_summary()
Agent → "Tóm tắt: ..."
```

**Phase 9** — ReAct, tool use, memory, multi-agent, LangGraph.

## 4. LLMOps & production monitoring (Core)

LLM không **deterministic** — cùng input, output khác nhau, và **hallucinate**. Bạn cần **monitor** token, latency, error rate, hallucination, drift.

```python
# Tracking token, latency, error, cost
log = {
    "user": "user_123",
    "model": "gpt-4o-mini",
    "prompt_tokens": 150,
    "completion_tokens": 80,
    "latency_ms": 1200,
    "cost": 0.0001,
    "error": None,
}
```

**Phase 12** — LangSmith, Langfuse, Helicone, Grafana, Prometheus.

## 5. Prompt engineering at scale (Core)

Prompt không chỉ là "nói cho LLM". Production cần:
- **Versioning** — prompt nào, hiệu quả nào, rollback.
- **A/B testing** — prompt A vs B, đo conversion.
- **Prompt injection** — bảo vệ user input.
- **Prompt compression** — giảm token, giảm cost.

```python
# Version prompt
PROMPT_V1 = "Tóm tắt: {text}"
PROMPT_V2 = "Tóm tắt cho trẻ 10 tuổi: {text}"  # A/B
```

**Phase 6** — prompt engineering, CoT, ReAct, structured output.

## 6. Fine-tuning & PEFT (Niche)

Khi RAG không đủ, bạn **fine-tune** — đặc biệt cho domain, format nhất quán, giảm cost. **LoRA** (PEFT) cho fine-tune 70B trên consumer GPU.

```text
Base model (LLaMA 3) → LoRA fine-tune → Domain model
```

**Phase 10** — LoRA, QLoRA, DPO, RLHF, dataset.

## 7. MCP (Model Context Protocol) (Hot)

MCP là **chuẩn mở** của Anthropic — kết nối AI với tool (DB, file, API, agent). Đây là **mới** 2025, nhưng **lên nhanh**.

```text
LLM ← MCP Client → MCP Server → DB / API / file
```

**Phase 7** — MCP server, client, tool.

## 8. Multimodal AI (Hot)

Image, audio, video — không chỉ text. GPT-4o, Claude 3 Vision, Gemini 2.5 **hỗ trợ**.

```text
User: "Mô tả ảnh này" → LLM → "Đây là ảnh ..."
```

**Phase 11** — diffusion, VAE, GAN, multimodal, voice, video.

## 9. Cost optimization & inference efficiency (Core)

AI **đắt**. Một chatbot 1 triệu request/th = 1.000-5.000 USD. Bạn cần **tối ưu**:
- **Model routing** — model rẻ cho 90% request.
- **Caching** — trùng câu hỏi, lấy từ Redis.
- **Quantization** — 4-bit, 8-bit, giảm cost.
- **vLLM** — PagedAttention, 24x throughput.

```python
# Cache trùng query
key = hash(query)
if redis.exists(key):
    return redis.get(key)
```

**Phase 12, 15** — quantization, vLLM, distillation, SLM.
