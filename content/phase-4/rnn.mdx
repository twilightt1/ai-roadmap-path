Recurrent Neural Networks (RNN) là kiến trúc cổ điển cho dữ liệu tuần tự: text, audio, time-series. Transformer đã thay thế RNN trong hầu hết NLP hiện đại, nhưng RNN/LSTM vẫn là bài học quan trọng: nó cho bạn thấy vấn đề memory, long-range dependency, vanishing gradient, và vì sao attention/transformer ra đời. Trước GPT/BERT, machine translation, speech recognition, sentiment analysis đều dùng RNN/LSTM/GRU; nhiều hệ thống time-series nhỏ vẫn dùng RNN variants. Khi học RNN, bạn sẽ hiểu sequence được xử lý từng bước thế nào, hidden state lưu "memory" ra sao, vì sao gradient qua thời gian bị vanish, vì sao LSTM cần gate, và vì sao attention giải quyết bottleneck. Trong vai trò giảng viên AI, tôi muốn bạn nắm hidden state, theo dõi input/output shape của `nn.RNN`/`nn.LSTM`/`nn.GRU`, hiểu vanishing/exploding gradient qua thời gian, giải thích LSTM gates, phân biệt LSTM và GRU, và biết khi nào dùng bidirectional RNN. Transformer không xuất hiện từ hư không — nó giải quyết các vấn đề của RNN, nên hiểu RNN giúp bạn hiểu transformer sâu hơn.

## 1. The Incident (Tình huống sự cố)

<Callout type="danger" title="Báo cáo sự cố: Sentiment classifier "quên" negation trên review dài, recall negative rơi 41%">
**Bối cảnh:** Một team build hệ thống phân loại cảm xúc cho sản phẩm review của sàn thương mại điện tử. Họ dùng basic `nn.RNN` (1 layer, hidden 64) vì "đơn giản, nhẹ". Review dài khoảng 200-400 token. Train trên tập review ngắn (15-50 token) đạt accuracy 92% trên test set.

Dùng basic `nn.RNN` không gate cho review dài 200+ token. Test set 80% là review ngắn, không đo theo bucket độ dài. Không gradient clipping. Không kiểm gradient norm. Đội QA phê duyệt dựa trên test set thiên lệch.

1. Basic `nn.RNN` không học dependency xa vì vanishing gradient → review có "not good" ở đầu bị classify "neutral".
2. Sau deploy, recall cho class "negative" trên review dài 200+ token tụt xuống 41%.
3. Loss train không giảm thêm, gradient norm rất nhỏ (< 1e-5) sau vài epoch — tín hiệu vanishing gradient bị bỏ qua.
4. Test set có 80% review ngắn → độ QA phê duyệt dựa trên metric thiên lệch, không phát hiện.
5. Không dùng LSTM/GRU có gate, không gradient clipping, không đo theo bucket độ dài.

**Hậu quả:** Hệ thống miss cảnh báo phản hồi tiêu cực, team customer service bị trễ vì dashboard sentiment luôn "neutral". Phải retrain 3 ngày mới kịp mở lại.
</Callout>

Bài học rút ra: basic RNN không học được dependency xa vì vanishing gradient qua thời gian. Đừng chỉ test trên tập ngắn — phải đo trên tệp dài theo độ dài sequence. Dùng LSTM/GRU có gate, gradient clipping, và (đặc biệt) đo độ dài sequence để phân tích. Sau này, attention/transformer giải quyết vấn đề này ở mức độ sâu hơn.

## 2. Architecture Blueprint (RNN/LSTM Forward Pass)

```text
Input sequence: x_1, x_2, x_3, ..., x_T  (batch=B, seq=T, dim=D)
        │
        ▼
┌──────────────────────────────────────────────────┐
│  Embedding / Token → Vector (B, T, E)             │
└──────────────────────────────────────────────────┘
        │
        ▼
┌──────────────────────────────────────────────────┐
│  RNN / LSTM / GRU cell (unrolled qua T time steps) │
│                                                   │
│  h_t = f(W_x x_t + W_h h_{t-1} + b)               │
│  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐         │
│  │ h_1 │→│ h_2 │→│ h_3 │→│ ... │→│ h_T │         │
│  └─────┘ └─────┘ └─────┘ └─────┘ └─────┘         │
│                                                   │
│  (LSTM) c_t = forget gate, input gate, output gate │
│  c_t gradient chảy tốt hơn → nhớ dependency xa    │
└──────────────────────────────────────────────────┘
        │
        ▼
┌──────────────────────────────────────────────────┐
│  Head: pooling / last hidden / attention pooling  │
│  (B, T, H) → (B, H)  (B, 2H) nếu bidirectional   │
└──────────────────────────────────────────────────┘
        │
        ▼
┌──────────────────────────────────────────────────┐
│  Output: logits / classes / next token            │
└──────────────────────────────────────────────────┘
        │
        ▼
    Loss ──► BPTT (Backpropagation Through Time)
```

Key insight: hidden state là "bộ nhớ tóm tắt" chạy qua T step. Mỗi step, gradient phải đi ngược qua nhiều `W_h` — nếu các hệ số nhỏ, gradient vanish, RNN quên quá khứ.

## 3. Core intuition: hidden state là memory

RNN đọc sequence từng token/time step. Tại mỗi bước, nó nhận input hiện tại `x_t` và hidden state trước `h_{t-1}`:

```text
h_t = f(W_x x_t + W_h h_{t-1} + b)
```

Hidden state giống "bộ nhớ tóm tắt" những gì đã đọc trước đó.

Ví dụ sentiment:

```text
"The movie was not ..."
```

Khi đến từ sau, hidden state cần nhớ từ `not` để hiểu sentiment.

<Callout type="info" title="RNN là cầu nối tư duy tới Transformer">
Transformer không xuất hiện từ hư không. Nó giải quyết các vấn đề của RNN: xử lý tuần tự chậm, memory bottleneck, khó học dependency xa. Hiểu RNN giúp bạn hiểu transformer sâu hơn.
</Callout>

## 4. RNN shape trong PyTorch

```python
import torch
from torch import nn

rnn = nn.RNN(input_size=32, hidden_size=64, batch_first=True)
x = torch.randn(16, 20, 32)  # batch=16, seq_len=20, input_size=32
out, h_n = rnn(x)

print(out.shape)  # (16, 20, 64)
print(h_n.shape)  # (1, 16, 64) -> num_layers, batch, hidden
```

`out` chứa hidden state ở mọi time step. `h_n` là hidden cuối cùng.

## 5. Worked example by hand: RNN update đơn giản

Giả sử scalar RNN:

```text
h_t = tanh(0.5*x_t + 0.8*h_{t-1})
h_0 = 0
sequence x = [1, 2]
```

Step 1:

```text
h_1 = tanh(0.5*1 + 0.8*0) = tanh(0.5) ≈ 0.462
```

Step 2:

```text
h_2 = tanh(0.5*2 + 0.8*0.462)
    = tanh(1 + 0.370)
    = tanh(1.370) ≈ 0.879
```

Hidden state tích lũy thông tin từ quá khứ.

## 6. Vanishing gradient qua thời gian

Backpropagation qua RNN gọi là BPTT (Backpropagation Through Time). Gradient từ cuối sequence phải đi ngược qua nhiều bước:

```text
gradient ≈ product of many W_h terms
```

Nếu các hệ số nhỏ hơn 1, gradient nhỏ dần. Nếu lớn hơn 1, gradient explode.

<Callout type="danger" title="Long dependency là điểm yếu RNN cơ bản">
RNN cơ bản rất khó nhớ thông tin từ hàng chục/hàng trăm bước trước. Đây là lý do LSTM/GRU ra đời, và sau đó attention/transformer thay thế trong NLP.
</Callout>

## 7. LSTM

LSTM thêm **cell state** `c_t` và gates để kiểm soát thông tin.

Các gates:

| Gate | Vai trò |
|---|---|
| Forget gate | quên phần nào của memory cũ |
| Input gate | ghi thông tin mới nào vào memory |
| Output gate | xuất phần nào của memory ra hidden state |

```python
lstm = nn.LSTM(input_size=32, hidden_size=64, num_layers=2, batch_first=True)
out, (h_n, c_n) = lstm(x)

print(out.shape)  # (16, 20, 64)
print(h_n.shape)  # (2, 16, 64)
print(c_n.shape)  # (2, 16, 64)
```

Cell state là đường truyền dài hạn giúp gradient chảy tốt hơn.

---

## 8. GRU

GRU đơn giản hơn LSTM:

- Không có cell state riêng.
- Có update gate và reset gate.
- Ít parameter hơn, nhanh hơn.

```python
gru = nn.GRU(input_size=32, hidden_size=64, batch_first=True)
out, h_n = gru(x)
```

Trong nhiều task, GRU performance gần LSTM nhưng nhanh hơn.

---

## 9. Bidirectional RNN

Bidirectional RNN đọc sequence hai chiều: trái sang phải và phải sang trái.

```python
bilstm = nn.LSTM(32, 64, batch_first=True, bidirectional=True)
out, _ = bilstm(x)
print(out.shape)  # (16, 20, 128) vì 2 chiều concat
```

Dùng khi bạn có toàn bộ sequence: classification, NER, tagging. Không dùng cho autoregressive generation vì khi sinh token, tương lai chưa biết.

---

## 10. Seq2Seq và beam search

Seq2Seq dùng encoder RNN đọc source, decoder RNN sinh target.

```text
source sentence → encoder hidden/context → decoder → target sentence
```

Bottleneck: toàn bộ source bị nén vào một context vector. Câu dài làm mất thông tin.

Beam search giữ top-k candidate sequences thay vì greedy:

```text
beam=3 → giữ 3 câu có score tốt nhất qua mỗi bước
```

## 11. Common mistakes

<Callout type="warning" title="Mistake 1: nhầm batch_first">
Nếu `batch_first=True`, input shape là `(batch, seq, feature)`. Nếu false, shape là `(seq, batch, feature)`. Nhầm shape là lỗi kinh điển. In shape trước khi gọi `rnn(x)`, thêm `assert` vào đầu forward.
</Callout>

<Callout type="warning" title="Mistake 2: lấy hidden state ở padding token">
Nếu lấy last timestep mà sequence thật ngắn hơn, bạn có thể lấy state của padding. Dùng `pack_padded_sequence` + `pad_packed_sequence` + gather last hidden theo sequence length, hoặc attention pooling.
</Callout>

<Callout type="danger" title="Mistake 3: dùng bidirectional cho generation autoregressive">
Bidirectional nhìn tương lai. Với generation token-by-token, điều đó là leakage (test leakage). Chỉ dùng bidirectional cho tasks có toàn bộ input. Dùng unidirectional cho autoregressive generation.
</Callout>

**Checklist:**

1. Input shape có đúng `batch_first` không?
2. Có padding mask/lengths không?
3. Hidden output dimension có nhân 2 vì bidirectional không?
4. Gradient có explode không? Có clipping không?
5. Task có cần future context không?
6. Sequence quá dài có khiến training chậm không?

## Implementation Playbook: LSTM classifier

```python
import torch
from torch import nn

class LSTMClassifier(nn.Module):
    def __init__(self, vocab_size, embed_dim, hidden_dim, num_classes):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim, padding_idx=0)
        self.lstm = nn.LSTM(embed_dim, hidden_dim, batch_first=True, bidirectional=True)
        self.fc = nn.Linear(2 * hidden_dim, num_classes)

    def forward(self, input_ids):
        x = self.embedding(input_ids)       # (B, T, E)
        out, (h_n, c_n) = self.lstm(x)      # out: (B, T, 2H)
        last = out[:, -1, :]                # đơn giản: lấy bước cuối
        return self.fc(last)

model = LSTMClassifier(vocab_size=10000, embed_dim=128, hidden_dim=64, num_classes=2)
input_ids = torch.randint(0, 10000, (32, 50))
logits = model(input_ids)
print(logits.shape)  # (32, 2)
```

Lưu ý: với padding, lấy `out[:, -1, :]` có thể lấy padding state. Production nên dùng sequence lengths hoặc attention pooling.

**Extend lab milestones:**

1. Thay `out[:, -1, :]` bằng `pack_padded_sequence` / `pad_packed_sequence` + gather last hidden theo sequence length thật.
2. Thêm gradient clipping trong training loop (`torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)`).
3. Đo accuracy trên từng bucket độ dài (short < 50, medium 50-200, long > 200) và log lại.
4. Thay LSTM bằng `nn.RNN` và `nn.GRU`, so sánh accuracy và gradient norm.
5. Thay `last = out[:, -1, :]` bằng attention pooling (mean-pooling mask hoặc attention pooling) để chống bias.
6. Thêm `num_layers=2` và `dropout=0.3` để check overfitting trên long sequence.

## Post-Mortem & Troubleshooting (SRE/Gỡ lỗi)

<Callout type="warning" title="Pitfall 1: nhầm batch_first">
**Triệu chứng:** Loss không giảm, hoặc shape mismatch, hoặc `out` shape sai (batch, seq, feature) vs (seq, batch, feature).

**Fix:** Nếu `batch_first=True`, input shape là `(batch, seq, feature)`. Nếu false, shape là `(seq, batch, feature)`. In shape trước khi gọi `rnn(x)`, thêm `assert` vào đầu forward.
</Callout>

<Callout type="warning" title="Pitfall 2: lấy hidden state ở padding token">
**Triệu chứng:** Accuracy trên test set có padding (batch > 1) tệ hơn nhiều so với batch=1, hoặc model học "nhồi" short sequence.

**Fix:** Dùng `pack_padded_sequence` + `pad_packed_sequence` + gather last hidden theo sequence length, hoặc attention pooling.
</Callout>

<Callout type="danger" title="Pitfall 3: dùng bidirectional cho generation autoregressive">
**Triệu chứng:** Loss train tốt, nhưng khi sinh token, output bị "nhìn thấy tương lai" (test leakage), hoặc model không thể deploy vì không biết token tương lai.

**Fix:** Bidirectional nhìn tương lai. Với generation token-by-token, điều đó là leakage. Chỉ dùng bidirectional cho tasks có toàn bộ input. Dùng unidirectional cho autoregressive generation.
</Callout>

## Cost / Latency / Reliability Unit Economics

| Quyết định | Lợi ích | Chi phí/rủi ro |
|---|---|---|
| Dùng `nn.RNN` cơ bản | Đơn giản, nhẹ, ít parameter, nhanh | Vanishing gradient, khó nhớ dependency xa, không phù hợp cho long sequence |
| Dùng `nn.LSTM` | Cell state nhớ dài hạn, gradient chảy tốt, ổn định | ~4× parameter hơn RNN, chậm hơn, train lâu hơn |
| Dùng `nn.GRU` | Ít parameter hơn LSTM, nhanh hơn, performance gần LSTM | Không có cell state riêng, đôi khi kém hơn LSTM trên long sequence |
| Dùng `nn.BiLSTM` | Đọc 2 chiều, context tốt hơn, accuracy cao cho classification/NER | 2× parameter, 2× forward, không dùng cho autoregressive generation |
| Dùng `pack_padded_sequence` | Xử lý variable length đúng, không hỏng hidden state | Cần tracking sequence lengths, dễ confuse khi unroll |
| Dùng gradient clipping | Ngăn exploding gradient, train ổn định | Chặn gradient > 1.0 có thể giảm tốc độ học, không giải quyết vanishing gradient |
| Dùng beam search | Tìm top-k candidate, BLEU tốt hơn greedy | Chậm hơn, 2× memory, khó debug, không phù hợp real-time |
| Dùng Seq2Seq RNN | Đơn giản, dễ train | Context bottleneck, không học dependency xa, không scale cho câu dài |
| Dùng attention/transformer | Tránh bottleneck, song song, dependency xa | Chỉ dùng khi có đủ data, không dùng cho time-series nhỏ |

## Answer key

1. Tóm tắt thông tin quá khứ đã đọc.
2. Backpropagation Through Time — lan gradient ngược qua các time steps.
3. Forget, input, output gate.
4. Ít gate hơn, không có cell state riêng, ít parameter hơn.
5. Hidden dim thường nhân đôi vì concat forward/backward.
6. Nén toàn source vào một context vector.
7. Greedy giữ 1 candidate tốt nhất; beam giữ top-k candidates.
