Unsupervised Learning là học khi **không có nhãn**. Trong AI production điều này rất thường xuyên: hàng triệu user logs không có label churn, hàng nghìn documents không biết topic, hàng triệu embeddings không biết cluster nào chứa lỗi, hàng nghìn query không biết intent nào mới nổi. Clustering và dimensionality reduction không phải "model phụ" — là công cụ khám phá data, debug RAG, phát hiện anomaly, hiểu representation. Embedding model của LLM chính là unsupervised-trained object lớn nhất bạn từng dùng. Nếu bạn chạy K-Means trên embedding 1536 chiều thô chưa normalize, centroid có thể hoàn toàn vô nghĩa và cluster "khám phá" ra chỉ là noise; nếu bạn over-interpret một plot UMAP/t-SNE 2D, bạn kết luận sai về cấu trúc high-D. Trong vai trò giảng viên AI, tôi muốn bạn nắm vững K-Means, DBSCAN/HDBSCAN, PCA/t-SNE/UMAP, distance metrics và anomaly detection — rồi luôn validate cluster bằng sample thật và downstream task, không tin plot đẹp.

Hãy tưởng tượng bạn xây hệ thống RAG: embed 50,000 chunks từ docs nội bộ, retrieval đôi khi sai. Bạn cần trả lời: documents có cluster rõ theo domain không? có chunks outlier do OCR lỗi không? query user nằm gần cluster relevant không? có nhóm docs trùng lặp không? có topic thiếu data không? Không có label — chỉ có unsupervised tools: K-Means, DBSCAN/HDBSCAN, PCA, UMAP, t-SNE. Supervised trả lời "dự đoán đúng không?"; unsupervised giúp hỏi "data của tôi có cấu trúc gì?" — thường là bước đầu để sửa retrieval, cải thiện dataset, phát hiện lỗi pipeline.

## 1. The Incident (Tình huống sự cố)

<Callout type="danger" title="Báo cáo sự cố: K-Means «khám phá» cluster trên embedding 1536 chiều thô, đặt tên sai topic">
**Bối cảnh:** Một team RAG audit 50k chunks bằng K-Means. Embedding `all-mpnet-base-v2` (768 chiều) và `text-embedding-3-large` (1536 chiều) — chưa L2 normalize. Chạy `KMeans(k=10)` trực tiếp, dùng Euclidean distance, `init="k-means++"`, `n_init=10`. Team inspect 5 sample mỗi cluster, đặt tên topic, ship vào dashboard.

Không normalize embedding trước khi distance. Không giảm chiều trước khi cluster (kẻ "curse of dimensionality"). Dùng Euclidean cho embedding thô. Dựa vào 5 sample/cluster để đặt tên. Không validate bằng downstream retrieval task.

1. Embedding 1536 chiều thô + Euclidean → curse of dimensionality: khoảng cách giữa mọi cặp điểm gần như bằng nhau, centroid meaningless, K-Means chia theo noise của magnitude vector thay vì semantic.
2. Một cluster có 5 sample toàn "ML fundamentals" → team đặt tên cluster đó là "ML"; thực tế cluster chứa 4500 docs hỗn hợp ML + engineering + DevOps.
3. Outlier (chunks OCR lỗi, embeddings có norm rất lớn vì embedding model gặp token lạ) bị centroid kéo về phía một cluster — làm cluster top-1 sample trông "đẹp" nhưng phần lớn member lệch.
4. Elbow plot cho elbow mơ hồ vì inertia trong high-D không còn phân biệt rõ giữa k=5, k=10, k=20.
5. Silhouette score thấp (0.08) nhưng team vẫn ship vì "có cluster tổng hơn không cluster".

**Hậu quả:** Dashboard topic báo cáo 10 cluster có tên rất tự tin. Plan nội bộ tag chunks theo cluster → retrieval quality giảm 15% sau 2 tuần vì chunks tag sai topic. Mất 4 ngày trace: (a) normalize embedding (`normalize_embeddings=True`), (b) giảm chiều UMAP/PCA xuống 50 trước khi K-Means, (c) thay distance thành cosine và t-SNE/UMAP để visualize. Cluster mới thật sự phân theo topic, silhouette 0.32.
</Callout>

Bài học rút ra: high-dimensional distance "concentration" làm K-Means trên embedding thô vô nghĩa. Ai distance — phải normalize hoặc dùng cosine. Luôn giảm chiều (PCA/UMAP) trước khi cluster trên embedding cao chiều. Đặt tên cluster chỉ dựa trên 5 sample là bẫy — sample ít nhất 20-50, kèm centroid/median; validate bằng downstream (retrieval same-cluster) hoặc domain review. Một cluster "đẹp trên paper" có thể hoàn toàn noise nếu distance metric sai.

## 2. Architecture Blueprint (Unsupervised Workflow)

                ┌──────────────────────────────────────────────────────────────┐
                │            UNSUPERVISED ANALYSIS WORKFLOW (RAG audit)         │
                └──────────────────────────────────────────────────────────────┘

   raw embeddings → ┌─────────────┐   ┌─────────────┐   ┌─────────────┐
   (high-dim)      │  normalize   │──▶│ reduce dim  │──▶│   cluster   │──▶ labels
                   │  (L2 / std)  │   │  PCA / UMAP │   │ KMeans/HDBSCAN│
                   └─────────────┘   └─────────────┘   └──────┬──────┘

                                 ┌─────────────┐                    ┌──────────────┐
                                 │  visualize  │                    │   validate   │
                                 │ UMAP / t-SNE│                    │ sample 20-50 │

## 3. Core intuition: clustering là gì?

Gom các điểm tương tự vào cùng nhóm. Input chỉ có `X` không có `y`. Output = cluster labels (không có ground truth).

| Data | Cluster có thể là |
|---|---|
| Support tickets | billing, login, refund, technical, security |
| User behavior | power users, dormant users, trial users, churned-pending |
| Documents | HR, finance, engineering, sales, marketing |
| Embeddings | semantic topics, intent clusters |
| Server logs | normal traffic, traffic spike, attack pattern |

<Callout type="info" title="Clustering không phải classification">
Classification có label. Clustering không có label — kết quả là hypothesis về cấu trúc, cần đánh giá bằng domain expertise, downstream task hoặc human review. Cluster số 0 không tự nhiên là "billing", bạn phải xem sample để đặt tên.
</Callout>

## 4. Formal definition

Clustering tìm partition `C_1, ..., C_k` của data sao cho within-cluster distance nhỏ, between-cluster distance lớn. K-Means minimize **inertia** (within-cluster sum of squares):

```text
inertia = Σ ||x - centroid(x)||^2
```

Distance metric quyết định cluster shape. Tổng quát:

```text
Euclidean :  d(x, y) = sqrt(Σ (x_i - y_i)^2)
Cosine     :  d(x, y) = 1 - (x · y) / (||x|| ||y||)
Manhattan  :  d(x, y) = Σ |x_i - y_i|
```

---

## 5. Distance metrics — lựa chọn quan trọng đầu tiên

Clustering dựa trên distance. Distance metric quyết định cluster shape.

### Euclidean

`d(x, y) = sqrt(Σ (x_i - y_i)^2)`. K-Means, DBSCAN mặc định dùng. Hợp feature numeric có scale tương tự.

### Cosine

`cosine(x, y) = (x · y) / (||x|| ||y||)`. Đo **góc** giữa vector, không quan tâm magnitude. Hợp text embedding (orientation quan trọng hơn magnitude). Hầu hết embedding model output đã normalized nên `cosine = dot product`.

### Manhattan (L1)

`d(x, y) = Σ |x_i - y_i|`. Robust với outlier hơn Euclidean. Hợp grid-like data.

### Mahalanobis

Tính distance có xét covariance — biến thành "số std theo hướng đó". Hợp khi feature có variance khác nhau và correlated.

<Callout type="warning" title="Scale trước khi distance">
K-Means/DBSCAN dùng Euclidean bị feature lớn thống trị. Numeric tabular **scale trước** (`StandardScaler`). Embedding đã normalize → dùng cosine/dot. Trộn metric có thể sai — embedding 1536 chiều thô + Euclidean hay bị "curse of dimensionality".
</Callout>

## 6. K-Means

Giả định mỗi cluster có centroid. Mỗi điểm thuộc cluster có centroid gần nhất.

### Thuật toán — Lloyd's algorithm

1. Khởi tạo `k` centroid (ngẫu nhiên hoặc K-Means++).
2. **Assignment**: gán mỗi điểm vào centroid gần nhất (Euclidean).
3. **Update**: centroid = mean các điểm trong cluster.
4. Lặp 2-3 đến khi assignment ổn định (hoặc max iter).

```python
from sklearn.cluster import KMeans

km = KMeans(n_clusters=5, init="k-means++", n_init=10, random_state=42)
labels = km.fit_predict(X)
print(km.cluster_centers_, km.inertia_)
```

`n_init=10`: chạy 10 lần khởi tạo khác, chọn inertia thấp nhất — K-Means chỉ hội tụ local optimum nên cần restarts. `init="k-means++"` khởi tạo thông minh, hội tụ nhanh hơn random.

### Objective và limit

K-Means minimize **inertia** (within-cluster sum of squares): `inertia = Σ ||x - centroid(x)||^2`. Giả định: cluster **hình cầu** (spherical), **kích thước tương tự**, **Euclidean** hợp lý. Nếu vi phạm, K-Means chia sai — ví dụ cluster hình vòng tròn, mật độ lệch, hoặc overlap.

## 7. Worked example by hand: 1D K-Means

Data 1D: `[1, 2, 3, 10, 11, 12]`. Chọn `k=2`, centroid ban đầu `c1 = 1, c2 = 12`.

**Lần lặp 1 — Assignment:** `1,2,3` gần c1 (khoảng cách 0,1,2); `10,11,12` gần c2 (2,1,0).
**Update:** `c1 = mean(1,2,3) = 2`, `c2 = mean(10,11,12) = 11`.

**Lần lặp 2 — Assignment** (c1=2, c2=11):

```text
1  gần c1 (|1-2|=1  < |1-11|=10)  → A
2  gần c1 (0       < 9)          → A
3  gần c1 (1       < 8)          → A
10 gần c2 (1       < 8)          → B
11 gần c2 (0       < 9)          → B
12 gần c2 (1       < 10)         → B
```

Không đổi. Hội tụ:

```text
Cluster A: [1,2,3] centroid 2
Cluster B: [10,11,12] centroid 11
inertia = (1+0+1) + (1+0+1) = 4
```

<Callout type="warning" title="K-Means không biết hình dạng phức tạp">
Thích cluster dạng cầu/tròn, kích thước tương tự. Nếu cluster hình vòng cung, mật độ khác, nhiều outlier → chia sai. DBSCAN/HDBSCAN hoặc spectral clustering phù hợp hơn.
</Callout>

## 8. Chọn số cluster k

Không có đáp án hoàn hảo. Heuristics:

### Elbow method

Chạy K-Means nhiều `k`, đo inertia, chọn điểm "gấp khuỷu" — inertia giảm chậm lại.

```python
from sklearn.cluster import KMeans

inertias = []
for k in range(2, 15):
    km = KMeans(n_clusters=k, n_init=10, random_state=42).fit(X)
    inertias.append(km.inertia_)
```

Vẽ inertia theo k — điểm uốn là heuristic. Mơ hồ, subjective — luôn kết hợp domain.

### Silhouette score

```text
s_i = (b_i - a_i) / max(a_i, b_i)
  a_i = mean distance to same cluster (cohesion)
  b_i = mean distance to nearest foreign cluster (separation)
```

- `s ≈ 1`: điểm đúng cluster, cách cluster khác xa.
- `s ≈ 0`: điểm giữa hai cluster (overlap).
- `s < 0`: điểm có thể sai cluster.

```python
from sklearn.metrics import silhouette_score

for k in range(2, 15):
    labels = KMeans(n_clusters=k, n_init=10, random_state=42).fit_predict(X)
    print(k, silhouette_score(X, labels))
```

### Davies-Bouldin index

Tỉ lệ within-cluster scatter vs between-cluster distance — càng thấp càng tốt. Không cần ground truth.

<Callout type="tip" title="Metric cluster là heuristic — luôn validate downstream">
Silhouette/elbow chỉ gợi ý `k`. Để quyết định: (a) sample 20-50 điểm mỗi cluster xem nội dung; (b) dùng cluster làm feature cho downstream supervised task; (c) gọi domain expert xem cluster có ý nghĩa không.
</Callout>

## 9. DBSCAN

Không cần chọn `k`. Tìm vùng **mật độ cao** phân tách bởi vùng mật độ thấp.

### Tham số

- `eps`: bán kính neighborhood.
- `min_samples`: số điểm tối thiểu trong bán kính `eps` để một điểm là **core point**.

### Khái niệm

- **Core point**: có ≥ `min_samples` neighbor trong bán kính `eps`.
- **Border point**: trong `eps` của một core, nhưng không phải core.
- **Noise**: không core, không border. Label `-1`.

Cluster hình bằng cách grow từ core points, nối các core chứa nhau.

```python
from sklearn.cluster import DBSCAN

db = DBSCAN(eps=0.5, min_samples=5)
labels = db.fit_predict(X_scaled)
print((labels == -1).sum(), "noise points")
```

### When DBSCAN works well

- Cluster hình dạng bất kỳ (vòng tròn, lưỡi liềm, spiral).
- Phát hiện outlier/anomaly (label `-1`).
- Không biết số cluster.

### Limit

- Khó chọn `eps` (dùng k-distance plot).
- Kém với high-dimensional sparse data.
- Nếu mật độ cluster khác nhau, một `eps` không đủ.

### HDBSCAN — improved version

Tự chọn `eps` theo hierarchy, robust hơn DBSCAN. Recommend over DBSCAN trong hầu hết use case.

```python
from sklearn.cluster import HDBSCAN

labels = HDBSCAN(min_cluster_size=10, min_samples=5).fit_predict(X)
```

<Callout type="danger" title="High-dimensional distance bị kém ý nghĩa">
Trong high-dimensional space, khoảng cách giữa điểm có xu hướng giống nhau hơn (concentration of distances). DBSCAN/KMeans trên embedding 1536 chiều thô có thể kém. Thường giảm chiều bằng PCA/UMAP trước để visualize hoặc cluster cẩn thận.
</Callout>

## 10. Anomaly detection — unsupervised

Unsupervised tự nhiên cho anomaly detection vì outlier là điểm "khác đa số".

| Method | Khi nào dùng |
|---|---|
| DBSCAN label -1 | Density-based outlier, shape bất kỳ |
| Isolation Forest | fast, scalable; tree-based outlier isolation |
| Local Outlier Factor (LOF) | Density-based, local outlier |
| One-Class SVM | Boundary of "normal" data, kernel trick |
| Reconstruction error (autoencoder) | Neural net: train trên normal, outlier có error cao |

```python
from sklearn.ensemble import IsolationForest

iso = IsolationForest(contamination=0.05, random_state=42)
labels = iso.fit_predict(X)   # -1 outlier, 1 normal
```

Trong production: monitor traffic, alert unusual user behavior, fraud flagging, RAG retrieve giám sát chất lượng. Anomaly detection dùng `contamination` để ước tỉ lệ outlier — phối hợp với downstream investigation (vì unsupervised, recall không measure trực tiếp nếu không có label).

## 11. PCA — Principal Component Analysis

Tìm trục mới **giữ variance lớn nhất**. Giảm chiều tuyến tính.

### Intuition

Data 2D nằm dọc đường chéo. PCA tìm:

- **Component 1**: hướng variance lớn nhất.
- **Component 2**: hướng vuông góc component 1, variance còn lại.

Nếu component 1 giữ 80% variance, component 2 giữ 15% → 95% thông tin trong 2 chiều.

```python
from sklearn.decomposition import PCA

pca = PCA(n_components=2)
X_pca = pca.fit_transform(X_scaled)
print(pca.explained_variance_ratio_)   # vd [0.80, 0.15] → 95% cumulative
```

### Formal view

PCA tìm orthogonal directions `v_1, v_2, ...` sao cho variance của projection `Xv` lớn nhất, subject to orthogonality:

```text
max_v  Var(X v)   s.t.   ||v|| = 1
components tiếp theo: orthogonal với components trước
```

### Liên hệ Phase 2 — SVD và eigendecomposition

PCA có hai cách tính:

1. **Eigendecomposition** của covariance matrix `C = (1/n) X^T X` (sau khi centered): eigenvalues = explained variance, eigenvectors = components.
2. **SVD** của centered data matrix `X = U Σ V^T`: columns của `V` là principal components, singular values² tied to variance.

sklearn dùng SVD — ổn định hơn eigendecomposition với matrix nearly singular.

### Khi nào dùng PCA

- **Giảm chiều trước clustering/visualization** — giảm curse of dimensionality.
- **Preprocessing cho supervised** — model linear hoạt động tốt hơn với high-collinearity.
- **Denoising** — giữ component chính, bỏ component nhỏ (noise).
- **Compression** — lưu data dạng thấp chiều.

### Limit

- **Tuyến tính**: capture quan hệ linear. Quan hệ phi tuyến cần t-SNE/UMAP/autoencoder.
- **Interpretability**: component là linear combination của feature gốc, khó diễn giải trực tiếp.
- **Outlier sensitive**: outlier kéo component hướng nó. Cần scale (và có thể robust PCA).
- **Variance ≠ relevance**: component variance lớn nhất không nhất thiết chứa signal dự đoán — high variance có thể là noise.

## 12. t-SNE — visualization không phải dimensionality reduction

t-SNE (t-distributed Stochastic Neighbor Embedding) non-linear, đẹp hơn PCA cho visualization **local neighborhoods**.

### Đặc điểm

- Tốt thấy cluster local — neighbors trong high-D vẫn neighbors trong 2D.
- **Chậm** với data lớn (O(n log n) Barnes-Hut).
- Khoảng cách **global không đáng tin** — hai cluster xa trên plot không chắc xa trong high-D.
- Nhạy `perplexity` (thường 5-50).
- **Không dùng cho downstream task** — không có `transform` ổn định cho data mới.

```python
from sklearn.manifold import TSNE

X_tsne = TSNE(n_components=2, perplexity=30, random_state=42).fit_transform(X)
```

---

## 13. UMAP — fast và giữ global structure

Nhanh hơn t-SNE, giữ **global structure** tốt hơn và **học transform có thể apply cho data mới**.

```python
import umap

reducer = umap.UMAP(n_neighbors=15, min_dist=0.1, random_state=42)
X_umap = reducer.fit_transform(embeddings)
new_emb = reducer.transform(new_embeddings)   # apply cho data mới
```

### Tham số chính

- `n_neighbors` nhỏ → focus local (như t-SNE). Lớn → focus global.
- `min_dist` nhỏ → cluster chặt. Lớn → cluster rải rác.

### UMAP cho clustering

Trong production RAG/embedding audit, flow phổ biến: **UMAP giảm xuống 5-10 chiều → HDBSCAN cluster**. UMAP làm HDBSCAN hoạt động tốt trên high-dimensional embeddings.

<Callout type="warning" title="Plot đẹp không đồng nghĩa model đúng">
UMAP/t-SNE tạo visualization để khám phá, không phải bằng chứng cuối cùng. Hai cluster xa trên plot không nhất thiết xa trong original space. Đừng over-interpret — sample thật ra xem mỗi cluster chứa gì.
</Callout>

## 14. Common mistakes

<Callout type="warning" title="Mistake 1: cluster trước khi scale">
K-Means/DBSCAN dùng distance, feature scale khác làm feature lớn thống trị. Numeric tabular → scale trước. Normalized embeddings → cosine/dot phù hợp hơn Euclidean thô.
</Callout>

<Callout type="warning" title="Mistake 2: ép mọi thứ vào cluster">
K-Means luôn gán mọi điểm vào cluster, kể cả outlier → outlier kéo centroid và hỏng cluster. Cần phát hiện noise → DBSCAN/HDBSCAN hoặc anomaly detection riêng.
</Callout>

<Callout type="danger" title="Mistake 3: đặt tên cluster theo vài ví dụ đẹp">
Nhìn 5 sample và đặt tên "billing" nhưng cluster có thể pha nhiều topic. Luôn sample 20-50 điểm, xem centroid/neighbors, kiểm tra downstream — chuẩn bị đổi tên khi inspect thêm.
</Callout>

<Callout type="warning" title="Mistake 4: over-interpret UMAP/t-SNE plot">
Plot 2D không phải ground truth. Khoảng cách global không đáng tin, hai cluster "gần nhau" trên plot có thể xa trong high-D. UMAP có `random_state` nhưng kết quả vẫn nhạy tham số.
</Callout>

<Callout type="warning" title="Mistake 5: dùng Euclidean cho embedding thô">
Embedding 1536 chiều thô + Euclidean = curse of dimensionality. Normalize và dùng cosine, hoặc giảm chiều bằng PCA/UMAP trước khi distance.
</Callout>

## Implementation Playbook: visualize document embeddings

Baseline RAG/embedding audit. Chạy được, đo được, sửa được.

```python
from sentence_transformers import SentenceTransformer
from sklearn.cluster import KMeans
from sklearn.metrics import silhouette_score
import umap
import matplotlib.pyplot as plt

texts = [
    "How do I reset my password?", "I cannot login to my account.",
    "Please refund my payment.", "Why was I charged twice?",
    "Cancel my subscription refund.",
    "How to deploy Docker on Kubernetes?", "My pod is crash looping in Kubernetes.",
    "kubectl logs returns empty.",
    "Upgrade my plan to pro.", "What is my current plan?",
]

model = SentenceTransformer("all-MiniLM-L6-v2")
emb = model.encode(texts, normalize_embeddings=True)   # L2 normalize → cosine hợp

best_k, best_s = None, -1
for k in range(2, 5):
    labels = KMeans(n_clusters=k, n_init=10, random_state=42).fit_predict(emb)
    s = silhouette_score(emb, labels, metric="cosine")
    if s > best_s:
        best_k, best_s = k, s
print("best k:", best_k, "silhouette:", round(best_s, 3))

labels = KMeans(n_clusters=best_k, n_init=10, random_state=42).fit_predict(emb)
reducer = umap.UMAP(n_neighbors=3, min_dist=0.1, random_state=42)
X2d = reducer.fit_transform(emb)

plt.scatter(X2d[:, 0], X2d[:, 1], c=labels)
for i, txt in enumerate(texts):
    plt.annotate(i, (X2d[i, 0], X2d[i, 1]))
plt.show()
```

Expected: login/password texts nên gần nhau; refund/billing texts nên gần nhau; Kubernetes texts nên gần nhau. Nếu không, embedding model có thể không phù hợp domain, dataset quá nhỏ/noisy, hoặc UMAP `n_neighbors` quá lớn cho n=10 (n_neighbors=3 hợp cho data ít).

**Extend lab milestones:**

1. Bỏ `normalize_embeddings=True`, chạy lại K-Means — đo silhouette với metric="euclidean" vs metric="cosine", chứng minh curse of dimensionality.
2. Bỏ giảm chiều UMAP và chạy K-Means trực tiếp trên embedding 384 chiều — so silhouette với pipeline có UMAP (5-10 chiều).
3. Thay K-Means bằng `HDBSCAN(min_cluster_size=2)` — đo số cluster và noise; so với K-Means cần chọn `k`.
4. Thay UMAP bằng `TSNE(perplexity=3)` — so thời gian và độ phân tách local neighborhood.
5. Inject 3 outlier text (random tokens / OCR garbage) — đo DBSCAN label `-1` của chúng, so với K-Means (force vào cluster).
6. Thay metric distance K-Means bằng cosine (`sklearn` không trực tiếp — dùng `sklearn.cluster.SpectralClustering(affinity="cosine")`) — đo khác biệt cluster shape.
7. Đo retrieval quality: query 5 câu, lấy top-3 retrieved, kiểm tra có cùng cluster với relevant doc không (`purity@3`).

## Empirical lab: scale, distance, k, reduce-dim

**Goal:** Lượng hóa tác động của normalization, distance metric, choice of `k`, và reduce-dim trước cluster lên silhouette và chất lượng cluster.

**Setup:** SentenceTransformer `all-MiniLM-L6-v2` (384 chiều), 100-500 support tickets 3-5 topic. K-Means/HDBSCAN. Mỗi trial chạy 3 lần lấy trung bình.

**Steps:**

1. Baseline: `normalize=True` + K-Means `k=5` + cosine silhouette (run trên Playbook).
2. Thay đổi lần lượt một biến: normalize, distance, `k`, giảm chiều, cluster algorithm.
3. Ghi silhouette, Davies-Bouldin, % noise, inter-cluster purity, failure mode.
4. Lặp 3 lần, lấy trung bình.

**Record:**

| Trial | Task | Tool | Latency | Cost | Quality | Failure mode |
|---|---|---|---|---|---|---|
| T1 | normalize + KMeans k=5 + cosine | sklearn KMeans | ~1s | ~$0 | sil 0.32 | ổn định |
| T2 | no normalize + KMeans k=5 + euclidean | sklearn KMeans | ~1s | ~$0 | sil 0.08 | curse of dims, centroid noise |
| T3 | normalize + KMeans k=3 + cosine | sklearn KMeans | ~1s | ~$0 | sil 0.40 | gộp topic, under-clustering |
| T4 | normalize + KMeans k=10 + cosine | sklearn KMeans | ~1s | ~$0 | sil 0.21 | chia nhỏ topic quá |

| T7 | t-SNE 2-dim + KMeans k=5 | sklearn t-SNE | ~10s | ~$0 | sil 0.18 | không hợp cho downstream (no transform) |

Pattern quan trọng: T1 vs T2 cho thấy normalize quay silhouette 4x; T6 vs T1 cho thấy HDBSCAN tự phát hiện noise (8%) thay vì ép vào cluster như K-Means. T7 cho thấy t-SNE không hợp cho downstream vì không có `transform` ổn định.

## Post-Mortem & Troubleshooting (SRE/Gỡ lỗi)

<Callout type="warning" title="Post-Mortem 1: silhouette rất thấp (< 0.1)">
**Triệu chứng:** K-Means silhouette 0.05-0.08 trên embedding 384+ chiều.

**Fix:** (a) kiểm đã L2 normalize embedding chưa — dùng `metric="cosine"` cho silhouette; (b) giảm chiều PCA/UMAP trước K-Means; (c) có thể data không có cluster rõ ràng — elbow mơ hồ cũng biểu hiện điều này; (d) thử HDBSCAN để xem có noise nhiều không.
</Callout>

<Callout type="danger" title="Post-Mortem 2: cluster đặt tên sai sau deploy">
**Triệu chứng:** Inspect 20 sample mỗi cluster thấy mỗi cluster pha nhiều topic, tên đặt trước sai.

**Fix:** Đừng đoán từ 5 sample. Sample ít nhất 20-50 điểm, xem median/centroid, group theo top-terms hoặc nearest neighbors. Validate bằng downstream task (retrieval purity), hoặc domain review. Sẵn sàng đổi tên/hợp cluster.
</Callout>

<Callout type="warning" title="Post-Mortem 3: outlier kéo centroid lệch">
**Triệu chứng:** Cluster A chứa chủ yếu outlier (vd OCR garbage), centroid nằm ở hướng phi topic.

**Fix:** K-Means ép mọi điểm vào cluster; outlier không có chỗ → hỏng cluster. Dùng DBSCAN/HDBSCAN để label `-1`; hoặc IsolationForest tiền-filter outlier trước khi cluster. Normalize embedding để outlier có norm tương tự.
</Callout>

<Callout type="danger" title="Post-Mortem 4: DBSCAN label toàn -1 (mọi điểm là noise)">
**Triệu chứng:** `labels == -1` chiếm 95+% sample.

**Fix:** `eps` quá nhỏ cho density. Dùng k-distance plot (sắp xếp khoảng cách nearest neighbor thứ `min_samples`) để chọn `eps` tại "elbow". Hoặc chuyển sang HDBSCAN, tự chọn `eps` theo hierarchy. Scale data trước khi DBSCAN nếu chưa.
</Callout>

<Callout type="warning" title="Post-Mortem 5: UMAP plot đổi mỗi lần chạy">
**Triệu chứng:** Cùng `random_state`, plot khác nhau khi `n_neighbors`/`perplexity` đổi, hoặc khi data thêm.

**Fix:** UMAP/t-SNE nhạy tham số. Plot chỉ là khám phá, không report stakeholder mà không có sample. Đặt `random_state` cho reproducibility. UMAP có `transform` cho data mới, t-SNE không.
</Callout>

**Checklist:**

1. Data đã scale/normalize đúng chưa?
2. Distance metric phù hợp không: Euclidean hay cosine?
3. Có outlier không — sử dụng DBSCAN/HDBSCAN/IsolationForest?
4. `k` được chọn bằng elbow + silhouette + domain không?
5. Visualization có bị over-interpreted không?
6. Cluster có đánh giá bằng downstream task hoặc human review không?
7. Nếu t-SNE/UMAP cho plot lạ, thử `n_neighbors`/`perplexity` khác, check data quality (null, duplicate)?

## Cost / Latency / Reliability Unit Economics

| Quyết định | Lợi ích | Chi phí/rủi ro |
|---|---|---|
| Normalize embedding (L2) | cosine hợp, distance có ý nghĩa semantic | cần áp dụng cho embedding mới lúc inference |
| StandardScaler trên tabular | Euclidean không bị feature lớn thống trị | mỗi feature phải numeric, không handle categorical |
| K-Means | đơn giản, nhanh, scale tốt | cluster hình cầu, k cần chọn, outlier hỏng |
| HDBSCAN | tự chọn eps, phát hiện noise, shape bất kỳ | chậm hơn K-Means với data rất lớn, nhạy min_cluster_size |
| DBSCAN | shape bất kỳ, outlier label -1 | khó chọn eps, kém với mật độ cluster khác nhau |
| PCA giảm chiều | nhanh, tuyến tính, giảm curse of dims | chỉ capture linear, outlier nhạy, variance ≠ signal |
| t-SNE visualize | đẹp local neighborhood | chậm O(n log n), global không đáng tin, không transform cho data mới |
| UMAP giảm chiều | nhanh hơn t-SNE, có transform cho data mới | có params (n_neighbors, min_dist), non-deterministic |
| UMAP→HDBSCAN (RAG flow) | cluster tốt trên high-dim, phát hiện noise | 2 bước, cần tune cả UMAP và HDBSCAN |
| IsolationForest anomaly | fast, scalable, robust | `contamination` phải ước, recall không measure nếu không label |
| Cosine distance (vs Euclidean) | hợp text embedding, bỏ qua magnitude | cần normalized vector, không hợp feature thô có scale ý nghĩa |
| Elbow + silhouette để chọn k | heuristic nhanh | mơ hồ, subjective — cần domain validate |
| Sample 20-50/cluster để đặt tên | tránh bẫy "5 sample đẹp" | tốn time, có thể vẫn lẫn topic — kết hợp downstream |

## Knowledge check

1. K-Means cần chọn hyperparameter chính nào? Constraint về cluster shape?
2. DBSCAN label `-1` nghĩa là gì? Ưu điểm so với K-Means?
3. PCA tối đa hóa đại lượng nào? Tìm component như thế nào?
4. t-SNE/UMAP chủ yếu dùng cho mục đích gì? Vì sao không dùng cho downstream classifier?
5. Vì sao scaling quan trọng cho clustering?
6. Cosine similarity thường hợp với loại data nào? Vì sao?
7. Cluster label có phải ground truth không? Làm sao validate cluster?
8. HDBSCAN khác DBSCAN ở điểm nào?
9. Vì sao K-Means trên embedding 1536 chiều thô được cảnh báo?

---

## Answer key

1. Số cluster `k`. Constraint: cluster dạng cầu (spherical), kích thước tương tự, dùng Euclidean.
2. Noise/outlier. Ưu điểm: cluster shape bất kỳ, phát hiện noise, không cần `k`.
3. Variance của projection `Xv`. Tìm component bằng SVD của centered data hoặc eigendecomposition của covariance matrix.
4. Visualization/exploration high-dimensional data. Không có transform ổn định cho data mới (đặc biệt t-SNE), không phải embedding học cho downstream.
5. Vì distance bị feature scale lớn thống trị. Nếu feature A variance 10000 và B variance 1, Euclidean chỉ nhìn vào A.
6. Text/document embeddings đã normalize hoặc semantic vector. Vì cosine đo hướng (orientation), hợp "độ tương đồng ngữ nghĩa" bỏ qua magnitude noise.
7. Không. Cluster là giả thuyết cấu trúc, cần kiểm tra bằng domain / human / downstream supervised.
8. HDBSCAN tự chọn `eps` theo hierarchy clusters of varying density, robust hơn DBSCAN với dense datasets.
9. Curse of dimensionality: khoảng cách "concentrate" — mọi điểm xa đều tương tự nhau. Cluster theo distance không còn ý nghĩa.

## Mini-project: RAG embedding audit

**Task:** Audit knowledge base nhỏ bằng embeddings và clustering để phát hiện data quality issues.

**Input:** 100-500 text chunks từ docs (5 topic khác nhau) hoặc tự tạo. Có thể parity với docs public (ML, AI safety, RAG...).

**Steps:**

1. Embed chunks bằng `sentence-transformers` (all-MiniLM-L6-v2 hoặc stronger).
2. Normalize embeddings (L2 norm).
3. Cluster bằng K-Means (`k` chọn bằng silhouette) hoặc HDBSCAN.
4. Visualize bằng UMAP (`n_neighbors` nhỏ cho data ít).
5. Inspect 10-20 chunks mỗi cluster — đặt tên topic.
6. Ghi nhận outliers, duplicates, topic gaps.
7. (Bonus) Đo retrieval quality: query 10 câu, xem top-3 retrieved có cùng cluster với relevant doc không.

**Deliverables:**

- Plot UMAP colored by cluster.
- Cluster summary table (cluster id, count, top terms, example).
- 5 vấn đề data quality phát hiện (duplicate chunks, OCR garbage, language mismatch, mixed topic, sparse cluster).
- Đề xuất cải thiện chunking/metadata/retrieval dựa trên audit.

**Success criteria:**

- Không over-interpret plot.
- Có sample-based cluster naming (20+ points mỗi cluster).
- Có nhận xét outlier/noise với label `-1`.
- Có liên hệ tới RAG retrieval quality (retrieved khác relevant/cluster, duplicate chunks cạnh tranh).
- Sử dụng cosine (vì embedding) không phải Euclidean thô.

<Callout type="tip" title="Thực tế production">
Embedding audit là hoạt động định kỳ ở các team RAG production. Khi docs update, embedding model upgrade, hoặc metrics retrieval decline, audit giúp localize "trục trặc" — chunks nào không đại diện đúng topic, query nào không match cluster. Đây là công việc AI engineer làm hàng tuần, không once-off.
</Callout>

## Điểm cần nhớ

- Unsupervised giúp khám phá data khi không label — hypothesis về cấu trúc, không ground truth.
- Distance metric quyết định cluster: Euclidean cho numeric scaled, cosine cho embedding normalized.
- K-Means tìm centroid và cần chọn `k`; tốt cho cluster dạng cầu, dễ bị outlier hỏng.
- DBSCAN/HDBSCAN dựa mật độ, phát hiện noise, không cần `k` nhưng nhạy `eps`/`min_samples`.
- PCA giảm chiều tuyến tính, giữ variance lớn nhất qua SVD/eigendecomposition.
- t-SNE/UMAP dùng để visualize; UMAP có `transform` cho data mới và giữ global structure tốt hơn t-SNE.
- Anomaly detection unsupervised: IsolationForest, DBSCAN -1, LOF, autoencoder reconstruction.
- Với embeddings/RAG, unsupervised tools giúp debug cluster, outlier, duplicate và topic coverage.
- Never trust cluster label/plot mà không sample thật ra xem — over-interpret là lỗi phổ biến nhất.
