Scikit-Learn là "xưởng thực hành" của Machine Learning classic. Nó không chỉ là thư viện model; nó là một **hệ sinh thái workflow**: preprocessing, pipeline, cross-validation, hyperparameter search, metrics, persistence, calibration, custom estimators. Nếu bạn biết dùng sklearn đúng, bạn sẽ tránh được rất nhiều lỗi production như leakage, preprocessing lệch train/test, model khó tái lập, và artifact thiếu thành phần. Pipeline giải quyết bằng cách đóng gói `raw input → preprocessing → model → prediction` thành một **estimator duy nhất** có `fit`/`predict`, dùng được trong CV, tuning, joblib artifact. Hiểu sklearn sâu là tiền đề cho PyTorch Lightning / HuggingFace Trainer / MLflow pipelines — đều mượn tư duy từ sklearn API. Trong vai trò giảng viên AI, tôi muốn bạn nắm vững `fit`/`transform`/`predict` mental model, Pipeline như "vaccine chống leakage", ColumnTransformer cho data hỗn hợp, GridSearchCV với cú pháp `step__param`, và joblib persistence với metadata sidecar.

Ví dụ spam classifier: `raw text → TF-IDF → Logistic Regression → prediction`. Nếu bạn chỉ lưu LogisticRegression mà quên TfidfVectorizer → production không biến text mới thành vector đúng. Fit vectorizer trên toàn data trước CV → leakage. Scale train và test bằng scaler khác nhau → prediction sai. Cột DataFrame khác thứ tự → model đọc nhầm feature. Pipeline giải quyết tất cả — đóng gói toàn workflow thành một object, fit trên train đúng cách, save/load bằng joblib, deploy chỉ một artifact.

## 1. The Incident (Tình huống sự cố)

<Callout type="danger" title="Báo cáo sự cố: Intent classifier ship thiếu TfidfVectorizer, category mới crash, column order lệch">
**Bối cảnh:** Team xây sklearn intent classifier cho 6 intent trên tabular-with-text data (cột `message` text, `country` categorical, `plan` categorical, `age`/`usage` numeric). Train trên Jupyter notebook, lưu `LogisticRegression` bằng joblib riêng (không lưu TfidfVectorizer, OneHotEncoder, StandardScaler). Production serving bằng FastAPI load artifact rồi nhận JSON cùng schema.

Không đóng gói preprocessing trong Pipeline; lưu mỗi `LogisticRegression`. Không dùng `handle_unknown="ignore"` cho OneHotEncoder. Convert DataFrame sang numpy rồi `model.predict(X)` — column order phụ thuộc DataFrame lúc inference. Không lưu metadata (sklearn version, data hash, threshold) cùng artifact. Threshold default 0.5 cho mọi intent.

1. Production nhận request với `country="VN"` (chưa thấy train, train chỉ có US/UK/DE) → OneHotEncoder mặc định raise `ValueError: unknown category` → 500 Internal Server Error cho 8% traffic.
2. Production nhận request JSON có key theo thứ tự khác (`plan` trước `country` trước `age`) → sau khi convert sang numpy, column thứ 2 thành `plan` thay vì `country` → prediction sai hoàn toàn nhưng không lỗi.
3. Token `refundr` (typo mới trong production) → TfidfVectorizer chưa từng học → vector toàn 0 → prediction về `general` mặc định. Vectorizer đã train trên train fold, không có `transform` ở production vì không được lưu cùng model.
4. Production chạy sklearn 1.4, artifact train bằng sklearn 1.1 → load joblib warn về `FutureWarning`, one coef có shape lệch → model dự đoán khác benchmark.
5. Threshold 0.5 cho intent `refund` (miss đắt) → recall chỉ 40%, trong khi intent `general` (ít rủi ro) chiếm phần lớn case — không ưu tiên đúng.

**Hậu quả:** Production intent classifier fail 8% (crash) + 15% (sai do column order) + 25% trường hợp `refund` bị bỏ sót. Mất 5 ngày trace: (a) lưu **toàn bộ Pipeline** bằng joblib (gồm `ColumnTransformer` + LogisticRegression); (b) `handle_unknown="ignore"` cho OneHot; (c) chỉ nhận DataFrame có column name, không convert sang numpy; (d) lưu metadata JSON sidecar (sklearn version, data hash, threshold); (e) tune threshold trên val rồi freeze trước deploy.
</Callout>

Bài học rút ra: production artifact phải là Pipeline (preprocessing + model), không chỉ model cuối. `handle_unknown="ignore"` cho categorical và OOV tokens cho text là bắt buộc production — distribution mới là chuyện thường, không phải exception. Column order phải khớp — stick với DataFrame và column names trong Pipeline, không convert sang numpy. Lưu metadata sidecar (sklearn version, data hash, threshold, metrics) để debug regression. Threshold tune trên validation, freeze, deploy cùng model.

## 2. Architecture Blueprint (sklearn Production Pipeline)

```text
                ┌──────────────────────────────────────────────────────────────┐
                │           SKLEARN PRODUCTION PIPELINE (anti-leakage)          │
                └──────────────────────────────────────────────────────────────┘

   raw DataFrame
   ┌──────┐    ┌───────────────────────┐   ┌──────────────┐    ┌──────────┐
   │age,  │───▶│   ColumnTransformer   │──▶│   Estimator  │───▶│ predict  │
   │usage,│    │  ┌────────┬─────────┐ │   │  (LogReg,RF, │    │ _proba   │
   │countr│    │  │ numeric │ categor │ │   │   XGBoost)  │    │  → tune  │
   │y,plan│    │  │ Scaler  │ OneHot │ │   └──────────────┘    │  thresh  │
   │,     │    │  │ Imputer │ ignore │ │                         │  on VAL  │
   │text  │    │  │         │        │ │                         └────┬─────┘
   │      │    │  │ + text  │        │ │                              │
   │      │    │  │ Tfidf   │        │ │                              ▼
   └──────┘    │  └────────┴─────────┘ │                       ┌─────────────┐
                └───────────────────────┘                       │  joblib.dump │
                       Pipeline.fit(X_train, y_train)          │  + metadata  │
                       Pipeline.transform / predict            │  JSON sidecar│
                                                                  └─────────────┘

   Rule: fit (và fit_transform) ONLY cho train; transform cho test. Pipeline =_estimator_ duy nhất.
```

Key insight: Pipeline là một estimator — có `fit`/`predict` — đóng gói cả workflow. CV và GridSearchCV chỉ cần truyền Pipeline vào, sẽ tự fit preprocessing trên mỗi train fold → không leakage. Artifact chỉ cần một joblib file để predict đúng.

## 3. Core intuition: sklearn API mental model

Ba loại object chính chia sẻ interface nhất quán.

### Estimator
Object có `.fit()` học từ data. Ví dụ `model.fit(X_train, y_train)` (supervised) và `scaler.fit(X_train)` (unsupervised, no y). Sau `fit`, state lưu trong attributes kết thúc `_` (vd `coef_`, `classes_`, `mean_`).

### Transformer
Học cách biến đổi rồi apply: `scaler.fit(X_train)`, `scaler.transform(X_train)`, `scaler.transform(X_test)`. Shortcut `fit_transform(X_train)`. **Quy tắc vàng**: `fit_transform` chỉ cho train, test luôn dùng `transform`.

### Predictor
Model đã fit có `predict`, `predict_proba` (phân phối xác suất), `score` (default metric).

### Meta-estimator
Gói estimator khác để thêm behavior: `GridSearchCV` (tune), `CalibratedClassifierCV` (calibrate), `Pipeline` (chuỗi transformers + estimator), `OneVsRestClassifier`/`MultiOutputClassifier` (multi-target).

<Callout type="warning" title="fit nghĩa là học từ data">
Bất cứ object có `fit` đều học thông tin từ data (mean/std, vocabulary, categories, weights, params). `fit` trên test là leakage/inconsistency. `fit_transform` chỉ cho train.
</Callout>

## 4. Preprocessing — transformers chính

### Numeric scaling

- `StandardScaler`: `(x - mean) / std` → zero mean, unit variance. Hợp logistic, SVM, KMeans, NN.
- `MinMaxScaler`: scale về [0, 1]. Hợp khi phân phối không Gaussian, outer bound cần ổn.
- `RobustScaler`: dùng median và IQR thay mean/std. **Robust với outlier**.
- Tree-based (RandomForest, XGBoost) **không cần scaling** — split threshold invariant với monotonic transform.

### Categorical encoding

- `OneHotEncoder`: mỗi category thành cột 0/1, hợp cardinality thấp. `handle_unknown="ignore"` **rất quan trọng production**: category mới → 0-vector, không crash.
- `OrdinalEncoder`: category → integer. Cẩn thận model diễn giải ordinal có thứ bậc. Chỉ hợp cho category có thứ tự thật (low/medium/high).
- `TargetEncoder` (sklearn 1.3+): encode bằng mean target. Hợp **cardinality cao** (zip, product ID). Cần CV trong Pipeline để tránh leakage.

### Text vectorization

```python
from sklearn.feature_extraction.text import TfidfVectorizer
vec = TfidfVectorizer(max_features=5000, ngram_range=(1, 2),
                      min_df=2, max_df=0.95, sublinear_tf=True)
```

- `max_features`: giới hạn vocab top-frequency — tránh OOM.
- `ngram_range=(1, 2)`: unigram + bigram — bắt phrase.
- `min_df=2`: bỏ term chỉ xuất hiện 1 doc (noise).
- `max_df=0.95`: bỏ term > 95% docs (stop-word corpus-specific).
- `sublinear_tf=True`: `log(1 + tf)` — giảm dominant của term phổ biến.

Vectorizer học **vocabulary** từ train; fit trên toàn data → leak.

### Imputation

- `SimpleImputer(strategy="median")` (numeric robust outlier), `mean` (Gaussian không outlier), `most_frequent` (categorical), `constant` với `fill_value` (giữ info "missingness" như feature signal).
- Advanced: `KNNImputer` (theo neighbor), `IterativeImputer` (MICE, round-robin regression; default `experimental` — phải `enable_iterative_imputer`).

## 5. Pipeline — vaccine chống leakage

Pipeline nối nhiều bước thành estimator duy nhất.

```python
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression

pipe = Pipeline([
    ("scaler", StandardScaler()),
    ("model",  LogisticRegression(max_iter=1000)),
])
```

### Pipeline.fit logic

1. `scaler.fit_transform(X)` → `X_scaled`.
2. `model.fit(X_scaled, y)`.

Ngoại trừ **bước cuối** (estimator) dùng `fit` thay `fit_transform`.

### Pipeline.predict logic

1. `scaler.transform(X)` → `X_scaled`.
2. `model.predict(X_scaled)` → `y_pred`.

Đảm bảo test đi qua cùng transformer đã học từ train — không thể leak.

<Callout type="tip" title="Pipeline là vaccine chống leakage">
Trong CV, Pipeline đảm bảo scaler/vectorizer fit lại chỉ trên train fold mỗi fold. Mọi workflow nghiêm túc nên dùng Pipeline — leakage preprocessing là bug khó debug vì metric sẽ "đẹp".
</Callout>

### Named steps và truy cập

```python
pipe.named_steps["scaler"]
pipe.named_steps["model"].coef_
pipe[:-1].transform(X_test)   # transformers only, cho visualization
```

### make_pipeline — tạo nhanh

`make_pipeline(StandardScaler(), LogisticRegression())` → steps tự tên `"standardscaler"`, `"logisticregression"`. Hợp nhanh, nhưng GridSearchCV cần tên rõ — `make_pipeline` thường không kiểm soát.

## 6. ColumnTransformer — preprocessing cho data hỗn hợp

Data thật có numeric (age, usage), categorical (country, plan, device), text (support message). ColumnTransformer áp transformer khác nhau cho từng cột, rồi concat.

```python
from sklearn.compose import ColumnTransformer
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler, OneHotEncoder
from sklearn.impute import SimpleImputer
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.linear_model import LogisticRegression

numeric = ["age", "usage"]
categorical = ["country", "plan"]
text = "message"

prep = ColumnTransformer([
    ("num", Pipeline([("imp", SimpleImputer(strategy="median")),
                      ("scaler", StandardScaler())]), numeric),
    ("cat", Pipeline([("imp", SimpleImputer(strategy="most_frequent")),
                      ("ohe", OneHotEncoder(handle_unknown="ignore"))]), categorical),
    ("text", TfidfVectorizer(max_features=5000, ngram_range=(1, 2)), text),
])

pipe = Pipeline([
    ("prep",  prep),
    ("model", LogisticRegression(max_iter=1000, class_weight="balanced")),
])
```

### Cú pháp cột

- `["col1", "col2"]` — list.
- `"col_name"` — single column (cho vectorizer 1D).
- `slice(0, 5)`.
- `(bool_mask,)`.
- `numerical` — selector helper.

```python
from sklearn.compose import make_column_selector as colsel

num_cols = colsel(dtype_include="number")
cat_cols = colsel(dtype_include="object")
```

Selector tự chọn theo dtype — robust khi production thêm cột mới.

### remainder — cột không matched

- `remainder="drop"` (default) — bỏ.
- `remainder="passthrough"` — giữ nguyên.
- `remainder=StandardScaler()` — apply transformer.

Default `drop` có thể làm mất feature.

Tên param dùng `step__param`. Pipeline lồng → `step__substep__param`.

## 11. Imbalanced-learn — mở rộng sklearn cho imbalance

`imbalanced-learn` thêm sampler đặt trong Pipeline (chỉ resample train fold, không val).

```python
from imblearn.pipeline import Pipeline as ImbPipeline
from imblearn.over_sampling import SMOTE
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression

pipe = ImbPipeline([
    ("scaler", StandardScaler()),
    ("smote",  SMOTE(random_state=42)),
    ("model",  LogisticRegression(max_iter=1000)),
])
```

**Quan trọng**: dùng `imblearn.pipeline.Pipeline`, không phải `sklearn.pipeline.Pipeline` — sampler chỉ resample sau khi fit và trước model, chỉ resample train fold trong CV. Sklearn Pipeline không biết sampler interface.

Sampling strategy:

- `SMOTE`: oversample minority bằng interpolation synthetic.
- `RandomUnderSampler`: undersample majority.
- `SMOTETomek`, `SMOTEENN`: kết hợp over + clean.
- So với `class_weight="balanced"`: sampling thay data, weight chỉ đổi loss. Test cả hai.

## 12. Model persistence — joblib

Lưu **toàn bộ pipeline** bằng joblib.

```python
import joblib

joblib.dump(gs.best_estimator_, "intent_classifier.joblib", compress=3)
loaded = joblib.load("intent_classifier.joblib")
loaded.predict(new_df)
```

### Vì sao lưu pipeline, không chỉ model

- LogisticRegression cần cùng feature order, same scaler, same vocabulary.
- Save riêng model → production phải reimplement preprocessing → bug-prone.
- Lưu Pipeline = deployment dùng cùng 1 object = consistent.

### Pickle/joblib caveats

- **Security**: joblib dùng pickle — load có thể execute code. Chỉ load artifact do pipeline của bạn tạo. Artifact nên đến từ CI/CD hoặc model registry đáng tin.
- **Versioning**: artifact phụ thuộc sklearn version, Python version. Load artifact sklearn 1.0 vào 1.4 có thể warning/error.
- **Size**: tree boosting/lightgbm có thể quá lớn cho pickle — dùng native format (`xgb.save_model`, `lgb.Booster.save_model`).

<Callout type="danger" title="Không load joblib từ nguồn không tin cậy">
joblib dùng pickle, có thể thực thi code khi load. Chỉ load artifact do pipeline của bạn tạo ra; artifact nên đến từ CI/CD hoặc model registry đáng tin.
</Callout>

### Artifact metadata — production readiness

Lưu kèm artifact metadata (JSON sidecar):

```json
{
  "model_id": "intent_v1",
  "sklearn_version": "1.4.0",
  "trained_at": "2025-07-01T10:00:00Z",
  "data_version": "data_hash_abc123",
  "metrics": {"f1": 0.82, "auc": 0.91},
  "features": ["age", "usage", "country", "plan", "message"],

```

Khi deploy check sklearn_version, data_version — mismatch cảnh báo regress.

## 13. Custom Estimator — mở rộng sklearn API

```python
from sklearn.base import BaseEstimator, TransformerMixin
import pandas as pd

class DateFeatureExtractor(BaseEstimator, TransformerMixin):
    def __init__(self, date_column="created_at"):
        self.date_column = date_column

    def fit(self, X, y=None):
        return self   # stateless

    def transform(self, X):
        X = X.copy()
        dt = pd.to_datetime(X[self.date_column])
        X["day_of_week"] = dt.dt.dayofweek
        X["hour"] = dt.dt.hour
        X["is_weekend"] = (dt.dt.dayofweek >= 5).astype(int)
        return X.drop(columns=[self.date_column])
```

Custom estimator phải có `fit` (trả `self`) và `transform`. Cẩn thận:

- `__init__` chỉ set params, không làm processing — sklearn clone bằng constructor args.
- `transform` phải return array/DataFrame có cùng số row.
- Tất cả learned state lưu trong attributes kết thúc `_` (sau `fit`).

## 14. Evaluation workflow sạch

Vòng đời sạch:

1. Tune hyperparameters trên train/val (CV hoặc hold-out).
2. Tune threshold trên val.
3. Freeze model + threshold.
4. Evaluate trên test **một lần** với cả hai.
5. Lưu artifact + metadata.

```python
proba_val  = pipe.predict_proba(X_val)[:, 1]
proba_test = pipe.predict_proba(X_test)[:, 1]

# 1) default threshold
y_pred_default = (proba_test >= 0.5).astype(int)
print(confusion_matrix(y_test, y_pred_default))

# 2) tune threshold trên VAL (chỉ mở test 1 lần cuối)
precisions, recalls, thresholds = precision_recall_curve(y_val, proba_val)
mask = recalls[:-1] >= 0.90
chosen = thresholds[mask].max() if mask.any() else 0.5

# 3) freeze rồi mới evaluate test
y_pred_tuned = (proba_test >= chosen).astype(int)
```

<Callout type="tip" title="Lưu threshold cùng model">
Trong production, model + threshold là cặp inseparable — lưu cùng nhau. Hoặc wrap thành custom estimator với threshold bên trong `predict`.
</Callout>

## 15. Common mistakes

<Callout type="warning" title="Mistake 1: preprocessing ngoài Pipeline">
Scale/vectorize trước CV → leakage. Đưa preprocessing vào Pipeline để sklearn fit đúng từng fold.
</Callout>

<Callout type="warning" title="Mistake 2: lưu model nhưng quên preprocessor">
Production cần cùng scaler/vectorizer/vocabulary. Lưu toàn bộ Pipeline bằng joblib.
</Callout>

<Callout type="danger" title="Mistake 3: category mới làm crash production">
OneHotEncoder mặc định có thể lỗi với category chưa thấy. Dùng `handle_unknown="ignore"`. TfidfVectorizer OOV token tự 0, ổn hơn.
</Callout>

<Callout type="warning" title="Mistake 4: column order lệch">
Train `["age", "country", "plan", "message", "usage"]` vs production `["country", "age", "plan", "usage", "message"]` — ColumnTransformer dùng tên cột thì OK; nếu convert sang numpy rồi truyền → sai cột. Stick với DataFrame + column names.
</Callout>

<Callout type="warning" title="Mistake 5: sampler đặt sai pipeline">
`imblearn` sampler phải dùng `imblearn.pipeline.Pipeline`, không sklearn Pipeline. Đặt SMOTE ngay trước model step (sau transformers) để chỉ resample train fold.
</Callout>

<Callout type="warning" title="Mistake 6: không lưu metadata">
Artifact mà không biết data/sklearn version, metric → không reproducible, không debug regression. Lưu sidecar JSON với metadata đầy đủ.
</Callout>

## Implementation Playbook: full sklearn workflow

Pipeline đầy đủ cho tabular + text với GridSearchCV, threshold tuning và joblib artifact. Chạy được, đo được, sửa được.

```python
from sklearn.model_selection import train_test_split, GridSearchCV

])

param_grid = {
    "prep__text__ngram_range":  [(1, 1), (1, 2)],
    "prep__text__max_features": [2000, 5000],
    "model__C":                 [0.1, 1, 10],
}

gs = GridSearchCV(pipe, param_grid, scoring="f1", cv=5, n_jobs=-1, refit=True)
gs.fit(X_train, y_train)
print("best:", gs.best_params_, gs.best_score_)

# Tune threshold trên VAL
proba_val  = gs.predict_proba(X_val)[:, 1]
precs, recs, ths = precision_recall_curve(y_val, proba_val)
mask = recs[:-1] >= 0.85
chosen = ths[mask].max() if mask.any() else 0.5

# Evaluate trên test đúng 1 LẦN
proba_test = gs.predict_proba(X_test)[:, 1]
y_pred = (proba_test >= chosen).astype(int)
print(confusion_matrix(y_test, y_pred))
print(classification_report(y_test, y_pred))
print("AUC-ROC:", roc_auc_score(y_test, proba_test))

# Lưu artifact + metadata
import sklearn
joblib.dump(gs.best_estimator_, "intent_classifier.joblib", compress=3)
metadata = {
    "model_id": "intent_v1",
    "sklearn_version": sklearn.__version__,
    "trained_at": datetime.now(timezone.utc).isoformat(),
    "threshold": float(chosen),
    "metric_auc": float(roc_auc_score(y_test, proba_test)),
    "features": list(X.columns),
}
with open("intent_classifier.meta.json", "w") as f:
    json.dump(metadata, f, indent=2)
```

## Empirical lab: Pipeline, scale, unknown, threshold

**Goal:** Lượng hóa tác động của Pipeline (vs preprocessing ngoài), `handle_unknown`, sampling imbalance, và threshold tuning lên recall và ổn định production.

**Setup:** tabular + text data, 500-2000 sample, 15% positive, 70/15/15 stratified. Mỗi trial chạy 3 lần lấy trung bình.

**Steps:**

1. Baseline: Pipeline + OneHot `handle_unknown="ignore"` + threshold 0.5 + LR `class_weight="balanced"` (run trên Playbook).
2. Thay đổi lần lượt một biến: leakage preprocessing, `handle_unknown`, column order, threshold, model, sampler, calibration.
3. Ghi recall class 1, F1, AUC-ROC, Brier, crash rate, failure mode.
4. Lặp 3 lần, lấy trung bình.

**Record:**

| Trial | Task | Tool | Latency | Cost | Quality | Failure mode |
|---|---|---|---|---|---|---|
| T1 | Pipeline + OneHot ignore + 0.5 + balanced | sklearn Pipeline | ~3s | ~$0 | recall 0.83, F1 0.62 | ổn định |
| T2 | preprocessing ngoài Pipeline (leak) | sklearn manual | ~2s | ~$0 | recall 0.86 (ảo) | metric optimist do leakage |
| T3 | OneHot default (no ignore) + new category | sklearn Pipeline | crash | ~$0 | n/a | 8% traffic 500 error |
| T4 | numpy thay DataFrame + column order lệch | sklearn Pipeline | ~3s | ~$0 | recall 0.55 | prediction sai vì cột lệch |
| T5 | threshold tuned recall≥0.85 trên val | sklearn Pipeline | ~3s | ~$0 | recall 0.88, F1 0.58 | precision giảm |
| T6 | RandomForest thay LR | sklearn Pipeline | ~10s | ~$0 | recall 0.85, F1 0.65 | artifact lớn hơn 10x |
| T7 | SMOTE trong imblearn Pipeline | imblearn Pipeline | ~5s | ~$0 | recall 0.86, F1 0.64 | tương đương class_weight |
| T8 | CalibratedClassifierCV isotonic + tuned | sklearn Pipeline | ~8s | ~$0 | recall 0.86, Brier 0.08 | probability tin cậy hơn |
| T9 | Optuna tuning 50 trial thay GridSearchCV 5^3 | optuna+sklearn | ~120s | ~$0 | recall 0.88, F1 0.66 | tune-aware, chậm hơn |

## Post-Mortem & Troubleshooting (SRE/Gỡ lỗi)

<Callout type="danger" title="Post-Mortem 1: OneHotEncoder crash với category mới">
**Triệu chứng:** Production `ValueError: Found unknown categories` cho category chưa thấy train.

**Fix:** Đặt `OneHotEncoder(handle_unknown="ignore", sparse_output=False)` (sklearn 1.2+) hoặc `handle_unknown="ignore"` (version cũ). Category mới → 0-vector, không crash. TfidfVectorizer token OOV tự 0, an toàn hơn.
</Callout>

<Callout type="warning" title="Post-Mortem 2: prediction lệch sau deploy">
**Triệu chứng:** Train metric F1 0.85, production F1 0.55 sâu.

**Fix:** Column order lệch — nếu convert DataFrame sang numpy thì cột thứ 2 ở train có thể khác thứ 2 ở production. Stick với DataFrame + column names trong Pipeline. Kiểm `X.columns` ở train vs `df.columns` ở inference khớp.
</Callout>

<Callout type="danger" title="Post-Mortem 3: load joblib FutureWarning / shape mismatch">
**Triệu chứng:** `FutureWarning`, `coef_` shape lệch, hoặc crash khi load artifact sklearn 1.0 vào 1.4.

**Fix:** Artifact phụ thuộc sklearn/Python version. Đặt version cố định trong CI; lưu `sklearn_version` trong metadata sidecar; mismatch → retrain trên version mới. Đừng load artifact từ source không tin cậy (joblib = pickle, có thể execute code).
</Callout>

<Callout type="warning" title="Post-Mortem 4: SMOTE leak validation">
**Triệu chứng:** CV metric cao bất thường sau khi thêm SMOTE.

**Fix:** Dùng `imblearn.pipeline.Pipeline`, không `sklearn.pipeline.Pipeline`. Sklearn Pipeline không biết sampler interface → SMOTE resample cả val fold → leakage. Sampler phải đặt sau transformer, trước model step.
</Callout>

<Callout type="warning" title="Post-Mortem 5: GridSearchCV chọn sai param vì nested Pipeline">
**Triệu chứng:** `GridSearchCV` không tác động lên hyperparam bạn muốn; hoặc param bị ignore.

**Fix:** Tên param phải đúng đường dẫn `step__substep__param`. vd `prep__text__ngram_range` (prep = ColumnTransformer, text = TfidfVectorizer). Inspect `pipe.get_params().keys()` để xem mọi param path khả dụng.
</Callout>

**Checklist:**

1. Preprocessing có trong Pipeline không?
2. Column names production giống train không (dùng ColumnTransformer với names, không numpy)?
3. Category/text OOV mới được handle không?
4. Text vectorizer lưu cùng model không?
5. GridSearchCV dùng `step__param` đúng, kể nested?
6. Artifact có version data/code/sklearn?
7. Sampling imbalance dùng `imblearn.Pipeline` (không sklearn)?
8. Threshold tune trên val và lưu cùng artifact?

## Cost / Latency / Reliability Unit Economics

| Quyết định | Lợi ích | Chi phí/rủi ro |
|---|---|---|
| Pipeline (vs preprocessing ngoài) | chống leakage, CV auto fit mỗi fold, artifact 1 object | cần đặt tên step cho GridSearchCV, init verbose hơn |
| ColumnTransformer | handling data hỗn hợp (num+cat+text) trong 1 object | cần biết cột nào thuộc type nào; `remainder` mặc định drop có thể mất feature |
| OneHotEncoder handle_unknown=ignore | category mới không crash production | mất signal — category mới thành 0-vector, hard to detect drift |
| TfidfVectorizer (vs Count) | giảm dominant của phổ biến term, robust hơn | cần tune `max_features, ngram, min_df, max_df` |
| StandardScaler | numeric ổn định cho logistic/SVM/KMeans | không robust outlier — RobustScaler hợp khi có outlier |
| RobustScaler | robust outlier (median + IQR) | ít familiar, không zero mean unit variance |
| TargetEncoder | cardinality cao (zip, product ID) | dễ leak — phải dùng CV trong Pipeline tránh leakage |
| class_weight="balanced" | tăng recall minority không cần gather data | có thể giảm precision, đẩy decision boundary |
| SMOTE (imblearn Pipeline) | synthetic minority, không cần gather data | có thể tạo sample phi thực tế, phải chặn leak val |
| joblib dump Pipeline | artifact 1 object, deploy consistent | pickle — không load từ nguồn không tin; deps sklearn version |
| metadata sidecar (JSON) | reproducible, debug regression, check version mismatch | cần duy trì có thể lệch nếu quên update |
| Calibration isotonic | probability đáng tin cho downstream | cần data đủ; calibrate trên val/CV riêng |
| GridSearchCV (Pipeline) | tune toàn bộ Pipeline với `step__param` | combinator explosion; nested Pipeline path dài |
| Optuna (Pipeline) | ít trial hơn random, prune trial kém | setup storage, vẫn hội tụ local |

## Knowledge check

1. `fit` khác `transform` thế nào? `fit_transform` nên dùng cho tập nào?
2. Vì sao Pipeline chống leakage?
3. ColumnTransformer dùng khi nào? `remainder="drop"` khác `passthrough` thế nào?
4. `model__C` nghĩa là gì trong GridSearchCV? Nested `prep__text__ngram_range` nghĩa gì?
5. Vì sao nên lưu toàn bộ pipeline, không chỉ model cuối?
6. `handle_unknown="ignore"` giải quyết gì? Khi nào cần?
7. `predict_proba` dùng để làm gì? Threshold tune ở đâu (val hay test)?
8. `RobustScaler` khác `StandardScaler` ở đâu? Khi nào chọn?
9. Sampler imbalance phải đặt trong Pipeline nào (sklearn hay imblearn)? Vì sao?
10. Custom estimator cần implement method nào? Vì sao `__init__` không nên làm processing?

## Answer key

1. `fit` học thông tin từ data; `transform` áp dụng biến đổi đã học. `fit_transform` chỉ cho train — test dùng `transform` riêng.
2. Preprocessing fit riêng trên train fold trong CV, không nhìn validation fold — không có chance để test info leak vào train.
3. Khi data có nhiều loại cột cần transformer khác nhau (numeric, categorical, text). `drop` bỏ cột không matched, `passthrough` giữ nguyên — thường `drop` an toàn hơn để không leak feature không định.
4. Parameter `C` của step tên `model`. Nested: `prep` (ColumnTransformer) → `text` (TfidfVectorizer) → `ngram_range`.
5. Vì production cần cùng preprocessing/vocabulary/scaler với training. Lưu model mà không có vectorizer/scaler → phải reimplement → bug-prone và khác biệt.
6. Category/token mới trong production không làm encoder crash — encode 0-vector (OneHot) hoặc bỏ qua (Tfidf). Hữu ích khi distribution drift, user mới, OOV tokens.
7. Lấy xác suất để tune threshold, ranking hoặc calibration. Tune trên **validation** — evaluation trên test threshold đã freeze.
8. RobustScaler dùng median và IQR thay mean/std — ít bị outlier kéo. Hợp khi data có outlier không phải noise thật.
9. `imblearn.pipeline.Pipeline`. Vì sampler cần re-fit chỉ trên train fold mỗi CV iteration — imblearn Pipeline xử lý điều này; sklearn Pipeline không biết sampler interface và sẽ leak nếu nhét sampler.
10. `fit` (trả `self`) và `transform`. `__init__` chỉ set parameters — sklearn clone object bằng `__init__` args, processing trong `__init__` phá clone.

## Điểm cần nhớ

- Sklearn API xoay quanh `fit`, `transform`, `predict`, `score` — nhất quán qua mọi estimator.
- `fit` học từ data; fit sai chỗ gây leakage. `fit_transform` chỉ cho train, `transform` cho test.
- Pipeline đóng gói preprocessing + model thành một estimator — best practice trong mọi CV/tuning.
- ColumnTransformer xử lý numeric/categorical/text trong cùng pipeline, có `remainder` và selector helper.
- FunctionTransformer và custom estimator cho phép mở rộng sklearn API mà vẫn giữ trong Pipeline.
- GridSearchCV tune pipeline bằng `step__param` cú pháp nested.
- Sampler imbalance phải dùng `imblearn.pipeline.Pipeline` (không sklearn), đặt trước model step.
- Lưu toàn bộ Pipeline bằng joblib, không chỉ estimator cuối. Lưu metadata (data/code/sklearn version, threshold) sidecar.
- `handle_unknown="ignore"` quan trọng cho categorical production; vectorizer OOV tokens ổn định.
- Threshold tuning trên validation; evaluation trên test một lần cuối — lưu threshold cùng model.
