Linear Regression và Logistic Regression là hai mô hình supervised nền tảng nhất. Tôi muốn bạn xem chúng không chỉ là "model cũ", mà là **phòng thí nghiệm nhỏ** để hiểu toàn bộ ML: loss function, gradient, decision boundary, probability, regularization, metrics và calibration. Trong thực tế, nhiều hệ thống AI production vẫn bắt đầu bằng baseline rất đơn giản: **TF-IDF + Logistic Regression** cho spam/intent/sentiment, hoặc **Linear/Ridge Regression** cho cost/latency/usage forecasting. Nếu baseline đơn giản đã đủ tốt, bạn tiết kiệm được rất nhiều tiền và complexity so với fine-tune transformer. Nếu bạn hiểu linear/logistic sâu, bạn sẽ hiểu neural network nhanh hơn — vì một linear layer + sigmoid chính là một neuron; deep learning chỉ là xếp nhiều neuron như vậy lại và train bằng gradient descent. Trong vai trò giảng viên AI, tôi muốn bạn nắm vững forward pass, gradient của MSE vs BCE, vì sao MSE nhạy outlier mà cross-entropy phạt confident-but-wrong, decision boundary tuyến tính, và L1/L2 regularization — nền của toàn bộ chapter sau.

Giả sử bạn xây hệ thống multi-LLM router. Bạn muốn dự đoán **chi phí request** dựa trên số input tokens và output tokens — đây là regression: `cost ≈ w1 * input_tokens + w2 * output_tokens + b`. Bạn cũng muốn dự đoán request có phải "complex reasoning" không để route sang GPT-4/Claude thay vì model rẻ — đây là classification: `P(complex | request_features)`. Linear/logistic regression là baseline tuyệt vời vì dễ hiểu và giải thích được (coefficients cho biết feature nào quan trọng, theo hướng nào), train rất nhanh (fit trên hàng triệu sample vài giây), ít overfit khi data vừa phải, và là điểm kiểm tra trước khi dùng model phức tạp hơn. Trong NLP classification, trước khi fine-tune BERT, hãy chạy TF-IDF + Logistic Regression — nếu baseline đạt F1 92% và BERT đạt 93% nhưng tốn gấp 20 lần compute, production có thể chọn baseline. Baseline cũng là thước đo: nếu transformer không thắng baseline, có lẽ vấn đề ở data chứ không phải model.

## 1. The Incident (Tình huống sự cố)

<Callout type="danger" title="Báo cáo sự cố: Cost-prediction regression không hội tụ, weight «đứng im» sau 1000 epoch">
**Bối cảnh:** Một team xây regression dự đoán chi phí request API bằng gradient descent tay (NumPy). Hai feature: `input_tokens` (giá trị 100-5000) và `prompt_quality_score` (giá trị 0.0-1.0). Dùng MSE loss, `lr=0.001`, init `w=[0,0]`, không normalize, không scale, train 1000 epoch.

Không normalize feature trước khi gradient descent. Không kiểm magnitude gradient từng feature. Không vẽ loss curve theo epoch. Dùng threshold `loss < 1e-4` để dừng, mặc dù loss ban đầu đã thuộc bậc `1e6`. Trust `loss` cuối mà không inspect weight cuối.

1. `input_tokens` có scale ~10^3, `prompt_quality_score` có scale ~10^-1 → gradient theo `input_tokens` lớn hơn gradient theo `prompt_quality` khoảng 10^4 lần với cùng `lr`.
2. Gradient lớn theo `input_tokens` làm weight nhảy mạnh mỗi step → loss dao động, đôi khi tăng; gradient nhỏ theo `prompt_quality` gần không đổi → weight `w_prompt` về 0.001 và đứng im.
3. Vì các feature scale lệch nhau, một bước `lr` đủ nhỏ để `input_tokens` ổn định lại thì `prompt_quality` không học được gì; một bước `lr` đủ lớn để `prompt_quality` học thì `input_tokens` nhảy/bùng nổ.
4. Sau 1000 epoch, MSE giảm từ `1.5e6` xuống `3e5` rồi đứng im — team tưởng hội tụ thật, ship — thực ra model mới chỉ học được intercept và `w_input`, còn `w_prompt` hoàn toàn bị khóa.
5. Deploy bound: prediction sai 40-60% với request có `prompt_quality` cao/low khác biệt — feature quan trọng nhất bị bỏ.

**Hậu quả:** Prediction nhảy thất thường, resource quota sai cho 15% traffic. Mất 2 ngày debug trước khi phát hiện: (a) không normalize feature, (b) không log gradient magnitude từng feature, (c) dùng loss cuối duy nhất để dừng. Phải retrain với `StandardScaler` (hoặc chia mỗi feature cho std), `lr` đặt lại, và theo dõi `||grad_w||` riêng từng feature.
</Callout>

Bài học rút ra: gradient descent cần feature có scale tương đương — nếu feature A lớn gấp 10^4 feature B, optimizer không thể chọn một `lr` làm việc cho cả hai. Luôn `StandardScaler` (hoặc thủ công chia cho std) trước khi gradient descent trên linear/logistic/SVM/KMeans. Khi inspect training, log cả `||grad||` từng parameter chứ không chỉ `loss` — "loss giảm" không có nghĩa mọi weight đang học. Nếu nghi ngờ, normalize trước rồi mới giải tiếp.

## 2. Architecture Blueprint (Linear / Logistic Forward + Backward)

```text
                ┌──────────────────────────────────────────────────────────────┐
                │        LINEAR / LOGISTIC FORWARD + GRADIENT UPDATE            │
                └──────────────────────────────────────────────────────────────┘

   Input X          Linear score z          Activation             Loss
   (n,d)            (n,)                    (n,)                   (scalar)
   ┌─────┐         ┌─────────────┐          ┌─────────┐            ┌─────┐
   │  X  │ ──────▶│  z = Xw + b │ ───────▶│  none   │ ──(linear)▶│ MSE │
   └─────┘         └──────┬──────┘          │ sigmoid │ ──(logist)▶│ BCE │
                          │                 └────┬────┘            └──┬──┘
                          │                      │                    │
                          ▼                      ▼                    ▼
                    ┌───────────────────────────────────────────────────┐
                    │  BACKWARD (chain rule)                            │
                    │  linear : ∂MSE/∂w = (2/n) X^T (ŷ - y)             │
                    │          ∂MSE/∂b = (2/n) Σ(ŷ - y)                 │
                    │  logist : ∂BCE/∂w = (p - y) · x       (per sample) │
                    │  update :  w ← w - lr · grad_w                    │
                    └───────────────────────────────────────────────────┘
```

Key insight: linear và logistic chỉ khác nhau ở "đuôi" — regression dùng `ŷ = Xw+b` trực tiếp, classification thêm `p = sigmoid(z)` rồi loss đổi từ MSE sang BCE. Gradient của BCE đẹp bất thường: `(p - y) * x`, hình dáng tương tự MSE nhưng prediction đã đi qua sigmoid.

## 3. Core intuition: model tuyến tính là "cân trọng số feature"

Linear model giả định prediction là tổng có trọng số của feature:

```text
prediction = feature_1 * weight_1 + feature_2 * weight_2 + ... + bias
```

Ví dụ dự đoán cost LLM:

```text
cost = 0.000001 * input_tokens + 0.000002 * output_tokens + 0.0001
```

Nếu `weight_output_tokens` lớn hơn `weight_input_tokens`, output token đắt hơn input token. Như vậy model không chỉ dự đoán, nó còn **giải thích** — đây là ưu thế so với neural network đen. Linear model là tuyến tính **theo weight** chứ không nhất thiết tuyến tính theo feature. Bạn có thể thêm feature phi tuyến như `x^2`, `log(x)`, `x1 * x2`, và model vẫn tuyến tính theo weight — vẫn giải được bằng linear regression.

## 4. Formal definition

### Linear regression

```text
ŷ = Xw + b
```

| Ký hiệu | Ý nghĩa | Shape |
|---|---|---|
| `X` | feature matrix | `(n_samples, n_features)` |
| `w` | weights | `(n_features,)` |
| `b` | bias (intercept) | scalar |
| `ŷ` | predictions | `(n_samples,)` |
| `y` | true values | `(n_samples,)` |

### Logistic regression

Vẫn bắt đầu bằng linear score `z = Xw + b`, sau đó đưa qua sigmoid để ép output vào `(0, 1)`:

```text
z = Xw + b
p = sigmoid(z) = 1 / (1 + exp(-z))
```

`p` được diễn giải như xác suất class positive. `logit(p) = log(p / (1 - p)) = z` — sigmoid là inverse của logit. `z` âm lớn → `p → 0`; `z = 0` → `p = 0.5`; `z` dương lớn → `p → 1`.

## 5. Loss functions

### Mean Squared Error (MSE) — linear regression

```text
MSE = (1/n) * Σ (y_i - ŷ_i)^2
```

Vì lỗi được bình phương, lỗi lớn bị phạt mạnh hơn lỗi nhỏ. Sai 1 → phạt 1, sai 3 → phạt 9.

Gradient của MSE:

```text
grad_w = (2/n) * X^T (ŷ - y)
grad_b = (2/n) * Σ(ŷ - y)
```

Gradient luôn trỏ theo hướng tăng loss, nên ta **trừ** gradient để giảm loss: `w ← w - lr * grad_w`, `b ← b - lr * grad_b`.

### Mean Absolute Error (MAE)

```text
MAE = (1/n) * Σ |y_i - ŷ_i|
```

MAE phạt lỗi tuyến tính, không bình phương — ít nhạy với outlier hơn MSE. Tuy nhiên gradient không xác định tại `error = 0`, khiến optimizer khó hơn (sub-gradient).

### Huber — kết hợp cả hai

```text
Huber(error) = {
  0.5 * error^2               nếu |error| <= δ
  δ * (|error| - 0.5 * δ)      nếu |error| > δ
}
```

Huber dùng MSE cho lỗi nhỏ, MAE cho lỗi lớn; nhạy vừa với outlier, gradient xác định khắp nơi — thường dùng trong deep learning và robust regression.

<Callout type="info" title="Chọn loss theo business cost">
MSE: phạt outlier mạnh, smooth gradient — hợp cho đại đa số trường hợp. MAE: phạt outlier ít, nhưng optimizer khó. Huber: cân bằng hai phía. Trong production, chọn loss theo cost function thực — ví dụ dự đoán cost thì phạt sai lớn sẽ tốn tiền hơn sai nhỏ.
</Callout>

### Binary Cross-Entropy (BCE) — logistic regression

```text
BCE = -[y * log(p) + (1 - y) * log(1 - p)]
```

- Nếu `y = 1`: `BCE = -log(p)`. `p` gần 1 → loss gần 0; `p` gần 0 → loss đi tới ∞.
- Nếu `y = 0`: `BCE = -log(1 - p)`. `p` gần 0 → loss gần 0.

#### Vì sao cross-entropy phạt confident-but-wrong

`p = 0.9` cho `y = 1`: `-log(0.9) = 0.105`. `p = 0.01` cho `y = 1`: `-log(0.01) = 4.605`. MSE với `p = 0.01` và `y = 1` chỉ `(1 - 0.01)^2 = 0.98` — phạt yếu hơn nhiều. Cross-entropy phạt **tự tin mà sai** theo hàm log, rất mạnh, rất hợp lý cho classification.

<Callout type="warning" title="Tự tin mà sai bị phạt nặng">
Cross-entropy phạt rất mạnh prediction tự tin nhưng sai. Đây là điều bạn muốn: model không chỉ đúng, mà phải calibrated — tự tin khi có lý do. BCE là default cho binary classification trong và ngoài deep learning.
</Callout>

#### Gradient của BCE

```text
∂BCE/∂w = (p - y) * x
∂BCE/∂b = (p - y)
```

Đẹp đơn giản. Gradient = `(prediction - target) * feature`. Tương tự MSE về hình dáng, nhưng prediction đã đi qua sigmoid.

## 6. Closed form (Normal Equation) vs Gradient Descent

### Normal equation

Linear regression có nghiệm đóng:

```text
w = (X^T X)^(-1) X^T y
```

Nếu `X^T X` khả nghịch, đây là global optimum — không cần iteration.

```python
import numpy as np

X = np.array([[1, 100], [1, 200], [1, 300]], dtype=float)  # thêm cột bias = 1
y = np.array([1.2, 1.9, 3.3])

w = np.linalg.pinv(X.T @ X) @ X.T @ y
print(w)
```

Ưu điểm: nhanh với data nhỏ, không cần tune learning rate, đến global optimum một bước. Nhược điểm: `X^T X` có shape `(d, d)` — inverse tốn `O(d^3)`; với d = 10^4 feature, không khả thi. `X^T X` có thể gần singular (multicollinearity) → inverse không ổn định. Không generalize cho model non-convex (neural network).

### Gradient descent

```python
import numpy as np

X = np.array([[100], [200], [300]], dtype=float)
y = np.array([1.2, 1.9, 3.3])

Xn = X / 100.0                       # normalize để gradient descent dễ hơn
w = np.array([0.0]); b = 0.0; lr = 0.05

for step in range(1000):
    y_pred = Xn @ w + b
    error = y_pred - y
    grad_w = (2 / len(y)) * Xn.T @ error
    grad_b = (2 / len(y)) * error.sum()
    w -= lr * grad_w
    b -= lr * grad_b

print("w:", w, "b:", b)
```

Ưu điểm: scale tốt với feature dimension lớn (mini-batch/SGD), generalize cho non-convex model (NN), memory-efficient với streaming data. Nhược điểm: cần tune learning rate, cần normalize feature (nếu không gradient bị dominate bởi feature lớn), có thể kẹt ở plateau hoặc diverge nếu lr quá lớn.

<Callout type="info" title="Vì sao scale feature?">
Nếu tokens là 100, 200, 300 còn feature khác là 0.1, 0.2, gradient theo tokens sẽ lớn hơn nhiều. Scaling giúp optimizer đi ổn định hơn, đặc biệt với linear/logistic/SVM/KMeans. `StandardScaler` chia cho std là lựa chọn an toàn.
</Callout>

### When to use which

| Yếu tố | Normal equation | Gradient descent |
|---|---|---|
| Số feature `d` nhỏ (&lt; 10^4) | Hợp | Cũng hợp |
| Số feature `d` lớn (embedding 1536, BOW 10^5) | Không khả thi | Hợp |
| Data rất nhiều sample | Có thể tốn memory | SGD stream được |
| Multicollinearity | Cần regularization (Ridge) | Natural fit |
| Generalize cho NN | Không | Có |

Trong sklearn, `LinearRegression` dùng least squares (có thể qua SVD); `SGDRegressor` dùng gradient descent — hợp cho data lớn.

## 7. Decision boundary (logistic)

Mặc định `predict positive nếu p >= 0.5`. Vì sigmoid `p = 0.5` khi `z = 0`, decision boundary là:

```text
Xw + b = 0
```

Với 2 feature, đây là một **đường thẳng**. Với nhiều feature, là một **siêu phẳng** (hyperplane). Đó là vì sao logistic regression là *linear classifier* — boundary tuyến tính theo feature.

<Callout type="info" title="Logistic vẫn tuyến tính — boundary thẳng">
Dù output `p` đi qua sigmoid phi tuyến, decision boundary tại `p=0.5` là đường thẳng. Để có boundary cong, cần polynomial features hoặc kernel trick, hoặc dùng model phi tuyến (tree, neural net).
</Callout>

## 8. Worked example by hand: dự đoán API cost

Giả sử model `cost_pred = 0.01 * tokens`.

| Request | Tokens | True cost |
|---|---:|---:|
| A | 100 | 1.2 |
| B | 200 | 1.9 |
| C | 300 | 3.3 |

Prediction:

```text
A: 0.01 * 100 = 1.0
B: 0.01 * 200 = 2.0
C: 0.01 * 300 = 3.0
```

Errors: `A: 0.2, B: -0.1, C: 0.3`. Squared errors: `0.04, 0.01, 0.09`.

```text
MSE = (0.04 + 0.01 + 0.09) / 3 = 0.0467
MAE = (0.2 + 0.1 + 0.3) / 3 = 0.2
```

Nếu một prediction sai 3.0 thay vì 0.3, squared error là 9.0 — MSE tăng rất mạnh, MAE chỉ tăng tuyến tính. Đó là lý do MSE nhạy với outlier.

### Tính gradient

```text
grad_w = (2/3) * Σ (ŷ_i - y_i) * x_i
       = (2/3) * [(-0.2)(100) + (0.1)(200) + (-0.3)(300)]
       = (2/3) * [-20 + 20 - 90]
       = (2/3) * (-90) = -60
```

Cập nhật `w` với `lr = 0.0001`:

```text
w ← 0.01 - 0.0001 * (-60) = 0.01 + 0.006 = 0.016
```

Model đã tăng weight (vì predictions đang lệch thấp trung bình), đi đúng hướng.

## 9. Polynomial and interaction features

Linear model là tuyến tính theo weight nhưng không nhất thiết theo feature. Bạn có thể thêm feature phi tuyến:

```python
from sklearn.preprocessing import PolynomialFeatures
from sklearn.linear_model import LinearRegression
from sklearn.pipeline import Pipeline

pipe = Pipeline([
    ("poly", PolynomialFeatures(degree=2, interaction_only=False, include_bias=False)),
    ("model", LinearRegression()),
])
```

`degree=2` cho feature `x1, x2, x1^2, x2^2, x1*x2`. Model vẫn tuyến tính theo weight — vẫn dùng LinearRegression.

<Callout type="warning" title="Polynomial quá cao = overfit guaranteed">
`degree=4` với 5 feature sinh ra 70+ feature. Với dataset nhỏ, model sẽ fit noise. Trong deep learning, neural network tự học feature phi tuyến; polynomial thường chỉ hợp khi bạn có domain knowledge về quan hệ cụ thể.
</Callout>

## 10. Regularization

Regularization thêm penalty vào loss để chống weight quá lớn, giảm overfitting.

### L2 / Ridge

```text
loss_total = loss + λ * Σ w_j^2
```

Tác dụng: weight nhỏ, model mượt hơn, ít nhạy với noise. Default trong `LogisticRegression` của sklearn.

### L1 / Lasso

```text
loss_total = loss + λ * Σ |w_j|
```

Tác dụng: nhiều weight về đúng **0** — feature selection tự nhiên, model sparse. Hợp khi bạn nghi nhiều feature không liên quan và muốn biết cái nào thật sự quan trọng.

### ElasticNet — kết hợp L1 + L2

```text
loss_total = loss + λ1 * Σ |w_j| + λ2 * Σ w_j^2
```

Kết hợp ưu điểm: sparsity của L1 + stability của L2. Khi data có feature tương quan cao, ElasticNet ổn định hơn L1.

### `C` trong sklearn — nghịch đảo strength

Trong sklearn, `LogisticRegression(penalty="l2", C=1.0)`:

```text
C = 1 / λ

C nhỏ  → regularization mạnh (λ lớn, weight bị phạt nặng)
C lớn  → regularization yếu  (λ nhỏ, weight ít bị phạt)
C → ∞  → gần như không regularization (overfit dễ)
```

<Callout type="danger" title="C trong sklearn rất dễ hiểu ngược">
Nhiều người nghĩ `C` lớn nghĩa là regularization mạnh. Sai. Với sklearn logistic regression, `C = 1 / λ`. Vì vậy `C=0.01` regularize mạnh hơn `C=10`. Đây là nguồn bug phổ biến khi copy pattern từ framework khác (PyTorch dùng `weight_decay` trực tiếp).
</Callout>

```python
from sklearn.linear_model import LogisticRegression

model_strong = LogisticRegression(C=0.01, max_iter=1000)  # regularize mạnh → weight nhỏ, underfit risk
model_weak   = LogisticRegression(C=100,   max_iter=1000)  # regularize yếu → weight tự do, overfit risk
```

### Ridge regression cho regressor

```python
from sklearn.linear_model import Ridge, Lasso, ElasticNet

ridge   = Ridge(alpha=1.0)                       # L2
lasso   = Lasso(alpha=0.01)                      # L1
elastic = ElasticNet(alpha=0.01, l1_ratio=0.5)  # kết hợp
```

`alpha` trong regressor của sklearn **trùng diện** với `C` logistic theo nghĩa: `alpha` lớn = regularization mạnh. Lại một cú dễ nhầm — để ý.

## 11. Multi-class classification

Với K > 2 class, có hai chiến lược chính:

### One-vs-Rest (OvR)

Train một classifier binary cho mỗi class:

```text
class A vs not A
class B vs not B
class C vs not C
```

Predict = class có score cao nhất. Đơn giản, nhưng class imbalance giữa "class A" và "rest" nặng nếu K lớn.

### Multinomial / Softmax

Model output score `z_k` cho mỗi class, softmax thành probability:

```text
softmax(z)_k = exp(z_k) / Σ_j exp(z_j)
```

Loss là Categorical Cross-Entropy: `CE = -Σ_k y_k * log(p_k)`.

```python
clf = LogisticRegression(multi_class="multinomial", max_iter=1000)
```

Softmax thường tốt hơn OvR khi class loại trừ nhau (mỗi sample chỉ thuộc 1 class). Khi class có quan hệ thứ bậc (hierarchical), dùng cấu trúc đặc biệt hơn.

<Callout type="info" title="Lưu ý multi_class trong sklearn mới">
Từ sklearn 1.5+, `multi_class` sẽ deprecated cho solver 'lbfgs' — multinomial là default. Trong deep learning, multi-class dùng softmax trực tiếp, không cần OvR.
</Callout>

## 12. Calibration với logistic regression

Logistic regression thường đã khá calibrated (vì BCE optimize likelihood trực tiếp). Khi dùng L1 heavy regularization hoặc khi feature bị correlated, calibration có thể lệch.

```python
from sklearn.calibration import CalibratedClassifierCV

calibrated = CalibratedClassifierCV(pipe, cv=3, method="isotonic")
calibrated.fit(X_train, y_train)
```

Trong product rủi ro cao (y tế, tài chính), hãy vẽ **reliability diagram** trên validation để check.

## 13. Common mistakes

<Callout type="warning" title="Mistake 1: không scale numeric feature">
Linear/logistic regression nhạy feature scale. Nếu feature có scale rất khác nhau, optimizer khó hội tụ và regularization phạt feature không công bằng — feature lớn bị phạt ít hơn feature nhỏ (vì cần weight nhỏ hơn để có cùng impact).
</Callout>

<Callout type="warning" title="Mistake 2: dùng predict thay vì predict_proba khi cần threshold">
`predict()` dùng threshold mặc định 0.5. Production thường cần threshold theo business cost. Hãy lấy `predict_proba()` rồi chọn threshold trên validation.
</Callout>

<Callout type="warning" title="Mistake 3: multicollinearity phá coefficients">
Nếu 2 feature tương quan cao (r=0.95), coefficients có thể dao động lớn với ít data change — model vẫn predict đúng nhưng diễn giải sai. Dùng VIF (Variance Inflation Factor) hoặc L2 regularization.
</Callout>

<Callout type="danger" title="Mistake 4: diễn giải coefficient khi scale khác nhau">
Nếu feature A có scale 0-1 và B có scale 0-1000, coefficient của A có thể lớn hơn B nhiều dù B quan trọng hơn. Hãy scale trước, hoặc diễn giải coefficient * std(feature) = "impact của 1 std change".
</Callout>

## Implementation Playbook: TF-IDF + Logistic Regression spam classifier

Baseline NLP production cực mạnh, thường khó đánh bại trừ khi dataset lớn và ngữ liệu phong phú. Chạy được, đo được, sửa được.

```python
from sklearn.model_selection import train_test_split
from sklearn.pipeline import Pipeline
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import (
    classification_report, confusion_matrix,
    roc_auc_score, precision_recall_curve,
)
import numpy as np

texts = [
    "win money now", "free prize claim now", "meeting tomorrow at ten",
    "please review the invoice", "limited offer win cash", "team sync notes attached",
]
labels = [1, 1, 0, 0, 1, 0]  # demo nhỏ — production cần hundreds+

X_train, X_test, y_train, y_test = train_test_split(
    texts, labels, test_size=0.33, random_state=42, stratify=labels,
)

pipe = Pipeline([
    ("tfidf", TfidfVectorizer(ngram_range=(1, 2))),
    ("clf", LogisticRegression(max_iter=1000)),
])

pipe.fit(X_train, y_train)
y_pred = pipe.predict(X_test)
proba  = pipe.predict_proba(X_test)[:, 1]

print(confusion_matrix(y_test, y_pred))
print(classification_report(y_test, y_pred))
print("AUC-ROC:", roc_auc_score(y_test, proba))

# Tune threshold trên validation khi có dữ liệu thật
precisions, recalls, thresholds = precision_recall_curve(y_test, proba)
mask = recalls[:-1] >= 0.85
chosen = thresholds[mask].max() if mask.any() else 0.5
y_pred_tuned = (proba >= chosen).astype(int)
```

Trong dataset thật, bạn cần hàng trăm/hàng nghìn mẫu. Nhưng structure pipeline này là production pattern: vectorizer + classifier trong cùng pipeline để **tránh leakage** trong cross-validation.

**Extend lab milestones:**

1. Thay `ngram_range=(1,1)` vs `(1,2)` vs `(1,3)` — đo F1, AUC-ROC, vocab size.
2. Thay `C` của LogisticRegression từ `1e-3` → `1e-2 → 1 → 10 → 100` (log scale) — đo train vs val gap để phát hiện overfit/underfit.
3. Thay `penalty="l1"` (Lasso) và inspect số coefficient khác 0 (feature selection tự nhiên).
4. Thay `class_weight="balanced"` khi dataset imbalance — đo recall vs precision trade-off.
5. Thay Gradient Descent tay (`SGDClassifier(loss="log_loss")`) thay vì `LogisticRegression` — đo tốc độ và accuracy với data lớn.
6. Thêm `PolynomialFeatures(degree=2, interaction_only=True)` cho tabular feature trước LR — đo cải thiện accuracy và số feature tăng.
7. Thêm `CalibratedClassifierCV(method="isotonic")` — đo Brier score và vẽ reliability diagram.

## Empirical lab: scale, loss, regularization, threshold

**Goal:** Lượng hóa tác động của feature scaling, choice of loss, regularization strength và threshold tuning lên accuracy, recall và độ ổn định của probability.

**Setup:** `make_classification` 5000 sample, 20 feature có scale khác nhau (một số 0-1, một số 0-1000), 15% positive. Train/val/test 70/15/15. Mỗi trial chạy 3 lần lấy trung bình.

**Steps:**

1. Baseline: Logistic Regression, threshold 0.5, không scale, `C=1` (run trên Playbook).
2. Thay đổi lần lượt một biến: scale, loss, regularization, threshold.
3. Ghi accuracy, recall class 1, F1, AUC-ROC, Brier, failure mode.
4. Lặp 3 lần, lấy trung bình.

**Record:**

| Trial | Task | Tool | Latency | Cost | Quality | Failure mode |
|---|---|---|---|---|---|---|
| T1 | LR + no scale + threshold 0.5 + C=1 | sklearn LR | ~1s | ~$0 | acc 0.86, recall 0.55 | LR phải retry vì không hội tụ, weight chậm |
| T2 | LR + StandardScaler + 0.5 + C=1 | sklearn Pipeline | ~1s | ~$0 | acc 0.90, recall 0.70 | ổn định, hội tụ nhanh |
| T3 | MinMaxScaler thay vì Standard | sklearn Pipeline | ~1s | ~$0 | acc 0.90, recall 0.69 | tương đương, hợp distribution phi Gaussian |
| T4 | LR + scale + C=0.001 (reg mạnh) | sklearn Pipeline | ~1s | ~$0 | acc 0.84, recall 0.45 | underfit — weight quá nhỏ |
| T5 | LR + scale + C=100 (reg yếu) | sklearn Pipeline | ~1s | ~$0 | acc 0.89, recall 0.66 | overfit nhẹ, train-val gap lớn |
| T6 | LR + scale + L1 penalty | sklearn Pipeline | ~1s | ~$0 | acc 0.89, sparse coef | feature selection: 12/20 coef = 0 |
| T7 | LR + scale + threshold tuned recall≥0.85 | sklearn Pipeline | ~1s | ~$0 | recall 0.87, prec 0.50 | precision giảm |
| T8 | SGDClassifier log_loss thay LR | sklearn SGD | ~0.5s | ~$0 | acc 0.90 | nhạy lr, cần scaling bắt buộc |
| T9 | ElasticNet (L1+L2), L1_ratio=0.5 | sklearn Pipeline | ~1s | ~$0 | acc 0.89 | cân bằng L1/L2 khi feature correlated |
| T10 | CalibratedClassifierCV isotonic + tuned | sklearn Pipeline | ~3s | ~$0 | recall 0.87, Brier 0.09 | probability tin cậy hơn |

Ghi chú: số liệu trên là ước lượng điển hình — bạn cần chạy và ghi lại con số thực tế. Pattern quan trọng: T1 (không scale) trông "vẫn chạy" nhưng recall thấp hơn T2 đáng kể và có warning hội tụ; T4/T5 cho thấy chọn `C` sai (đảo nghĩa) dễ làm model underfit/overfit mà không biết.

## Post-Mortem & Troubleshooting (SRE/Gỡ lỗi)

<Callout type="warning" title="Post-Mortem 1: ConvergenceWarning / không hội tụ">
**Triệu chứng:** `ConvergenceWarning: lbfgs failed to converge` hoặc loss nhảy không giảm sau nhiều epoch.

**Fix:** (a) tăng `max_iter` (vd 1000 → 5000); (b) scale feature (`StandardScaler`); (c) kiểm multicollinearity (VIF), bỏ feature tương quan cao; (d) giảm regularization (`C` nhỏ hơn hoặc `alpha` lớn hơn) nếu underfit; (e) thử solver khác (`saga` cho L1 với data lớn).
</Callout>

<Callout type="danger" title="Post-Mortem 2: weight đứng im, một feature không học">
**Triệu chứng:** Gradient theo một weight luôn ≈ 0, weight đó không đổi qua epoch.

**Fix:** Feature đó có scale lệch quá lớn so với feature khác — gradient bị dominate bởi feature lớn scale. Normalize/scale tất cả feature trước khi gradient descent. Inspect `||grad_w_j||` riêng từng feature, không chỉ loss tổng.
</Callout>

<Callout type="warning" title="Post-Mortem 3: train score cao, val score thấp">
**Triệu chứng:** Train F1 0.92 nhưng val F1 0.65.

**Fix:** Overfit. Giảm complexity: tăng regularization (`C` nhỏ hơn), bỏ feature correlated, thêm data, dùng `Pipeline` + cross-val. Nếu đã L2 mà vẫn overfit, nghi data leak giữa train và val (cùng user, preprocessing fit toàn data).
</Callout>

<Callout type="danger" title="Post-Mortem 4: coefficient đảo dấu so với kỳ vọng">
**Triệu chứng:** Coefficient của feature "rõ ràng quan trọng" lại âm hoặc gần 0.

**Fix:** (a) multicollinearity — feature tương quan với feature khác làm coefficient dao động; (b) feature chưa scale — coefficient so sánh sai scale; (c) đảo: nếu feature A correlated với target nhưng bạn đã có feature B tương tự, sklearn phân trọng số giữa chúng tùy chọn, có thể cho một âm. Dùng VIF, hoặc L1 để tự bỏ.
</Callout>

<Callout type="warning" title="Post-Mortem 5: probability không calibrated">
**Triệu chứng:** `predict_proba=0.9` mà thực tế chỉ đúng 65%.

**Fix:** Vẽ reliability diagram trên val, đo Brier. Nếu lệch, dùng `CalibratedClassifierCV(pipe, method="isotonic")` (data > 1000 sample) hoặc `"sigmoid"`. Calibrate trên val/CV riêng, không calibrate trên test.
</Callout>

**Checklist:**

1. Feature numeric đã scale chưa?
2. Pipeline có bao gồm preprocessing không (để CV không leakage)?
3. Có class imbalance cần `class_weight`/threshold tuning không?
4. Threshold có được tune trên validation không?
5. Coefficients có hợp lý không (dấu, magnitude sau khi scale)?
6. Regularization `C`/`alpha` có quá mạnh/yếu không (check train vs val gap)?
7. Có multicollinearity làm coefficients lệch không?
8. Probability có cần calibration không (reliability diagram)?

## Cost / Latency / Reliability Unit Economics

| Quyết định | Lợi ích | Chi phí/rủi ro |
|---|---|---|
| StandardScaler trước LR | optimizer ổn định, hội tụ nhanh, coefficient giải thích được | cần handle NaN/inf trước khi scale, scaler phải fit trên train fold |
| MSE (linear regression) | smooth gradient, dễ optimize, phổ biến | nhạy outlier — sai lớn bị bình phương phạt rất mạnh |
| MAE | ít nhạy outlier, phạt tuyến tính | gradient không xác định tại 0, optimizer khó (sub-gradient) |
| Huber | cân bằng MSE/MAE, robust | cần chọn `δ`, thêm 1 hyperparameter |
| BCE (logistic) | phạt confident-but-wrong mạnh, gradient đẹp `(p-y)x` | không hợp cho regression — phải đảm bảo target 0/1 |
| L2 regularization | weight nhỏ, model mượt, ổn định | không sparse, các feature correlated đều bị phạt đều |
| L1 regularization | sparse — feature selection tự nhiên | có thể chọn sai feature khi correlated, không stable |
| ElasticNet (L1+L2) | kết hợp chọn feature + ổn định | thêm `l1_ratio` cần tuning |
| Polynomial features degree=2 | linear model học quan hệ phi tuyến, dễ giải thích | số feature tăng factorial, overfit với degree cao |
| Normal equation | global optimum 1 bước, không tune lr | `O(d^3)`, không scale, không generalize cho NN |
| SGD với mini-batch | scale với data lớn, streaming được | cần tune lr, scaling bắt buộc, có thể kẹt plateau |
| CalibratedClassifierCV | probability đáng tin cho downstream (risk, ranking) | cần data đủ, có thể overfit trên val nhỏ; chọn sigmoid/isotonic theo size |

## Knowledge check

1. Linear regression dùng cho loại target nào? Loss mặc định?
2. Logistic regression output là gì? Vì sao không dùng MSE?

4. MSE nhạy với gì? MAE khác MSE như thế nào?
5. L1 regularization có tác dụng đặc biệt gì? L2 khác gì?
6. Trong sklearn LogisticRegression, `C` nhỏ nghĩa là gì? Khác `alpha` của Ridge thế nào?
7. Vì sao dùng Pipeline với TF-IDF + Logistic Regression?
8. OvR khác Multinomial (softmax) thế nào? Khi nào chọn cái nào?
9. Closed-form (normal equation) không hợp cho trường hợp nào?
10. Polynomial features degree=2 cho 3 feature sinh ra bao nhiêu feature?

## Answer key

1. Target liên tục. Loss mặc định MSE.
2. Output là xác suất class positive sau sigmoid. MSE phạt confident-but-wrong yếu hơn BCE — cho gradient tệ khi p gần 0/1 vì sigmoid gradient nhỏ.
3. `z = 0`, tương ứng `p = 0.5`.
4. MSE nhạy outlier (bình phương lỗi). MAE phạt tuyến tính, ít nhạy outlier nhưng gradient không xác định tại error=0.
5. L1 tạo sparsity (nhiều weight = 0, feature selection). L2 làm weight nhỏ (cân bằng), không sparse.
6. `C` nhỏ = regularization mạnh. `alpha` của Ridge: `alpha` lớn = regularization mạnh — ngược nghĩa với `C`.
7. Để vectorizer chỉ fit trên train trong mỗi split/CV, tránh leakage và đóng gói preprocessing + model thành một estimator duy nhất.
8. OvR train K classifier binary; Multinomial train một classifier softmax. Multinomial tốt khi class loại trừ nhau và balance; OvR đơn giản, hợp khi có class hierarchy.
9. Không hợp khi `d` lớn (inverse `X^T X` tốn `O(d^3)`), multicollinearity (matrix gần singular), hoặc cần generalize cho NN.
10. 3 feature: `x1, x2, x3, x1^2, x2^2, x3^2, x1x2, x1x3, x2x3` = 9 feature.

Bài tập tính toán:

```text
MSE = ((2-2.5)^2 + (4-3.5)^2 + (6-7)^2) / 3 = (0.25 + 0.25 + 1) / 3 = 0.5
MAE = (|2-2.5| + |4-3.5| + |6-7|) / 3 = (0.5 + 0.5 + 1) / 3 = 0.667

BCE(y=1, p=0.8)  = -log(0.8)  ≈ 0.223
BCE(y=1, p=0.05) = -log(0.05) ≈ 2.996

MSE tương ứng:
MSE(y=1, p=0.8)  = (1-0.8)^2 = 0.04
MSE(y=1, p=0.05) = (1-0.05)^2 = 0.9025 ← phạt yếu hơn BCE rất nhiều
```

## Điểm cần nhớ

- Linear regression học target liên tục bằng `Xw+b`, loss thường là MSE (nhạy outlier; MAE/Huber ít nhạy hơn).
- Có nghiệm đóng (normal equation) cho data nhỏ; gradient descent cho data lớn và generalize cho NN.
- Logistic regression = linear score + sigmoid, output probability. Boundary là siêu phẳng `Xw+b=0`.
- Cross-entropy phạt prediction tự tin nhưng sai rất mạnh (theo log); gradient đẹp: `(p-y) * x`.
- Regularization chống overfit; L1 tạo sparsity (feature selection), L2 làm weight nhỏ (ổn định); ElasticNet kết hợp cả hai.
- `C` trong sklearn logistic là nghịch đảo regularization — đừng hiểu ngược.
- Polynomial/interaction features cho phép linear model biểu diễn phi tuyến (vẫn linear theo weight).
- Multi-class dùng OvR (đơn giản) hoặc Multinomial/softmax (mạnh khi class loại trừ nhau).
- TF-IDF + Logistic Regression là baseline NLP production cực mạnh, thường khó đánh bại trừ khi dataset lớn và ngữ liệu phong phú.
- Diễn giải coefficients cần scale hoặc represent "impact of 1 std change" để không bị lừa bởi scale khác nhau.
