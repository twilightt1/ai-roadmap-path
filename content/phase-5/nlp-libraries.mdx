NLP production không chỉ là thuật toán. Bạn cần biết thư viện nào dùng cho việc gì, tradeoff ra sao, và cách ghép chúng thành pipeline có thể chạy ổn định. Chương này map các tool chính: NLTK, spaCy, Hugging Face Transformers/Datasets, sentence-transformers, tiktoken, evaluate — cùng cách chọn tool đúng việc. Trong vai trò giảng viên AI, tôi muốn bạn hiểu tool đúng giúp giảm complexity, không phải dùng LLM lớn cho mọi việc.

## 1. The Incident (Tình huống sự cố)

<Callout type="danger" title="Báo cáo sự cố: dùng LLM lớn cho tokenization đơn giản, cold start 30s, 5,000 user timeout">
**Bối cảnh:** Một team build RAG ingestion. Họ dùng LLM API cho sentence segmentation, NER PII masking, và token counting. Họ không có baseline, không có fallback, không log latency.

**Sự cố:** Mỗi request load model trong request handler, gọi LLM cho tokenization, không có cache. Cold start 30s, p95 latency 45s. 5,000 user timeout. Hóa đơn LLM API $2,000/tháng cho tokenization đơn giản.

1. Dùng LLM cho tokenization đơn giản — quá nặng, tốn cost.
2. Load model trong request handler — cold start 30s.
3. Không có cache — mỗi request load model lại.
4. Không có fallback — LLM API lỗi, toàn pipeline fail.
5. Không log latency — không biết pipeline chậm.

**Hậu quả:** 5,000 user timeout. Họ phải tắt chatbot, thay LLM bằng spaCy cho preprocessing, tiktoken cho token counting, và cache model. Hóa đơn incident response 2 tuần.
</Callout>

Bài học rút ra: chọn tool đúng layer, không ép một tool làm mọi thứ. spaCy/tiktoken cho preprocessing, LLM cho reasoning. Production cần pin versions, manage model loading, batching, monitoring.

## 2. Architecture Blueprint (NLP Pipeline)

Pipeline NLP production thường gồm nhiều component:

```text
[ Data loading ]     → Datasets / pandas
        │
        ▼
[ Preprocessing ]    → regex / spaCy / custom cleaners
        │
        ▼
[ Token counting ]   → tiktoken / HF tokenizer
        │
        ▼
[ Model inference ]  → Transformers / API SDK / vLLM
        │
        ▼
[ Embeddings ]        → sentence-transformers / provider API
        │
        ▼
[ Evaluation ]        → sklearn / evaluate / custom rubric
        │
        ▼
[ Serving ]           → FastAPI / batch jobs / queues
```

Key insight: mục tiêu là chọn tool đúng layer, không ép một tool làm mọi thứ.

## 3. Core intuition: libraries are pipeline components

Một AI Engineer thường phải làm nhiều việc khác nhau trong cùng một project:

```text
clean text → tokenize → train/evaluate model → generate embeddings → retrieve → evaluate → deploy
```

Không có một thư viện duy nhất tốt nhất cho tất cả. Ví dụ:

- NLTK tốt để học classic NLP.
- spaCy tốt cho preprocessing/NER production nhẹ.
- Hugging Face Transformers tốt để dùng/fine-tune pretrained models.
- sentence-transformers tốt cho embeddings/RAG prototype.
- tiktoken tốt để đếm token OpenAI models.
- evaluate tốt để chuẩn hóa metrics.

<Callout type="tip" title="Tool đúng giúp giảm complexity">
Nhiều hệ thống lỗi không phải vì model yếu, mà vì chọn sai tool: dùng LLM lớn cho tokenization đơn giản, tự viết NER rules khó maintain, hoặc không đếm token trước khi gọi API.
</Callout>

## 6. spaCy

spaCy hướng production: nhanh, pipeline rõ, model nhỏ dễ deploy, hỗ trợ NER/POS/dependency parsing/sentence segmentation.

```python
import spacy

nlp = spacy.load("en_core_web_sm")
doc = nlp("OpenAI is based in San Francisco.")

for ent in doc.ents:
    print(ent.text, ent.label_)
```

Sentence segmentation:

```python
doc = nlp("Dr. Smith works at OpenAI. He builds NLP systems.")
for sent in doc.sents:
    print(sent.text)
```

Rule-based matching:

```python
import spacy
from spacy.matcher import Matcher

nlp = spacy.load("en_core_web_sm")
matcher = Matcher(nlp.vocab)

pattern = [{"LOWER": "refund"}, {"LOWER": {"IN": ["policy", "request"]}}]
matcher.add("REFUND_PATTERN", [pattern])

doc = nlp("Where is the refund policy?")
for match_id, start, end in matcher(doc):
    print(nlp.vocab.strings[match_id], doc[start:end].text)
```

<Callout type="tip" title="spaCy rất tốt cho preprocessing trước RAG">
Nếu cần sentence segmentation, NER để mask PII, hoặc rules tốc độ cao trước chunking/embedding, spaCy thường nhẹ và ổn định hơn gọi LLM.
</Callout>

Ưu điểm: production-friendly, nhanh, pipeline components rõ ràng, rule-based + statistical models kết hợp tốt, dễ package model vào Docker nếu chuẩn bị đúng.

Nhược điểm: model nhỏ không hiểu sâu như transformer lớn, multilingual/domain support phụ thuộc model, fine-tuning cần học workflow riêng.

## 7. Hugging Face Transformers

Transformers là thư viện chuẩn để dùng pretrained transformer models.

### Pipeline API

```python
from transformers import pipeline

clf = pipeline("sentiment-analysis")
print(clf("I love this roadmap!"))
```

Các pipeline phổ biến: `sentiment-analysis`, `text-classification`, `token-classification`, `question-answering`, `summarization`, `translation`, `text-generation`, `feature-extraction`.

### AutoTokenizer and AutoModel

```python
from transformers import AutoTokenizer, AutoModelForSequenceClassification

checkpoint = "distilbert-base-uncased"
tok = AutoTokenizer.from_pretrained(checkpoint)
model = AutoModelForSequenceClassification.from_pretrained(checkpoint)

batch = tok(
    ["I love this roadmap!", "This is confusing."],
    padding=True,
    truncation=True,
    return_tensors="pt",
)

out = model(**batch)
print(out.logits.shape)
```

### Generation

```python
from transformers import AutoTokenizer, AutoModelForCausalLM

checkpoint = "gpt2"
tok = AutoTokenizer.from_pretrained(checkpoint)
model = AutoModelForCausalLM.from_pretrained(checkpoint)

inputs = tok("The future of AI is", return_tensors="pt")
ids = model.generate(
    **inputs,
    max_new_tokens=30,
    do_sample=True,
    temperature=0.8,
    top_p=0.9,
)

print(tok.decode(ids[0], skip_special_tokens=True))
```

<Callout type="warning" title="Pipeline tiện nhưng không phải luôn production-ready">
`pipeline()` rất tốt để prototype. Production thường cần control batching, device placement, token limits, error handling, model loading lifecycle và monitoring.
</Callout>

Tokenize dataset:

```python
from transformers import AutoTokenizer
from datasets import load_dataset

checkpoint = "distilbert-base-uncased"
tok = AutoTokenizer.from_pretrained(checkpoint)
ds = load_dataset("imdb")


def tokenize(batch):
    return tok(batch["text"], truncation=True)


ds_tok = ds.map(tokenize, batched=True)
print(ds_tok["train"][0].keys())
```

## 9. sentence-transformers

sentence-transformers giúp tạo sentence/chunk/document embeddings nhanh.

Dùng cho: semantic search, clustering, deduplication, RAG prototype, similarity scoring, reranking với CrossEncoder variants.

```python
from sentence_transformers import SentenceTransformer
import numpy as np

texts = [
    "Refunds are available within 30 days.",
    "Reset your password from account settings.",
    "Kubernetes pods can restart after crashes.",
]
query = "How can I get my money back?"

model = SentenceTransformer("all-MiniLM-L6-v2")
doc_emb = model.encode(texts, normalize_embeddings=True)
query_emb = model.encode([query], normalize_embeddings=True)[0]

scores = doc_emb @ query_emb
for idx in np.argsort(scores)[::-1]:
    print(float(scores[idx]), texts[idx])
```

<Callout type="warning" title="Chọn embedding model theo ngôn ngữ/domain">
`all-MiniLM-L6-v2` tốt để học/prototype tiếng Anh, nhưng không phải luôn tốt cho tiếng Việt, code, legal, medical hoặc domain nội bộ. Hãy benchmark với eval set của bạn.
</Callout>

Cross-encoder reranking:

```python
from sentence_transformers import CrossEncoder

pairs = [
    ("How can I get my money back?", "Refunds are available within 30 days."),
    ("How can I get my money back?", "Reset your password in settings."),
]

reranker = CrossEncoder("cross-encoder/ms-marco-MiniLM-L-6-v2")
print(reranker.predict(pairs))
```

Bi-encoder nhanh cho retrieval. Cross-encoder chậm hơn nhưng chính xác hơn để rerank top-k.

## 10. tiktoken

`tiktoken` là tokenizer nhanh cho OpenAI-style models, thường dùng để đếm token/cost/context budget.

```python
import tiktoken

enc = tiktoken.encoding_for_model("gpt-4o-mini")
prompt = "Tôi đang học NLP và tokenization 🚀"

ids = enc.encode(prompt)
print(len(ids))
print(enc.decode(ids))
```

Function reusable:

```python
import tiktoken


def count_tokens(text: str, model_name: str = "gpt-4o-mini") -> int:
    try:
        enc = tiktoken.encoding_for_model(model_name)
    except KeyError:
        enc = tiktoken.get_encoding("cl100k_base")
    return len(enc.encode(text))
```

Use cases: tránh vượt context window, estimate cost, trim RAG context, log token usage, compare chunk sizes.

<Callout type="danger" title="Không dùng tiktoken cho mọi provider một cách tuyệt đối">
Claude, Gemini, Hugging Face models có tokenizer khác. `tiktoken` tốt cho OpenAI-compatible counting, nhưng multi-provider apps cần token counter/provider-specific estimates.
</Callout>

## 14. Example pipeline: RAG ingestion

```text
PDF/HTML text
→ custom cleaning/regex
→ spaCy sentence segmentation
→ token-aware chunking with tiktoken/HF tokenizer
→ sentence-transformers embeddings
→ vector DB
→ evaluate retrieval recall@k
```

```python
import re
import spacy
import tiktoken
from sentence_transformers import SentenceTransformer

nlp = spacy.load("en_core_web_sm")
enc = tiktoken.encoding_for_model("gpt-4o-mini")
embedder = SentenceTransformer("all-MiniLM-L6-v2")


def clean(text):
    text = re.sub(r"\s+", " ", text)
    return text.strip()


def sentences(text):
    return [sent.text for sent in nlp(text).sents]


def token_len(text):
    return len(enc.encode(text))


def chunk_sentences(sents, max_tokens=300):
    chunks = []
    current = []
    current_tokens = 0

    for sent in sents:
        n = token_len(sent)
        if current and current_tokens + n > max_tokens:
            chunks.append(" ".join(current))
            current = []
            current_tokens = 0
        current.append(sent)
        current_tokens += n

    if current:
        chunks.append(" ".join(current))
    return chunks


raw = "Refunds are available within 30 days. Contact support for details."
text = clean(raw)
chunks = chunk_sentences(sentences(text))
embeddings = embedder.encode(chunks, normalize_embeddings=True)

print(chunks)
print(embeddings.shape)
```

## 16. Production considerations

Khi đưa NLP libraries vào production, kiểm tra:

- model loading time,
- CPU/GPU memory,
- batching,
- concurrency,
- Docker image size,
- model/cache download at build time vs runtime,
- deterministic versions,
- monitoring latency/error rate,
- fallback khi model/service lỗi,
- data privacy và logging.

Pin versions:

```text
transformers==x.y.z
torch==x.y.z
spacy==x.y.z
sentence-transformers==x.y.z
```

Không nên để production tự tải model từ internet ở runtime nếu môi trường cần ổn định.

<Callout type="danger" title="Model artifacts là dependency production">
Tokenizer/model weights/spaCy models cũng là dependency. Version mismatch giữa tokenizer và model có thể gây bug khó debug.
</Callout>

## 17. Tokenization pitfalls

<Callout type="warning" title="Mistake 1: bỏ qua baseline scikit-learn">
Nhiều bài toán classification/search có thể có baseline TF-IDF rất mạnh. Nếu không baseline, bạn không biết transformer/LLM có thật sự đáng cost không.
</Callout>

<Callout type="warning" title="Mistake 2: load model trong mỗi request">
Load transformer/spaCy model trong request handler sẽ làm latency cực tệ. Load một lần khi app start, tái sử dụng object model.
</Callout>

<Callout type="danger" title="Mistake 3: tokenizer/model mismatch">
Dùng tokenizer khác checkpoint hoặc khác provider để count/chunk có thể làm truncation, cost và model input sai. Luôn gắn tokenizer với model.
</Callout>

## Implementation Playbook: NLP Toolkit Pipeline

```python
import re
import spacy
import tiktoken
from sentence_transformers import SentenceTransformer
from sklearn.pipeline import Pipeline
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.linear_model import LogisticRegression

nlp = spacy.load("en_core_web_sm")
enc = tiktoken.encoding_for_model("gpt-4o-mini")
embedder = SentenceTransformer("all-MiniLM-L6-v2")


def clean(text):
    text = re.sub(r"\s+", " ", text)
    return text.strip()


def sentences(text):
    return [sent.text for sent in nlp(text).sents]


def token_len(text):
    return len(enc.encode(text))


def chunk_sentences(sents, max_tokens=300):
    chunks = []
    current = []
    current_tokens = 0
    for sent in sents:
        n = token_len(sent)
        if current and current_tokens + n > max_tokens:
            chunks.append(" ".join(current))
            current = []
            current_tokens = 0
        current.append(sent)
        current_tokens += n
    if current:
        chunks.append(" ".join(current))
    return chunks


def build_classifier(train_texts, train_labels):
    pipe = Pipeline([
        ("tfidf", TfidfVectorizer(ngram_range=(1, 2), min_df=2)),
        ("clf", LogisticRegression(max_iter=1000)),
    ])
    pipe.fit(train_texts, train_labels)
    return pipe


def semantic_search(query, docs, top_k=5):
    doc_emb = embedder.encode(docs, normalize_embeddings=True)
    q_emb = embedder.encode([query], normalize_embeddings=True)[0]
    scores = doc_emb @ q_emb
    idx = np.argsort(scores)[::-1][:top_k]
    return [(docs[i], float(scores[i])) for i in idx]
```

## Post-Mortem & Troubleshooting (SRE/Gỡ lỗi)

<Callout type="warning" title="Pitfall 1: bỏ qua baseline scikit-learn">
**Triệu chứng:** Nhiều bài toán classification/search có thể có baseline TF-IDF rất mạnh. Nếu không baseline, bạn không biết transformer/LLM có thật sự đáng cost không.

**Fix:** Luôn có baseline TF-IDF + Logistic Regression. So sánh quality, cost, latency với model phức tạp hơn.
</Callout>

<Callout type="warning" title="Pitfall 2: load model trong mỗi request">
**Triệu chứng:** Load transformer/spaCy model trong request handler sẽ làm latency cực tệ. Cold start 30s, p95 latency 45s.

**Fix:** Load một lần khi app start, tái sử dụng object model. Dùng model cache. Pin versions.
</Callout>

<Callout type="danger" title="Pitfall 3: tokenizer/model mismatch">
**Triệu chứng:** Dùng tokenizer khác checkpoint hoặc khác provider để count/chunk có thể làm truncation, cost và model input sai. Model hiểu sai.

**Fix:** Luôn gắn tokenizer với model. Dùng tokenizer đúng provider. Log token count.
</Callout>
