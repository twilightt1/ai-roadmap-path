Secure AI API Integration là nơi AI Engineering gặp security engineering. Với LLM apps, bạn không chỉ bảo vệ API key; bạn bảo vệ prompts, retrieved documents, embeddings, tool permissions, logs, user data và cả chi phí vận hành. Một hệ thống AI không có threat model rõ ràng sẽ bị prompt injection, data leakage, quota abuse hoặc tool misuse sớm hay muộn.

---

## 1. The Incident (Tình huống sự cố)

<Callout type="danger" title="Báo cáo sự cố: API key bị leak trong frontend, 5,000 user bị duplicate refund, 50,000 USD/tháng wasted">
**Bối cảnh:** Một team build chatbot RAG. Họ dùng OpenAI, frontend React gọi trực tiếp provider API bằng key trong browser. Họ không có backend proxy, không có rate limit, không có PII redaction, không có audit logging.

**Sự cố:** API key bị lộ trong frontend. Attacker dùng key để tạo bill 50,000 USD/tháng. Họ không enforce per-user quota, không filter documents theo permission, không audit tool calls. Dữ liệu nhạy cảm đi ra ngoài không kiểm soát.

1. Không có threat model — không biết assets, actors, trust boundaries.
2. Không có backend proxy — API key expose trong frontend.
3. Không có rate limit — attacker dùng key tạo bill.
4. Không có PII redaction — user data nhạy cảm đi ra provider.
5. Không có audit logging — không biết tool call nào cho ai.

**Hậu quả:** API key bị leak. 50,000 USD/tháng wasted. Họ phải tắt chatbot, rotate API key, audit log, và thiết kế lại toàn bộ security. Hóa đơn incident response 2 tuần.
</Callout>

Bài học rút ra: security boundary phải nằm trước model. Đừng đưa dữ liệu/secret/action không được phép vào prompt rồi hy vọng model không tiết lộ hoặc không dùng. Nếu model thấy dữ liệu, bạn đã mất một lớp bảo vệ quan trọng. Cần threat model, secrets lifecycle, backend proxy, secure RAG authorization, PII redaction, tool permission, audit logging.

## 2. Architecture Blueprint (Security Boundary Before Model)

Một request LLM production phải đi qua nhiều trust boundaries:

```text
[ User request ]
        │
        ▼
[ Auth: who are you? ]
        │
        ▼
[ Authorization: what can you do? ]
        │
        ▼
[ Rate limit: per-user quota ]
        │
        ▼
[ PII redaction: remove sensitive data ]
        │
        ▼
[ Backend proxy: API key ]
        │
        ▼
[ Provider call ]
        │
        ▼
[ Response validation ]
        │
        ▼
[ Audit logging ]
        │
        ▼
[ Return to user ]
```

Key insight: security boundary phải nằm **trước model**. Nếu model thấy dữ liệu, bạn đã mất một lớp bảo vệ.

## 3. Core intuition: every LLM request crosses trust boundaries

Mỗi request LLM production đi qua nhiều trust boundaries:

```text
user → app → backend → provider → model
```

Mỗi boundary là một decision point:

- **Auth:** who are you?
- **Authorization:** what can you do?
- **Rate limit:** per-user quota.
- **PII redaction:** remove sensitive data.
- **Backend proxy:** API key.
- **Response validation:** check output.
- **Audit logging:** who did what.

<Callout type="danger" title="Security boundary phải nằm trước model">
Đừng đưa dữ liệu/secret/action không được phép vào prompt rồi hy vọng model không tiết lộ hoặc không dùng. Nếu model thấy dữ liệu, bạn đã mất một lớp bảo vệ quan trọng.
</Callout>

## 6. Never expose API keys in frontend

### Why?

```text
Frontend React → API key → provider
```

Hậu quả:

- key bị lộ,
- attacker dùng key để tạo bill,
- không enforce được per-user quota,
- không filter được documents theo permission,
- không audit được tool calls,
- dữ liệu nhạy cảm đi ra ngoài không kiểm soát.

### Fix: backend proxy

```text
Frontend → backend proxy → API key → provider
```

<Callout type="danger" title="API key không bao giờ expose trong frontend">
API key phải nằm trong backend, không bao giờ trong frontend. Frontend chỉ gọi backend proxy, không gọi provider trực tiếp.
</Callout>

## 9. Backend proxy pattern

### Why?

```text
Frontend → backend proxy → API key → provider
```

### Why?

- API key không expose trong frontend.
- Per-user quota.
- Rate limit.
- PII redaction.
- Audit logging.
- Tool permission.
- Response validation.

### Why?

```python
from fastapi import FastAPI, Request, HTTPException
from pydantic import BaseModel

app = FastAPI()


class ChatRequest(BaseModel):
    messages: list[dict]
    user_id: str


@app.post("/chat")
async def chat(request: ChatRequest, user: dict = Depends(get_current_user)):
    if not check_permission(user, "chat"):
        raise HTTPException(status_code=403, detail="permission denied")
    if not rate_limiter.allow(user["id"]):
        raise HTTPException(status_code=429, detail="rate limit exceeded")
    messages = redact_pii(request.messages)
    response = provider.chat(messages)
    response = validate_response(response)
    audit_log(user["id"], request, response)
    return response
```

## 11. Secure RAG authorization

### Why?

RAG retrieve documents từ vector DB. Nếu không có authorization, user A có thể retrieve documents của user B.

### Why?

```text
[ User query ]
        │
        ▼
[ Retrieve documents ]
        │
        ▼
[ Filter by user permission ]
        │
        ▼
[ Rank documents ]
        │
        ▼
[ Return to user ]
```

### Why?

```python
def retrieve_with_auth(query: str, user_id: str, top_k: int = 5) -> list[dict]:
    results = vector_db.search(query, top_k=top_k * 2)
    authorized = [r for r in results if check_doc_permission(user_id, r["doc_id"])]
    return authorized[:top_k]
```

### Why?

- Document-level ACL: user A có thể đọc doc 1, không doc 2.
- Chunk-level ACL: user A có thể đọc chunk 1, không chunk 2.
- Tenant-level ACL: user A thuộc tenant X, không tenant Y.
- Role-based ACL: user A là admin, user B là user.

## 14. PII and secret redaction

### Why?

```text
User: "My email is john@example.com"
```

Nếu không redact, email đi ra provider.

### Why?

```python
import re


def redact_pii(text: str) -> str:
    patterns = {
        "email": r"\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}\b",
        "phone": r"\b\d{3}-\d{3}-\d{4}\b",
        "ssn": r"\b\d{3}-\d{2}-\d{4}\b",
        "credit_card": r"\b\d{4}-\d{4}-\d{4}-\d{4}\b",
    }
    for name, pattern in patterns.items():
        text = re.sub(pattern, f"[{name.upper()}_REDACTED]", text)
    return text


def redact_secrets(text: str) -> str:
    patterns = {
        "api_key": r"sk-[A-Za-z0-9]{20,}",
        "aws_key": r"AKIA[A-Z0-9]{16}",
        "github_token": r"gh[ps]_[A-Za-z0-9]{36}",
    }
    for name, pattern in patterns.items():
        text = re.sub(pattern, f"[{name.upper()}_REDACTED]", text)
    return text
```

<Callout type="warning" title="Regex masking có giới hạn">
Regex masking không phải biện pháp hoàn hảo. Cần NER-based redaction, LLM-based redaction, và human review cho dữ liệu nhạy cảm.
</Callout>

## 15. Secure logging

### Why?

```text

```

Nếu log có PII, log có thể leak.

### Why?

```python
def safe_log(user_id: str, request: dict, response: dict):
    log.info(
        "request_id=%s user_id=%s prompt_version=%s model=%s input_tokens=%d output_tokens=%d latency_ms=%d",
        request["request_id"],
        hash(user_id),
        request["prompt_version"],
        request["model"],
        request["input_tokens"],
        response["output_tokens"],
        response["latency_ms"],

```

### Why?

- Log không có PII.
- Log có hash user_id.
- Log có request_id, prompt_version, model, tokens, latency.

## 16. Tool security and capability permissions

### Why?

```text
Model: "I'll run delete_all_files() for you."
```

Tool không có permission.

### Why?

```python
def execute_tool(tool_name: str, arguments: dict, user_id: str) -> dict:
    if not validate_tool_call(tool_name, arguments):
        return {"error": "invalid arguments"}
    if not check_permission(user_id, tool_name, arguments):
        return {"error": "permission denied"}
    return run_tool(tool_name, arguments)
```

### Why?

| Tool | Permission | Risk |
|---|---|---|
| send_email | user | low |
| create_refund | user | high |
| delete_file | admin | high |
| read_file | user | low |
| write_file | user | medium |
| execute_command | admin | high |

### Why?

- Tool permission: user, admin.
- Tool validation: validate arguments.
- Tool execution: run with permission.
- Tool audit: log tool calls.
- Tool rate limit: per-user quota.

- invalid JSON rate,

### Why incident response?

```text
incident → detect → respond → recover → post-mortem
```

### Why incident response?

| Step | Mục đích |
|---|---|
| Detect | monitoring |
| Respond | triage, contain |
| Recover | restore service |
| Post-mortem | root cause, prevention |

### Why incident response?

- API key leak: rotate key, audit log.
- Prompt injection: block user, update defense.
- Data exfiltration: audit log, notify users.
- Quota abuse: rate limit, block user.
- Tool misuse: revoke permission, audit log.

## Implementation Playbook: Secure AI Proxy Skeleton

```python
import json
import re
import hashlib
import time
from dataclasses import dataclass
from fastapi import FastAPI, Request, HTTPException, Depends
from pydantic import BaseModel


app = FastAPI()


@dataclass
class User:
    id: str
    permissions: list[str]
    tenant_id: str


class ChatRequest(BaseModel):
    messages: list[dict]
    user_id: str


def authenticate(token: str) -> User:
    return User(id="user_123", permissions=["chat"], tenant_id="t_abc")


def check_permission(user: User, action: str) -> bool:
    return action in user.permissions


def redact_pii(text: str) -> str:
    patterns = {
        "email": r"\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}\b",
        "phone": r"\b\d{3}-\d{3}-\d{4}\b",
        "ssn": r"\b\d{3}-\d{2}-\d{4}\b",
    }
    for name, pattern in patterns.items():
        text = re.sub(pattern, f"[{name.upper()}_REDACTED]", text)
    return text


def redact_secrets(text: str) -> str:
    patterns = {
        "api_key": r"sk-[A-Za-z0-9]{20,}",
        "aws_key": r"AKIA[A-Z0-9]{16}",
    }
    for name, pattern in patterns.items():
        text = re.sub(pattern, f"[{name.upper()}_REDACTED]", text)
    return text


def rate_limiter(user_id: str, max_requests: int = 100) -> bool:
    return True


def audit_log(user_id: str, request: dict, response: dict):
    print(
        "request_id=%s user_id=%s tenant_id=%s model=%s input_tokens=%d output_tokens=%d latency_ms=%d",
        request.get("request_id"),
        hashlib.sha256(user_id.encode()).hexdigest(),
        request.get("tenant_id"),
        request.get("model"),
        request.get("input_tokens"),
        response.get("output_tokens"),
        response.get("latency_ms"),
    )


@app.post("/chat")
async def chat(request: ChatRequest, token: str = Depends(...)):
    user = authenticate(token)
    if not check_permission(user, "chat"):
        raise HTTPException(status_code=403, detail="permission denied")
    if not rate_limiter(user.id):
        raise HTTPException(status_code=429, detail="rate limit exceeded")
    messages = request.messages
    for msg in messages:
        msg["content"] = redact_pii(msg["content"])
        msg["content"] = redact_secrets(msg["content"])
    response = provider.chat(messages)
    audit_log(user.id, request, response)
    return response
```
