Information theory là toán của **thông tin** — đo "bất ngờ", "không chắc chắn", và "khoảng cách" giữa các phân phối. Nghe trừu tượng, nhưng nó giải thích trực tiếp vì sao **cross-entropy là loss cho classification** và **perplexity đánh giá LLM**. Bài này gộp các khái niệm lại với ứng dụng AI rõ ràng.

## Entropy — đo độ không chắc chắn

Entropy đo mức "khó đoán" của một phân phối. Phân phối đều (mọi kết quả như nhau) → khó đoán nhất → entropy max. Phân phối tập trung 1 kết quả → dễ đoán → entropy thấp.

```
H(X) = -Σ p(x) · log₂ p(x)      (bits)
```

```python
import numpy as np

def entropy(p):
    p = np.array(p)
    p = p[p > 0]            # bỏ 0 (log(0) vô nghĩa)
    return -np.sum(p * np.log2(p))

entropy([0.5, 0.5])         # 1.0 bit — đồng xu, tối đa bất ngờ
entropy([0.99, 0.01])       # ~0.08 bit — gần chắc chắn, ít bất ngờ
entropy([1.0])              # 0 — hoàn toàn chắc chắn
```

<Callout type="info" title="Bits vs nats">
Dùng `log₂` → đơn vị **bit** (thông tin). Dùng `ln` (log tự nhiên) → đơn vị **nat**. AI thường dùng nat (vì đạo hàm `ln` gọn). Hai cách chỉ khác hệ số log.
</Callout>

<Callout type="tip" title="Entropy giải thích perplexity">
Perplexity (đánh giá LLM) = `2^H` — "số lựa chọn trung bình mà model phân vân". Perplexity thấp = model tự tin (ít lựa chọn) = tốt. Perplexity 1 = hoàn toàn chắc chắn (entropy 0).
</Callout>

## Cross-entropy — loss tự nhiên cho classification

Cross-entropy đo **chi phí** khi dùng phân phối `q` (prediction) để mã hóa sự kiện thực sự đến từ `p` (truth):

```
H(p, q) = -Σ p(x) · log q(x)
```

Khi `p` là true label (one-hot, chỉ 1 vị trí = 1), công thức rút gọn:

```
H(p, q) = -log q(correct class)
```

Đó chính là **cross-entropy loss** bạn dùng hàng ngày:

```python
import torch
import torch.nn.functional as F

logits = torch.tensor([[2.0, 1.0, 0.1]])     # raw output model
target = torch.tensor([0])                    # class đúng là 0

loss = F.cross_entropy(logits, target)
# = -log(softmax(logits)[0])
```

<Callout type="warning" title="Dùng logit, không dùng probability">
`F.cross_entropy` nhận **logit** (raw output), bên trong nó tự `log_softmax` để ổn định số. Đừng truyền probability đã softmax rồi — sẽ mất ổn định (log của số rất nhỏ → NaN). Dùng `F.nll_loss` nếu đã có log-prob.
</Callout>

### Vì sao cross-entropy chứ không phải MSE?

Cho classification, MSE có gradient `∂L/∂z ∝ σ·(1-σ)` qua sigmoid → **vanishing** khi output gần 0/1. Cross-entropy (với softmax) có gradient gọn `predicted - target` — **không vanishing**. Đây là lý do toản học đứng sau việc dùng cross-entropy cho classification.

## KL Divergence — khoảng cách (không đối xứng)

KL divergence đo "khoảng cách" từ phân phối `p` (truth) sang `q` (prediction):

```
KL(p || q) = Σ p(x) · log(p(x) / q(x))
```

```python
def kl_divergence(p, q):
    p = np.array(p); q = np.array(q)
    mask = p > 0
    return np.sum(p[mask] * np.log(p[mask] / q[mask]))

kl_divergence([1, 0], [0.5, 0.5])   # 1.0 — sai hoàn toàn
kl_divergence([1, 0], [0.9, 0.1])   # ~0.15 — gần đúng
```

**Tính chất**:
- `KL ≥ 0`, và `= 0` khi `p = q`.
- **Không đối xứng**: `KL(p||q) ≠ KL(q||p)` — nên gọi "divergence" chứ không "distance".

<Callout type="danger" title="KL không đối xứng — cẩn thắc">
Vì `KL(p||q) ≠ KL(q||p)`, chọn chiều quan trọng:
- **Forward KL** `KL(p||q)`: q phải "che" mọi chỗ p có mass → q tránh bị 0 nơi `p > 0` (**mode-covering**).
- **Reverse KL** `KL(q||p)`: q chỉ cần khớp 1 mode của p → q tránh bị dàn trải (**mode-seeking**).
VAE dùng forward KL; GAN dùng reverse KL — đây là lý do chúng sinh ra kết quả khác nhau.
</Callout>

### Liên hệ với cross-entropy

```
H(p, q) = H(p) + KL(p || q)
```

Vì `H(p)` (entropy của truth) cố định, **tối thiểu cross-entropy ≡ tối thiểu KL** ≡ đưa prediction `q` gần truth `p`. Đây là lý do cốt lõi cross-entropy là loss cho classification.

## Ứng dụng KL trong AI

| Bài toán | Cách dùng KL |
|---|---|
| **Knowledge distillation** | Ép học sinh khớp logits giáo viên (soft labels) |
| **VAE** | `KL(q(z|x) || p(z))` ép posterior gần prior |
| **RLHF / PPO** | Thêm `KL(policy_new || policy_old)` tránh thay đổi quá nhiều |
| **Model compression** | Giảm tham số giữ phân phối output |

```python
# Distillation: KL giữa soft label (giáo viên) và học sinh
kl = F.kl_div(
    F.log_softmax(student_logits / T, dim=-1),
    F.softmax(teacher_logits / T, dim=-1),
    reduction="batchmean"
)
```

<Callout type="info" title="KL penalty trong RLHF">
Khi fine-tune LLM bằng RL (PPO), ta không cho policy tự do thay đổi — thêm penalty `KL(new || base)` để policy mới không lệch quá xa base model (tránh reward hacking, giữ khả năng ngôn ngữ). Đây là cơ chế chính của RLHF.
</Callout>

## Mutual information

Mutual information `I(X;Y)` đo **lượng thông tin chung** giữa 2 biến:

```
I(X;Y) = KL(P(X,Y) || P(X)·P(Y))
```

- `I = 0` khi X, Y độc lập.
- `I` lớn khi biết Y cho nhiều thông tin về X.

Dùng trong: feature selection (chọn feature nhiều thông tin với label), representation learning (maximize MI giữa input và representation), clustering evaluation.

## Điểm cần nhớ

- **Entropy** `H = -Σ p log p` đo độ khó đoán; phân phối đều → max, tập trung → thấp. Bits (log₂) vs nats (ln).
- **Perplexity = 2^H** — số lựa chọn trung bình model phân vân; thấp = tốt.
- **Cross-entropy** `H(p,q) = -Σ p log q`; với one-hot truth → `-log q(correct)` — loss tự nhiên cho classification.
- Dùng **logit** cho `F.cross_entropy` (tự softmax+log ổn định), đừng truyền probability.
- Cross-entropy thắng MSE cho classification vì gradient `predicted - target` không vanishing.
- **KL divergence** `KL(p||q)` ≥ 0, không đối xứng; forward (mode-covering) vs reverse (mode-seeking).
- `H(p,q) = H(p) + KL(p||q)` → tối thiểu cross-entropy ≡ tối thiểu KL.
- KL dùng trong distillation, VAE, **RLHF penalty**, model compression.
