Linear algebra là **ngôn ngữ** của AI. Mọi embedding, weight, attention score đều là vector/ma trận. Khi bạn hiểu cosine similarity, bạn hiểu **cách semantic search hoạt động**; khi hiểu eigenvalues, bạn hiểu PCA; khi hiểu SVD, bạn hiểu model compression. Bài này đi từ vector đến SVD, gắn mỗi khái niệm với ứng dụng AI thật.

## Vector — hình học và đại số

Vector là **danh sách số có thứ tự**. Có thể hiểu hai cách:

- **Đại số**: `v = [3, 4]` — 2 số (vector 2D).
- **Hình học**: mũi tên từ gốc tọa độ, có hướng và độ dài.

```python
import numpy as np

v = np.array([3, 4])
w = np.array([1, 0])

# Cộng vector (hình học: nối đầu mũi tên — "tip-to-tail")
v + w               # [4, 4]

# Nhân vô hướng (kéo dài/thu ngắn + đảo chiều nếu âm)
2 * v               # [6, 8]
-0.5 * v            # [-1.5, -2]
```

### Vector ở đâu trong AI

- **Embedding**: 1 vector `d` chiều (thường 768/1536/4096) biểu diễn "ý nghĩa" của token.
- **Weight**: 1 vector trong `linear layer` (1 neuron).
- **State**: vector `hidden state` trong RNN/Transformer.

## Dot product — phép toán cốt lõi

```python
v = np.array([3, 4])
w = np.array([1, 0])

np.dot(v, w)        # 3
v @ w               # 3 (Python 3.5+)
```

**Công thức**: `v · w = Σ vᵢ wᵢ = |v| |w| cos(θ)`

Ý nghĩa hình học: dot product đo **độ "cùng hướng"**. Nếu `θ=0` (cùng hướng) → max; `θ=90°` (vuông góc) → 0; `θ=180°` (ngược hướng) → min (âm).

### Dot product = projection

```python
# Chiếu v lên w (lấy độ dài của v theo hướng w)
v = np.array([3, 4])
w = np.array([1, 0])  # vector đơn vị

projection = v @ w  # 3 — độ dài của v theo hướng w
```

<Callout type="info" title="Dot product ở đâu trong AI?">
- **Attention**: `Q · K` (query dot key) tính độ liên quan giữa token. Token càng "liên quan" → logit càng lớn → attention càng cao.
- **Embedding similarity**: `emb_a · emb_b` đo độ giống nhau.
- **Logit của linear layer**: `W·x + b` — mỗi neuron là một dot product.
- **Margin**: `w·x + b` (SVM) quyết định class.
</Callout>

## Norm — độ dài vector

```python
v = np.array([3, 4])

np.linalg.norm(v)            # 5.0  (L2 mặc định)
np.linalg.norm(v, ord=1)     # 7    (L1: |3|+|4|)
np.linalg.norm(v, ord=np.inf)# 4    (L∞: max(|3|,|4|))
```

| Norm | Công thức | Đặc điểm |
|---|---|---|
| **L1** | `Σ |vᵢ|` | Promote sparsity (dùng cho L1 regularization) |
| **L2** | `√(Σ vᵢ²)` | Mượt, phổ biến nhất |
| **L∞** | `max(|vᵢ|)` | Chỉ phần tử lớn nhất |
| **Lp** | `(Σ |vᵢ|ᵖ)^(1/p)` | Tổng quát, p≥1 |

**Chuẩn hóa (normalize)** về vector đơn vị (độ dài = 1):

```python
v = np.array([3, 4])
v_unit = v / np.linalg.norm(v)     # [0.6, 0.8], norm = 1
```

**Unit vector** (vector đơn vị) chỉ giữ hướng, không quan tâm độ dài. Phổ biến trong embedding normalization, attention scaling.

## Cosine similarity — cách embedding hoạt động

Đây là **phép đo độ giống** phổ biến nhất trong AI:

```
cos(θ) = (v · w) / (|v| |w|)
```

Kết quả từ **-1 đến 1**:
- `1` = cùng hướng (rất giống)
- `0` = vuông góc (không liên quan)
- `-1` = ngược hướng

```python
def cosine_sim(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

emb_doc = np.array([0.2, 0.8, 0.1, 0.5])
emb_query = np.array([0.3, 0.7, 0.2, 0.4])

cosine_sim(emb_query, emb_doc)   # ~0.95 → rất giống
```

<Callout type="tip" title="Vì sao cosine chứ không phải Euclidean?">
Cosine similarity **bất biến với độ lớn** — chỉ quan tâm hướng, không quan tâm độ dài. Trong NLP, văn bản dài có embedding "độ lớn" khác văn bản ngắn, nhưng có thể cùng chủ đề. Cosine bỏ qua độ lớn, so sánh **hướng** (chủ đề) → hợp lý hơn cho semantic search.

Mẹo: nếu bạn normalize trước, `cosine_sim == dot product` → nhanh hơn. Đó là lý do vector database thường normalize embedding.
</Callout>

Quy trình RAG thực tế: embed query + tất cả chunk, tính cosine similarity với mỗi chunk, lấy top-K chunk giống nhất đưa vào LLM.

## Orthogonality — vuông góc

Hai vector **vuông góc** (orthogonal) khi `v · w = 0`. Hình học: góc 90°.

```python
v = np.array([1, 0])
w = np.array([0, 1])
np.dot(v, w)   # 0 — orthogonal
```

**Tính chất**:
- Vector orthogonal có **thông tin độc lập** — không "trùng lặp" nhau.
- Vector đơn vị orthogonal gọi là **orthonormal**.

<Callout type="info" title="Orthogonal ở đâu trong AI?">
- **Positional encoding**: dùng hàm sin/cos tạo các vector orthogonal theo vị trí.
- **Attention**: `Q·K` = 0 → hai token không liên quan.
- **Weight init**: khởi tạo orthogonal (nn.init.orthogonal_) giúp gradient ổn định.
- **SVD**: `U` và `V` là ma trận orthogonal — không trộn thông tin.
</Callout>

## Ma trận — mảng 2D số

Ma trận là **mảng 2D số**. Mỗi dòng/cột là vector. Trong AI, **một ma trận weight** là một "hàm biến đổi" từ input sang output.

```python
A = np.array([[1, 2],
              [3, 4]])
B = np.array([[5, 6],
              [7, 8]])

A + B               # cộng element-wise
A.T                 # transpose (đảo dòng/cột)
```

### Nhân ma trận — KHÔNG phải element-wise

```python
A = np.array([[1, 2],
              [3, 4]])      # shape (2,2)
B = np.array([[5, 6],
              [7, 8]])      # shape (2,2)

A @ B          # matmul, shape (2,2)
# [[1*5+2*7, 1*6+2*8],
#  [3*5+4*7, 3*6+4*8]]
# = [[19, 22], [43, 50]]

A * B          # element-wise (Hadamard) — KHÁC!
# [[5, 12], [21, 32]]
```

**Quy tắc shape**: `(m,n) @ (n,p) = (m,p)`. Số cột của A phải = số dòng của B.

<Callout type="danger" title="@ vs * — bug kinh điển">
`@` = nhân ma trận (transform); `*` = nhân element-wise. Trong AI, `W @ x` là linear transform (weight áp cho input); `mask * x` là gating element. Nhầm 2 cái này là bug rất khó tìm vì code vẫn chạy, chỉ ra kết quả sai.

PyTorch: `A @ B` ≡ `A.matmul(B)`. `A * B` ≡ `A.mul(B)` (element-wise).
</Callout>

## Linear transformations — matmul là phép biến đổi

Nhân ma trận `A @ x` là **phép biến đổi tuyến tính** (linear transformation) — nó "đẩy" vector `x` sang một vị trí khác.

```python
# Phép biến đổi: scale + rotate
A = np.array([[2, 0],    # scale x2 theo trục x
              [0, 0.5]])  # scale 0.5 theo trục y

x = np.array([1, 1])
A @ x   # [2, 0.5] — x bị kéo dài, y bị nén
```

**Tính chất**:
- `A @ (x + y) = A @ x + A @ y` (cộng tuyến tính).
- `A @ (c·x) = c·(A @ x)` (scale tuyến tính).
- **Composition**: `A @ (B @ x) = (A @ B) @ x` — áp biến đổi liên tiếp.

<Callout type="info" title="Linear layer = matmul + bias">
Mỗi linear layer trong neural network: `y = W @ x + b`. `W` là phép biến đổi, `b` là phép dịch. Mỗi layer là một "hàm" biến đổi input → output. Mạng sâu = composition của nhiều biến đổi: `W₃ @ (W₂ @ (W₁ @ x))`.
</Callout>

## Ma trận đơn vị, nghịch đảo, định thức

```python
A = np.array([[4, 7],
              [2, 6]])

np.linalg.inv(A)        # A⁻¹: sao cho A @ A⁻¹ = I
np.linalg.det(A)        # định thức
np.eye(2)               # I (đơn vị)
```

- **I (đơn vị)**: ma trận mà `A @ I = A` (như số 1 cho phép nhân). `I` là "phép biến đổi" không đổi gì.
- **A⁻¹ (nghịch đảo)**: "chia" ma trận. `A @ A⁻¹ = I`. Nghịch đảo là "phép biến đổi ngược". Chỉ tồn tại nếu `det(A) ≠ 0`.
- **Định thức**: về hình học, biểu diễn **thể tích biến đổi**. `det=0` → ma trận "phẳng" (rank thiếu), không nghịch đảo được.

<Callout type="warning" title="Đừng tính nghịch đảo để giải hệ">
Trong code thật, hiếm khi cần `inv(A)`. Để giải `Ax = b`, dùng `np.linalg.solve(A, b)` — ổn định số hơn nhiều. `inv` chỉ cho hiểu khái niệm.

Ma trận weight lớn nghịch đảo cũng tốn memory (n²) và dễ numerically unstable. Forward pass dùng `A @ x`, không dùng `inv(A)`.
</Callout>

## Hệ phương trình tuyến tính: Ax = b

```python
A = np.array([[3, 2],
              [1, -1]])
b = np.array([5, 0])

# Giải hệ phương trình: 3x + 2y = 5, x - y = 0
x = np.linalg.solve(A, b)
print(x)   # [1, 1] — nghiệm
```

**Hệ suy nghiệm**:
- **Square** (n×n, `det ≠ 0`): nghiệm duy nhất (dùng `solve`).
- **Overdetermined** (`m>n`, nhiều hơn phương trình): thường không có nghiệm chính xác → **least squares** (xấp xỉ).
- **Underdetermined** (`m<n`, ít hơn phương trình): vô số nghiệm.

<Callout type="info" title="Linear regression là least squares">
Linear regression giải hệ `Ax = b` với `A` là feature matrix (m mẫu × n feature), `b` là target. Hệ overdetermined (`m>n`), không có nghiệm chính xác → tìm `x` tối thiểu `‖Ax - b‖²` → đó chính là **least squares**. Đây là lý do `np.linalg.lstsq` giải linear regression.
</Callout>

## Least squares

```python
# Tìm x tối thiểu ‖Ax - b‖²
A = np.array([[1, 1], [1, 2], [1, 3], [1, 4]])  # design matrix
b = np.array([6, 5, 7, 10])

x, residuals, rank, sv = np.linalg.lstsq(A, b, rcond=None)
print(x)   # [3.5, 1.4] — line fit y = 3.5 + 1.4x
```

**Công thức**: `x = (AᵀA)⁻¹ Aᵀb`. (Code thực tế không dùng công thức này — dùng SVD để ổn định.)

## Trace of a matrix

```python
A = np.array([[1, 2],
              [3, 4]])
np.trace(A)   # 5 = 1 + 4 (tổng đường chéo)
```

**Tính chất**:
- `tr(A) = tr(Aᵀ)` — tổng đường chéo.
- `tr(A + B) = tr(A) + tr(B)`.
- `tr(AB) = tr(BA)` — **cyclic property** (cyclic permutations).
- `tr(A) = Σ eigenvalues` của A.

<Callout type="info" title="Trace trong AI">
- **Frobenius norm**: `‖A‖²_F = tr(AᵀA)` — phổ biến trong regularization.
- **L2 regularization**: `‖w‖²` — chính là norm của vector weight.
- **PCA**: `tr(Covariance) = total variance` — tổng thông tin.
</Callout>

## Rank — bậc ma trận

Rank = số vector dòng (hoặc cột) **độc lập tuyến tính**. Đo "thông tin thực sự" trong ma trận:

```python
A = np.array([[1, 2, 3],
              [2, 4, 6]])     # dòng 2 = 2×dòng 1
np.linalg.matrix_rank(A)      # 1 (chỉ 1 thông tin độc lập)

B = np.array([[1, 2, 3],
              [0, 1, 4]])     # 2 dòng độc lập
np.linalg.matrix_rank(B)      # 2
```

Rank thấp → ma trận dư thừa, có thể nén. Đây là nền tảng của **dimensionality reduction** (PCA, SVD).

## Eigenvalues và eigenvectors — tại sao quan trọng cho PCA

Cho ma trận `A`, eigenvector `v` là hướng **không đổi** khi nhân với `A` (chỉ bị co giãn):

```
A v = λ v      (λ là eigenvalue)
```

```python
A = np.array([[2, 0],
              [0, 3]])
eigenvalues, eigenvectors = np.linalg.eig(A)
# eigenvalues: [2, 3]
# eigenvectors: [[1,0],[0,1]] (trục tọa độ)
```

**Ý nghĩa**:
- Eigenvector `v` là hướng **không đổi** (v chỉ bị co giãn).
- Eigenvalue `λ` là hệ số co giãn (`|λ| > 1` → kéo dài, `|λ| < 1` → nén, `λ < 0` → đảo).

<Callout type="info" title="PCA dùng eigenvector">
PCA tìm các hướng **phương sai lớn nhất** trong data. Các hướng đó chính là eigenvector của ma trận hiệp phương sai. Eigenvalue lớn = hướng chứa nhiều thông tin → giữ lại; nhỏ → bỏ (giảm chiều). Trong AI, PCA giảm 1000 chiều embedding xuống 50 mà giữ phần lớn thông tin.

**Tính chất**:
- `tr(A) = Σ eigenvalues`
- `det(A) = Π eigenvalues`
- Symmetric matrix (như covariance) có eigenvalue thực và eigenvector orthogonal.
</Callout>

## SVD — Singular Value Decomposition

SVD phân tách **mọi ma trận** (kể cả không vuông) thành 3 phần:

```
A = U Σ Vᵀ
```

- `U`, `Vᵀ`: ma trận trực giao (xoay).
- `Σ`: ma trận đường chéo chứa **singular values** (sắp xếp giảm dần).

```python
A = np.random.rand(5, 3)      # ma trận 5×3
U, S, Vt = np.linalg.svd(A)
# U: (5,5), S: (3,), Vt: (3,3)
```

**SVD ≠ Eigen decomposition**:
- Eigendecomposition chỉ dùng cho **ma trận vuông** (n×n).
- SVD dùng cho **mọi ma trận** (m×n).
- SVD là "phiên bản tổng quát" của eigendecomposition.

### Truncated SVD — nén dữ liệu

```python
# Truncated SVD: giữ top-k singular value
U, S, Vt = np.linalg.svd(A)
k = 2
A_approx = U[:, :k] @ np.diag(S[:k]) @ Vt[:k, :]
# xấp xỉ A với rank k
```

<Callout type="tip" title="SVD cho nén dữ liệu">
Singular value lớn = "hướng" chứa nhiều thông tin. Giữ top-k singular value, bỏ phần còn lại → xấp xỉ ma trận gốc bằng ma trận thấp hạng. Đây là cách **LSA** (Latent Semantic Analysis) và nhiều hệ gợi ý hoạt động. Trong AI hiện đại, SVD cũng dùng để **xấp xỉ ma trận weight** (model compression).

**Ứng dụng**:
- **LSA**: giảm chiều term-document matrix → tìm chủ đề ẩn.
- **PCA**: SVD trên ma trận covariance → PCA.
- **Recommender**: SVD lên user-item matrix → hệ gợi ý.
- **Model compression**: xấp xỉ weight matrix → giảm tham số.
</Callout>

## Tổng kết: Linear algebra trong AI

<Callout type="info" title="Linear algebra là ngôn ngữ của AI">
- **Vector** = embedding, weight, state.
- **Dot product** = attention, logit, similarity.
- **Matmul** = linear transform (layer).
- **Norm** = regularization, normalize.
- **SVD/PCA** = giảm chiều, nén.
- **Cosine similarity** = semantic search.

Mỗi model AI lớn = hàng triệu ma trận nhân với nhau. Hiểu linear algebra là hiểu "ngôn ngữ" của model.
</Callout>

## Điểm cần nhớ

- **Vector** = danh sách số; dot product `v·w = |v||w|cos(θ)` đo độ cùng hướng.
- **Norm** L1 (sparsity), L2 (mượt), L∞; normalize = chia cho norm để độ dài = 1.
- **Cosine similarity** = dot product chuẩn hóa → đo độ giống hướng (bỏ qua độ lớn) — nền tảng semantic search/RAG.
- **Orthogonality** (`v·w = 0`) = thông tin độc lập; phổ biến trong positional encoding, attention.
- **`@` = matmul** (transform), `*` = element-wise (gating) — đừng nhầm, đây là bug kinh điển.
- **Linear transformation** = matmul; `A @ x` đẩy vector sang vị trí khác.
- Ma trận nghịch đảo chỉ tồn tại khi `det ≠ 0`; giải hệ `Ax=b` dùng `solve`, không dùng `inv`.
- **Least squares** = giải hệ overdetermined (`m>n`) → linear regression.
- **Trace** = tổng đường chéo, `tr(A) = Σ eigenvalues`, `tr(AB) = tr(BA)`.
- **Rank** = thông tin độc lập thực sự; rank thấp → nén được.
- **Eigenvalues/eigenvectors** của ma trận hiệp phương sai = nền tảng PCA (giảm chiều giữ thông tin).
- **SVD** phân tách mọi ma trận, giữ top-k singular value để nén/xấp xỉ hạng thấp.

Bài tiếp: **Calculus** — đạo hàm, gradient, và chain rule — cách backpropagation "lan truyền" gradient ngược qua mạng.
